C# Temelleri — Unity'nin dili

Unity'de oyununu canlandıran her tıklama, her hareket, her ses bir C# script'iyle yazılır. Bu bölüm sıfırdan başlar: programlama mantığı ne demek, C# neye benzer, değişken-koşul-döngü-fonksiyon nasıl yazılır. Sonunda küçük bir oyuncu hareket scripti yazıp Inspector'dan kontrol ediyor olacaksın.

4.1

Programlama mantığı — Hiç kod yazmadan kavra

Programlamayı korkutucu bir şey gibi anlatırlar ama özünde sadece adım adım talimat vermektir. Bir arkadaşına çay yapmasını anlatıyorsan, sen zaten programlıyorsun:

1
Çaydanlığa su koy
2
Ocağı yak
3
Su kaynayana kadar bekle
4
Eğer demlik dolu değilse, çay koy
5
Demli oluncaya kadar tekrar bekle
6
Bardağa boşalt

İşte bu bir algoritmadır — bir işi yapmanın adım adım planı. Programlama, tam olarak bu planı bilgisayarın anladığı bir dile çevirmektir. Her programlama dilinde — C#, Python, JavaScript, hepsinde — aynı 4 temel kavram vardır:

Sıra (Sequence)

Komutlar yukarıdan aşağı, sırayla çalışır. "Önce şunu, sonra şunu" mantığı.

Koşul (Condition)

"Eğer şu doğruysa bunu yap, değilse şunu yap." Yol ayrımları.

Döngü (Loop)

"Şu koşul doğru olduğu sürece tekrar et." Aynı işi defalarca yapmak.

Veri (Data)

Değerleri saklamak: sayılar, yazılar, listeler, objeler. Değişkenler.

Bu 4 kavram her dilde aynıdır — C#'ı öğrendiğinde aslında programlama mantığını öğreniyorsun. Sonra Python, JavaScript, hatta C++'a geçişin sadece sözdizimi (syntax) öğrenmek olur. Mantığı bir kez kavradın mı, gerisi pratik.
4.2

Neden C#? — Unity'nin tercih dili

Unity 2017'den önce iki dil destekliyordu: C# ve UnityScript (JavaScript benzeri). UnityScript artık yok. Bugün Unity'de yazılan her şey C#.

C# nedir, neye benzer?

C# (okunuşu "C sharp"), Microsoft tarafından 2000'lerde tasarlanmış bir programlama dilidir. Java'ya çok benzer; her ikisi de C ailesindendir. C#'ı şu özellikler avantajlı kılar:

Tip güvenli (type-safe): Bir değişken int ise, ona yanlışlıkla yazı atayamazsın. Hatalar derleme anında yakalanır — oyun çalışmadan önce.
Garbage collected: Bellek yönetimini C# halleder. C++'taki gibi new/delete ile uğraşmazsın.
OOP odaklı: Class, inheritance, interface gibi nesne yönelimli programlama özellikleri tam destekli.
Olgun ekosistem: 25 yıllık dil, devasa kütüphane desteği, mükemmel IDE'ler (Visual Studio, Rider).

C# vs diğer oyun dilleri

C# (Unity)

Kolay öğrenilir, hızlı derlenir, güvenlidir. Pek çok endüstri standart kütüphanesi mevcut. Doğrudan başlamak için ideal.

  • Hızlı geliştirme
  • Bellek otomatik yönetilir
  • Hata yakalama derleme anında
C++ (alternatif motorlar)

Daha güçlü ama daha karmaşık. Bellek elle yönetilir. Yavaş derleme. Performans-kritik AAA için tercih edilir.

  • En yüksek performans
  • Bellek elle yönetilir (ağır)
  • Öğrenme eğrisi dik
Performans endişesi yok: Unity C# kodunu IL2CPP ile derlerken C++ koduna çevirir. Yayınlanan oyun neredeyse native C++ hızında çalışır. Mobil ve konsol için bu standarttır.
4.3

İlk script'in — "Hello, Unity"

Önce klasik bir başlangıç: bir mesajı Console'a yazdıralım. Unity'de yeni script oluşturmak:

1
Project panelinde Assets > sağ tık > Create > C# Script.
2
Adını HelloUnity yap (boşluksuz, büyük harfle başlasın).
3
Çift tıkla — Visual Studio (veya VS Code) açılır.

Otomatik gelen boş şablon böyle:

HelloUnity.cs
using UnityEngine;

public class HelloUnity : MonoBehaviour
{
    // Start is called before the first frame update
    void Start()
    {

    }

    // Update is called once per frame
    void Update()
    {

    }
}

Şablonu satır satır oku

using UnityEngine; → Unity'nin tüm fonksiyonlarını ve sınıflarını kullanılabilir kılar.
public class HelloUnity : MonoBehaviour → "HelloUnity" adlı bir class oluşturuyorsun; MonoBehaviour'dan türetiyorsun (yani bu bir Unity Component).
void Start() → Sahne yüklendikten sonra, ilk frame öncesinde bir kez çalışır.
void Update() → Her frame'de bir kez çalışır (saniyede 60 kez veya daha fazla).

Şimdi mesaj yazdıralım

HelloUnity.cs
using UnityEngine;

public class HelloUnity : MonoBehaviour
{
    void Start()
    {
        Debug.Log("Merhaba, Unity!");
    }
}

Debug.Log() Unity Console'una mesaj yazar. Şimdi:

1
Dosyayı kaydet (Ctrl+S). Unity Editor'e geri dön.
2
Hierarchy'de bir GameObject oluştur (Create Empty). Adını "Manager" yap.
3
HelloUnity script'ini Project'ten Manager'a sürükle (Component olarak eklenir).
4
Play'e bas. Console'da "Merhaba, Unity!" yazısı görmelisin.
Dosya adı = class adı kuralı: Unity, dosya adı ile class adının aynı olmasını zorunlu tutar. HelloUnity.cs dosyasında HelloUnity class'ı olmalı. Aksi takdirde Component olarak eklenemez.
4.4

Değişkenler ve veri tipleri

Değişken = etiketlenmiş bir kutu. İçine bir değer koyarsın, sonra adıyla erişirsin. C#'ta değişken tiplidir — yani kutunun ne tür şey alacağını önceden belirtmen gerekir.

Temel veri tipleri

VariableExamples.cs
// Sayısal tipler
int can = 100;              // Tamsayı: -2 milyar ile +2 milyar arası
float hiz = 5.5f;           // Ondalıklı sayı (sonuna f koy)
double hassas = 3.14159;   // Daha hassas ondalık (Unity'de az kullanılır)

// Mantıksal
bool oyuncuYasiyor = true;  // true veya false

// Metin
string isim = "Ahmet";       // Yazı (çift tırnak)
char harf = 'A';            // Tek karakter (tek tırnak)

// Unity'ye özel tipler
Vector3 konum = new Vector3(0, 1, 0);  // 3D konum/yön
Color renk = Color.red;                  // Renk
GameObject hedef;                          // Henüz değer atanmamış (null)

İsimlendirme kuralları

İlk harf büyük olamaz (C# camelCase: oyuncuHizi, maxCan).
Türkçe karakter kullanma. hız yerine hiz.
Boşluk yok. oyuncu hizi ❌, oyuncuHizi ✓.
Anlamlı isim ver. x kötü, oyuncuHizi iyi.
Sayıyla başlayamaz: 3kare ❌, kare3 ✓.

public vs private — Inspector görünürlüğü

PlayerStats.cs
using UnityEngine;

public class PlayerStats : MonoBehaviour
{
    public float hareketHizi = 5f;       // Inspector'da görünür ve değiştirilebilir
    public int maksimumCan = 100;          // Inspector'da görünür

    private int mevcutCan;                  // Görünmez, sadece kod içinde kullanılır
    private float sonHareketZamani;          // Görünmez

    [SerializeField] private int bonusCan = 10; // private ama Inspector'da görünür (en iyi pratik)
}
Profesyonel pratik: Değişkenleri tercihen private tut, Inspector'da görmek istediklerini [SerializeField] ekle. Bu, dış sınıfların değiştirememesini sağlar ama Inspector'dan ayarlanmasına izin verir. public kullanımı kapsülleme açısından kötü pratiktir.
4.5

Operatörler — Matematik ve karşılaştırma

Aritmetik operatörler

MathOperators.cs
int a = 10;
int b = 3;

int toplam   = a + b;   // 13
int fark     = a - b;   // 7
int carpim   = a * b;   // 30
int bolum    = a / b;   // 3 (tam sayılarda kesir atılır!)
int kalan    = a % b;   // 1 (mod — bölümden kalan)

// Atama kısayolları
a += 5;   // a = a + 5;  → a artık 15
a -= 2;   // a = a - 2;  → a artık 13
a *= 2;   // a = a * 2;  → a artık 26
a++;       // a = a + 1;  → bir artır
a--;       // a = a - 1;  → bir azalt
Klasik tuzak: int / int sonucu her zaman int. 5 / 2 sonucu 2, "2.5" değil. Ondalık istiyorsan en az birini float yap: 5f / 2 → 2.5f.

Karşılaştırma operatörleri

Comparisons.cs
int can = 50;

bool esit       = (can == 100);   // false
bool esitDegil  = (can != 100);   // true
bool buyuk      = (can > 25);     // true
bool kucukEsit  = (can <= 50);    // true

// Mantıksal birleştirme
bool uygun = (can > 0) && (can < 100);  // VE — ikisi de doğruysa true
bool kritik = (can < 10) || (can == 0);   // VEYA — biri doğruysa true
bool degil = !(can == 100);                  // DEĞİL — tersini çevir
Önemli: Atama (=) ile karşılaştırma (==) farklıdır. can = 100 demek "can'ı 100 yap"; can == 100 demek "can 100'e eşit mi?". En sık yapılan hata budur — dikkat.

Yarı yoldasın — Şimdi ne biliyorsun?

Programlamanın 4 ayağını (sıra, koşul, döngü, veri) gördün. C#'ı neden Unity seçtiğini biliyorsun. İlk script'ini yazdın, değişkenler tanımladın, public/private farkını öğrendin, operatörlerle hesap yaptın. Şimdi kontrol akışı ve fonksiyonlara geçiyoruz — programa "karar verme" ve "yeniden kullanma" yetenekleri.

4.6

Koşullar — if, else, switch

Koşul = programın yol ayrımı. "Eğer can sıfırsa, oyuncu öldü; değilse, oynamaya devam et."

if / else

HealthCheck.cs
public int can = 100;

void Update()
{
    if (can <= 0)
    {
        Debug.Log("Oyuncu öldü!");
    }
    else if (can <= 25)
    {
        Debug.Log("Düşük can! Dikkat.");
    }
    else
    {
        // Normal durum, hiçbir şey yapma
    }
}

Süslü parantezleri unutma

Tek satırlık if'lerde süslü parantezler isteğe bağlıdır ama her zaman kullan — sonradan ekleme yaparken hata yapmazsın:

BracketsMatter.cs
// Kötü — başta çalışır, sonradan ekleme yapınca böcek
if (can <= 0) Debug.Log("Öldü");

// İyi — her zaman bu
if (can <= 0)
{
    Debug.Log("Öldü");
}

switch — Çoklu seçim daha temiz

DifficultySelector.cs
public int zorluk = 2;

void Start()
{
    switch (zorluk)
    {
        case 1:
            Debug.Log("Kolay");
            break;
        case 2:
            Debug.Log("Normal");
            break;
        case 3:
            Debug.Log("Zor");
            break;
        default:
            Debug.Log("Bilinmeyen zorluk");
            break;
    }
}
Ne zaman switch? 3+ aynı değişkenle eşitlik kontrolü yapacaksan switch daha okunabilirdir. 2 if/else yeterliyse if kullan.
4.7

Döngüler — for, while, foreach

for — Belirli sayıda tekrarla

SpawnEnemies.cs
public GameObject dusmanPrefab;

void Start()
{
    for (int i = 0; i < 10; i++)
    {
        Vector3 konum = new Vector3(i * 2f, 0, 0);
        Instantiate(dusmanPrefab, konum, Quaternion.identity);
    }
}

for üç parçadan oluşur, noktalı virgüllerle ayrılır: başlangıç; koşul; her döngü sonrası. Yukarıdaki örnek 10 düşman üretir, her birini 2 birim sağa kayık konumlandırır.

while — Koşul doğru olduğu sürece

CountdownExample.cs
int kalanSure = 10;

while (kalanSure > 0)
{
    Debug.Log("Kalan: " + kalanSure);
    kalanSure--;
}
Debug.Log("Süre bitti!");
Sonsuz döngü tuzağı: while koşulunu içeride değiştirmeyi unutursan döngü sonsuza dek çalışır, Unity donar. Yukarıdaki örnekte kalanSure-- kritik. Çıkarsan, ekranı kapatmak zorunda kalırsın.

foreach — Koleksiyon üzerinde gez

EnemyHealer.cs
void HeralAllEnemies()
{
    GameObject[] dusmanlar = GameObject.FindGameObjectsWithTag("Enemy");

    foreach (GameObject dusman in dusmanlar)
    {
        Enemy e = dusman.GetComponent<Enemy>();
        e.can = e.maxCan;
    }
}
foreach ne zaman? Bir dizinin/listenin her elemanı üzerinde sırayla iş yapacaksan. İndeks gerekmiyorsa foreach daha temiz. İndeks gerekliyse (örn. "üçüncü elemana farklı işlem") for kullan.
4.8

Fonksiyonlar (Metotlar) — Yeniden kullanılabilir kod

Aynı kodu defalarca yazmak yerine, bir kez yaz ve gerektiğinde çağır. C#'ta buna metot denir (Java/JS'de "fonksiyon").

Temel sözdizimi

MethodBasics.cs
public class Player : MonoBehaviour
{
    public int can = 100;

    // Parametresiz, dönüş tipsiz (void)
    void Olum()
    {
        Debug.Log("Karakter öldü");
        Destroy(gameObject);
    }

    // Parametreli
    void HasarAl(int miktar)
    {
        can -= miktar;
        if (can <= 0)
        {
            Olum();
        }
    }

    // Geriye değer döner
    bool YasiyorMu()
    {
        return can > 0;
    }

    void Update()
    {
        if (Input.GetKeyDown(KeyCode.H))
        {
            HasarAl(25);  // H'ye basınca 25 hasar al
        }
    }
}

Anatomisi

Erişim: public, private, protected. Diğer scriptler bu metodu görüp çağırabilsin mi?
Dönüş tipi: void = dönmüyor; int = int döner; bool = true/false döner; vb.
İsim: Büyük harfle başlar (PascalCase). Action verb tercih edilir: HasarAl, HesaplaToplam.
Parametreler: Parantez içinde tip ve isim. Birden fazlaysa virgülle ayrılır.
Gövde: Süslü parantezler arası. Eğer dönüş tipi void değilse, return ile değer döndür.

Method overloading — Aynı isim, farklı parametreler

Overloading.cs
// Aynı isimde 2 farklı metot — parametreleri farklı
void HasarAl(int miktar)
{
    can -= miktar;
}

void HasarAl(int miktar, string hasarTuru)
{
    can -= miktar;
    Debug.Log("Hasar türü: " + hasarTuru);
}

// Çağırırken hangisini istediğine göre
HasarAl(10);
HasarAl(10, "Yangın");
4.9

Class — Veri ve davranışı birlikte tutmak

Bir class, ilişkili değişkenleri ve metotları tek bir paket altında toplar. Şu ana kadar her script'in zaten bir class'tı (HelloUnity, Player...). MonoBehaviour'dan türemiş bir class, Component olur.

Pure class — MonoBehaviour'sız veri tutucu

WeaponInfo.cs
// MonoBehaviour'dan türemiyor — GameObject'e eklenmez, sadece veri taşır
[System.Serializable]
public class WeaponInfo
{
    public string isim;
    public int hasar;
    public float menzil;

    public WeaponInfo(string ad, int dmg, float r)
    {
        isim = ad;
        hasar = dmg;
        menzil = r;
    }
}

Bu class'ı bir MonoBehaviour içinde kullanabilirsin:

PlayerInventory.cs
public class PlayerInventory : MonoBehaviour
{
    public WeaponInfo[] silahlar;   // Inspector'da düzenlenebilir liste

    void Start()
    {
        foreach (WeaponInfo w in silahlar)
        {
            Debug.Log(w.isim + " - hasar: " + w.hasar);
        }
    }
}
[System.Serializable] attribute'u, class'ın Inspector'da görünebilmesini sağlar — değerlerini Editor'den ayarlayabilirsin. MonoBehaviour'dan türeyen class'larda buna gerek yoktur (zaten serileştirilir); ama düz veri class'larında şarttır.

Class'lar, inheritance (kalıtım), interface'ler, polymorphism gibi daha derin konuları Bölüm 25 — C# İleri'de göreceğiz. Şimdilik şu kadarını bil: her script'in bir class, ve class'lar veri+metot paketi.

Pratik: 3 mini proje

P1

Tuşa basınca console mesajı

Hedef: Boşluk tuşuna basınca Console'a "Boşluk basıldı!" yaz.

1
Yeni script: KeyListener.cs.
2
Update içinde: if (Input.GetKeyDown(KeyCode.Space)) Debug.Log("Boşluk basıldı!");
3
Empty GameObject'e ekle. Play'e bas, boşluğa bas, Console'u izle.
Tamamlama Kontrolü
  • Script çalışıyor
  • Boşluk her basıldığında yeni log satırı çıkıyor
Input + Debug.Log + if'in bir araya gelişini hisset.
P2

Hasar sistemi

Hedef: Inspector'dan maxCan ayarla, H tuşuna basınca 10 hasar al, can sıfırlanınca "Öldün" yaz.

1
PlayerHealth.cs oluştur. public int maxCan = 100; ve private int can;
2
Start'ta can = maxCan;
3
void HasarAl(int miktar) metodu yaz. Update'te H tuşuna basınca HasarAl(10).
4
HasarAl içinde if (can <= 0) Debug.Log("Öldün");
Tamamlama Kontrolü
  • maxCan Inspector'dan ayarlanabiliyor
  • H her bastığında can düşüyor
  • Can sıfırda "Öldün" mesajı çıkıyor
Metot, koşul ve Inspector entegrasyonunu birleştir.
P3

For döngüsü ile küp dizisi

Hedef: Start'ta for döngüsüyle 5 küp oluşturup x ekseninde sıralı yerleştir.

1
CubeRow.cs. public GameObject kupPrefab;
2
Start içinde: for (int i = 0; i < 5; i++)
3
Döngü içinde: Vector3 konum = new Vector3(i * 2f, 0, 0); Instantiate(kupPrefab, konum, Quaternion.identity);
4
Önce bir Cube hazırlayıp Prefab yap, sonra Inspector'dan kupPrefab'a ata.
Tamamlama Kontrolü
  • Play'de 5 küp x ekseninde sıralı oluşuyor
  • Aralarındaki mesafe 2 birim
  • For döngüsü doğru çalışıyor
For döngüsü ile Prefab spawn — çok kullanacağın bir desen.
