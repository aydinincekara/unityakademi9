12. Animation Sistemi — Mecha-Nim'in Kalbi

Bir karakter nasıl koşar, zıplar, ölür? Bir kapı nasıl açılır? Unity'nin Mecanim sistemi, Animator Controller'ı, State Machine'i, Blend Tree'leri ve Avatar Mask'ı bu bölümde adım adım çözüyoruz.

LTS + Unity 6 ⏱ ~70 dk 🟡 Orta
12.1

Animation Sistem Mimarisi — 3 Katman

Unity'de bir karakteri canlandırmak için 3 farklı bileşen birlikte çalışır:

1. Animation Clip

Tek bir hareket. "Idle", "Run", "Jump" gibi. .anim uzantılı veya FBX'ten gelen.

2. Animator Controller

Clip'lerin durum makinesi (State Machine). Hangi clip, ne zaman, hangi şartlarla çalsın? .controller dosyası.

3. Animator Component

GameObject'e eklenir. Controller'ı çalıştırır, runtime'da parametre değiştirir.

Bu üçlü hiyerarşi mimaride çok güçlüdür: aynı Animator Controller'ı 100 farklı karakter kullanabilir; sadece Avatar değişir.

12.2

Animation Clip — Tek Bir Hareket

Bir Animation Clip, property'lerin zaman içinde nasıl değiştiğini tanımlar. Bir kapıyı 0. saniyede 0°'de, 2. saniyede 90°'de göstermek bir clip'tir.

İki Tür Clip

TürGeliş Yeri
Imported ClipFBX'in içinden gelir. Read-only — silinemez, sadece import settings'ten değiştirilir.
Authored Clip (.anim)Unity'de Window → Animation → Animation ile sıfırdan yapılır. Düzenlenebilir.

Animation Window — Clip Düzenleme

Bir GameObject seçili iken Ctrl+6 ile Animation Window'u açarsın. Üst kısımda clip seçicisi, sol tarafta property listesi (Position.x, Rotation.y, MyScript.health...), sağ tarafta timeline.

  • Record (kırmızı top) tuşu: ON iken yaptığın her değişiklik o anki frame'e keyframe olarak kaydedilir.
  • Add Property butonu: Hangi property animate edilsin?
  • Keyframe'lere sağ tıkla: Linear / Constant / Free / Clamped Auto interpolation seçilebilir.
  • Curves view: Dopesheet yerine Bezier eğrileri olarak görüntüle (ease-in/out yapmak için).

Animation Event — Clip İçinden Kod Çağırma

Animasyonun belirli bir anında bir fonksiyon çağırmak istersen Animation Event ekle. Klasik örnek: "kılıç swing'i 0.4. saniyede MakeDamage() fonksiyonunu tetiklesin".

using UnityEngine; { public class Attacker : MonoBehaviour { public void OnAttackHit() // Animation Event tetikler { Debug.Log("Hasar verme zamanı!"); // hitbox enable, ses çal, particle spawn... } } }

Animation Window'da timeline üzerine sağ tıkla → "Add Animation Event" → fonksiyon adını gir.

12.3

Animator Controller — State Machine

Birden fazla clip'i nasıl bir araya getirip "Idle → Run → Jump → Idle" akışı kuracağız? Animator Controller bunu yapar. Window → Animation → Animator ile açılan pencerede grafik bir state machine görürsün.

State (Durum)

Her dikdörtgen bir state'tir, içinde bir clip vardır. State'ler arasında Transition (geçiş) okları bulunur.

  • Entry: Yeşil ok. Animator çalışmaya başladığında ilk hangi state'e geçilecek?
  • Default State: Turuncu state. Entry'nin gittiği yer.
  • Any State: Mavi. Herhangi bir state'ten oraya geçişi tetikleyebilir (ölüm animasyonu için ideal).
  • Exit: Sub-state machine'den çıkış noktası.

Transition (Geçiş)

İki state arasındaki ok. Üstüne tıklayınca Inspector'da ayarları açılır:

⚙️ Transition Settings
Has Exit Time✓
Exit Time0.92
Fixed Duration✓
Transition Duration0.25
Transition Offset0
Interruption SourceNone ▾
ConditionsSpeed Greater 0.1
  • Has Exit Time: ✓ ise mevcut clip'in belirli %'sine geldikten sonra geçer (Attack animasyonu bitmeden Idle'a geçmesin). KAPALI ise koşul sağlandığı anda hemen geçer (Idle → Run gibi anında geçişlerde).
  • Transition Duration: İki clip arasında blend süresi. 0.25s tipik. 0 verirsen anlık atlama (snap).
  • Conditions: Geçiş için sağlanması gereken parametre durumu.
12.4

Animator Parametreleri

Kod ile Animator'a "değer" verebileceğin köprü. 4 tür var:

TipKullanım
FloatSürekli değer (Speed, Health)
IntTamsayı (WeaponType: 0=Sword, 1=Bow)
Booltrue/false (IsGrounded, IsCrouching)
TriggerTek seferlik tetikleyici (Jump, Attack). Auto-reset.

Koddan Parametre Değiştirme

using UnityEngine; { public class PlayerAnim : MonoBehaviour { private Animator anim; private void Awake() => anim = GetComponent<Animator>(); { private void Update() { float speed = Input.GetAxis("Vertical"); anim.SetFloat("Speed", speed); { if (Input.GetButtonDown("Jump")) anim.SetTrigger("Jump"); { anim.SetBool("IsGrounded", CheckGround()); } } }
⚠️ String Yerine Hash Kullan (Performans)

SetFloat("Speed", value) her seferinde string'i hash'e çeviriyor. Hot path'lerde önceden hash'leyip kullan:

static readonly int SpeedHash = Animator.StringToHash("Speed");
anim.SetFloat(SpeedHash, value);
12.5

Blend Tree — Birden Fazla Clip'i Karıştırma

Karakter ileri yürürken sola/sağa dönmesi: bu 3 farklı clip değil, tek bir Blend Tree state'tir. Float parametreye göre clip'ler arasında linear interpolation yapılır.

1D Blend Tree

Tek bir float parametreyle clip'leri sıralarsın:

  • Speed = 0 → Idle clip
  • Speed = 0.5 → Walk clip
  • Speed = 1.0 → Run clip

Speed = 0.7 olduğunda Unity Walk ve Run'ı %30/%70 oranında karıştırır → ara hız animasyonu.

2D Blend Tree — Strafe Karışımı

İki parametre kullanır (örn. SpeedX ve SpeedZ):

  • (0,0): Idle
  • (0,1): Run Forward
  • (0,-1): Run Back
  • (1,0): Strafe Right
  • (-1,0): Strafe Left
  • (0.7, 0.7): Diagonal — Unity 4 clip'i karıştırır

2D Blend Tree'nin 3 alt tipi vardır: 2D Simple Directional (hareket yönü), 2D Freeform Directional (her noktada bir clip), 2D Freeform Cartesian (XY koordinat sistemi).

💡 Locomotion Pattern

AAA oyunlarda karakter hareketi tipik olarak tek bir 2D Freeform Cartesian Blend Tree ile çözülür: 8 yön + idle = 9 clip. Animator graph'ta sadece bir state vardır; karmaşıklık Blend Tree'nin içindedir.

🛑 Mini Checkpoint

Clip, Controller, State Machine, Parametreler, Blend Tree — temeli kurduk. Şimdi gelişmiş konulara geçiyoruz: Layers (üst beden alt beden ayrımı), Avatar Mask, Root Motion ve State Machine Behaviours.

12.6

Animator Layers — Üst/Alt Beden Ayrımı

Karakter koşarken aynı anda silah ateş etmeli. Tek bir state machine'de bu kombinasyonu yönetmek (Run+Shoot, Idle+Shoot, Walk+Shoot, ...) çok karmaşıklaşır. Çözüm: iki layer.

  • Base Layer: Tüm beden (locomotion: Idle/Walk/Run).
  • Upper Body Layer: Sadece üst beden (Shoot/Reload/Aim).

Animator penceresinin sol üstündeki Layers sekmesinden yeni layer eklenir. Her layer'ın kendi state machine'i vardır.

Layer Settings

⚙️ Layer — Upper Body
Weight1
MaskM_UpperBody ▾
BlendingOverride ▾
Sync☐
IK Pass☐
  • Weight: 0-1 arası. 0 = bu layer hiç çalışmıyor, 1 = tam etki. Koddan anim.SetLayerWeight(1, 0.5f) ile değiştirilebilir.
  • Mask: Avatar Mask ile hangi kemiklerin etkileneceği seçilir.
  • Blending — Override: Bu layer'ın değerleri alt layer'ı override eder. Additive: Alt layer'ın üstüne eklenir.

Avatar Mask Oluşturma

Project window'da sağ tıkla → Create → Avatar Mask. Mask'a çift tıklayıp Humanoid bölümünden vücudun hangi parçalarının etkileneceğini seç (yeşil = aktif). "Sadece kollar ve baş" demek için bacakları kapat.

12.7

Root Motion — Karakter Hareketi Nereden Geliyor?

Karakterinin pozisyon değişimini animasyondan mı alacaksın yoksa script'ten mi?

Root Motion ONRoot Motion OFF
Animasyon karakterin pozisyonunu hareket ettirir. Realistic koşu, dans, savaş sahneleri için. Animasyon "olduğu yerde" oynar. Pozisyonu CharacterController/Rigidbody/transform script ile değişir. FPS oyunlarda standard.

Animator Component'inde Apply Root Motion checkbox'ı kontrol eder. ON ise FBX import'taki "Bake Into Pose" ayarları kritik olur.

💡 OnAnimatorMove

Root Motion'u kısmen kontrol etmek istersen MonoBehaviour'a şu fonksiyonu ekle:

void OnAnimatorMove() {
    // anim.deltaPosition: animasyonun bu frame'deki pozisyon değişimi
    // anim.deltaRotation: rotation değişimi
    // Burada özel hesap yapabilir, transform'u kendin uygulayabilirsin
    transform.position += anim.deltaPosition * 2f;  // 2x hızlı
}
12.8

State Machine Behaviour — State'e Script Ekleme

Bir state'e girince, içindeyken veya çıkınca kod çalıştırmak istersen StateMachineBehaviour kullan. State seçili iken Inspector'da "Add Behaviour" butonu vardır.

using UnityEngine; { public class AttackState : StateMachineBehaviour { public override void OnStateEnter(Animator a, AnimatorStateInfo s, int i) { Debug.Log("Attack state başladı"); // hitbox aç, ses çal } { public override void OnStateExit(Animator a, AnimatorStateInfo s, int i) { Debug.Log("Attack state bitti"); // hitbox kapat } { public override void OnStateUpdate(Animator a, AnimatorStateInfo s, int i) { // Her frame, state aktifken çağrılır } } }

Bu pattern Animation Event'lere alternatiftir — clip değiştiğinde Event ölür, ama state behaviour state'e bağlıdır.

🎮 Gerçek Dünya: Cuphead'in Animator Cinleri

Cuphead (Studio MDHR, 2017) tamamen 1930'lar tarzı el-çizimi animasyonlardan oluşur. Hiçbir 3D model yok — her şey sprite sheet. Ama oyun aslında Unity'de yapıldı.

Ekibin yaklaşımı şöyleydi: Her boss için ortalama 50,000+ kare sprite çizildi (kağıda fırçayla). Bu kareler Unity'ye 2D Sprite Multiple olarak import edildi, animation clip'lere dönüştürüldü. Tek bir boss'un Animator Controller'ı 30-60 state içeriyordu: Phase1_Idle, Phase1_Attack_Bullet, Phase1_Transition, Phase2_Idle, ...

Phase geçişleri tamamen Animator Parameters ile yönetildi: int CurrentPhase, bool IsAngry, trigger TakeDamage. State Machine Behaviour ile her phase girişinde müzik tempo'su, partikül emisyonu ve attack frequency değiştirildi. Cuphead'in Unity Animator'ı geleneksel Mecanim'in ötesinde, tam bir oyun mantığı altyapısına dönüştürülmüştü.

🛠️ Uygulama: 3 Proje

Proje 1: Açılan Kapı (Authored Clip)

🎯 Hedef: Bir küpten kapı yapıp Animation Window ile açılma animasyonu yarat.

  • Bir Cube oluştur, Scale (0.1, 2, 1) — kapı şekli.
  • Window → Animation → Animation. Pencere açıkken küp seçili: "Create" butonu çıkar.
  • A_DoorOpen.anim olarak kaydet (Assets/Animations/).
  • Record (kırmızı) ON. Frame 0: Rotation Y = 0. Frame 60: Rotation Y = 90. Record OFF.
  • Otomatik oluşan Animator Controller'da bu clip Default. Play tuşuna bas — kapı dönmeli.
  • BONUS: Animation Event ekle, "OnDoorOpened" fonksiyonu Debug.Log("açıldı") yazsın.

Proje 2: Karakter Locomotion (Blend Tree)

🎯 Hedef: WASD ile karakter Idle ↔ Walk ↔ Run geçişi.

  • Mixamo'dan Idle, Walking, Running animasyonlarını indir (in-place, no skin).
  • Karakter prefab'ına Animator Controller oluştur, Bölüm 11'deki karakterle bağla.
  • Default state olarak Blend Tree (1D). Parameter = "Speed" (float).
  • Threshold: 0 → Idle, 0.5 → Walk, 1.0 → Run.
  • Script: anim.SetFloat("Speed", Mathf.Clamp01(input.magnitude * (isRunning ? 1 : 0.5f)))

Proje 3: Layered Upper Body Shoot

🎯 Hedef: Karakter koşarken sol click ile el silahı atış animasyonu (sadece üst beden).

  • Avatar Mask oluştur (Create → Avatar Mask). Bacakları ve root'u kapat, üst bedeni aç.
  • Animator'da yeni Layer ekle, Mask alanına M_UpperBody'yi sürükle, Blending = Override.
  • Bu layer'da Empty (default) ve Shoot state'i. Shoot, "ShootTrigger" parametresiyle tetiklensin.
  • Script: if (Input.GetMouseButtonDown(0)) anim.SetTrigger("ShootTrigger");
  • Karakter koşarken ateş edebilmeli — alt beden koşmaya devam, üst beden ateş eder.
