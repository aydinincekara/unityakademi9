C# İleri — Coroutine, Event, ScriptableObject, async/await, LINQ

Coroutine derinlikleri, delegate & event sistemleri, ScriptableObject ile veri mimarisi, async/await, generics, LINQ — profesyonel Unity kodu için araç kutusu.

⏱ Okuma süresi: ~32 dk 💻 12 ileri kavram 🎯 Zorluk: İleri
25.1

🎓 Neden İleri C#?

Temel MonoBehaviour bilgisi ile küçük oyunlar yapılır. Ama bir oyun büyüdükçe — onlarca düşman, karmaşık envanter, kayıt yükleme, ağ — kodun da büyür. İyi mimari olmadan 50 dosyalı proje çığ gibi karmaşıklaşır.

Bu bölümde kodu modüler, yeniden kullanılabilir, test edilebilir yapan C# araçlarına bakıyoruz. Bunlar Unity'ye özgü değil — saf C# özellikleri — ama Unity'de profesyonelce kullanmanın yolları farklı.

🎮 Gerçek Dünyadan: Cuphead'in 8000+ Coroutine'i

Cuphead'in geliştirici ekibi (Studio MDHR), her bossun saldırı düzenini koroutine'lerle yazdı. Her saldırı bir IEnumerator → fazlar arası geçiş, animasyon eşitlemesi, fizik atışları zamanlamalı. 30+ boss, her birinde onlarca saldırı → toplam 8000+ coroutine. Unity'nin coroutine sistemi bu boyutta yazılmış AAA-kalitesi kodun belkemiği oldu.

25.2

⏳ Coroutine — Derinlemesine

Coroutine, zamana yayılmış bir fonksiyon. Normal fonksiyon başlar-biter bir frame'de; coroutine birden fazla frame'e yayılır, "bekle, devam et" yapabilir. 

private IEnumerator FadeOut(float duration)
{
    float elapsed = 0;
    var color = sprite.color;
    while (elapsed < duration)
    {
        elapsed += Time.deltaTime;
        color.a = 1 - (elapsed / duration);
        sprite.color = color;
        yield return null;  // bir frame bekle
    }
    sprite.gameObject.SetActive(false);
}

// Başlatma:
StartCoroutine(FadeOut(2f));

yield Return Türleri

  • yield return null; — Bir frame bekle (sonraki Update'te devam).
  • yield return new WaitForSeconds(2f); — 2 saniye bekle.
  • yield return new WaitForSecondsRealtime(2f); — TimeScale=0 olsa bile gerçek 2 saniye.
  • yield return new WaitForEndOfFrame(); — Frame sonunda (ekran çizildikten sonra).
  • yield return new WaitForFixedUpdate(); — Sonraki FixedUpdate'i bekle.
  • yield return new WaitUntil(() => player.health <= 0); — Koşul sağlanana kadar.
  • yield return new WaitWhile(() => isPaused); — Koşul yanlış olana kadar.
  • yield return StartCoroutine(OtherRoutine()); — Diğer coroutine bitene kadar.

Coroutine Durdurma

private Coroutine activeRoutine;

private void StartFade()
{
    if (activeRoutine != null) StopCoroutine(activeRoutine);
    activeRoutine = StartCoroutine(FadeOut(2f));
}

private void OnDisable()
{
    // GameObject deaktive olunca tüm coroutine'leri durdur
    StopAllCoroutines();
}
⚠️ Coroutine'in Tuzakları:
  • GameObject deaktive olursa coroutine otomatik durur, MonoBehaviour silinirse de.
  • Coroutine, çağıran MonoBehaviour'a aittir — başka MB'da çalışmaz.
  • WaitForSeconds çoğu zaman ufak garbage üretir; cache'le: private static WaitForSeconds wait1 = new WaitForSeconds(1f);
  • Çok sayıda coroutine = performans yükü. 1000+ aktif coroutine için Job System veya async düşün.
25.3

📡 Delegate & Event — Loose Coupling

Bir sistem "olay olduğunda haber ver" mantığı ister. Delegate ve event, sınıfların birbirine doğrudan referans olmadan iletişim kurmasını sağlar.

Basit Delegate

public delegate void EnemyDiedHandler(Vector3 position, int xp);

public class Enemy : MonoBehaviour
{
    public EnemyDiedHandler OnDied;

    private void Die()
    {
        OnDied?.Invoke(transform.position, 50);
        Destroy(gameObject);
    }
}

// Abone:
private void Start()
{
    enemy.OnDied += HandleEnemyDeath;
}

private void HandleEnemyDeath(Vector3 pos, int xp)
{
    SpawnXpGem(pos, xp);
}

event Anahtar Sözcüğü

event ile işaretlenmiş delegate'i sadece tanımlandığı sınıf Invoke edebilir; dış sınıflar sadece += / -= ile abone olur. Bu koruyucudur. 

public class PlayerHealth : MonoBehaviour
{
    public event System.Action<int, int> OnHealthChanged;
    public event System.Action OnDied;

    private int currentHealth = 100;
    private int maxHealth = 100;

    public void TakeDamage(int dmg)
    {
        currentHealth -= dmg;
        OnHealthChanged?.Invoke(currentHealth, maxHealth);
        if (currentHealth <= 0) OnDied?.Invoke();
    }
}

// HealthBar UI dinler:
playerHealth.OnHealthChanged += (current, max) => healthBar.fillAmount = (float)current / max;
playerHealth.OnDied += () => gameOverScreen.SetActive(true);

System.Action = return değeri void olan delegate. Action<int> = bir int parametre alan; Action<int,int> = iki int.

System.Func<T1,T2,TResult> = return değeri olan delegate (LINQ, predicates için).

📌 Unsubscribe Önemi: Eklediğin handler'ı OnDestroy'da kaldır: 
private void OnDestroy()
{
    if (player != null) player.OnHealthChanged -= HandleHealthChanged;
}
Yoksa silinmiş MonoBehaviour'a event giderse MissingReferenceException ya da memory leak.
25.4

🪝 UnityEvent — Inspector'dan Bağlanan Event

System.Action kodda harika ama Inspector'dan görmek istiyorsan: UnityEvent. Button.OnClick gibi. 

using UnityEngine.Events;

public class Door : MonoBehaviour
{
    public UnityEvent OnOpened;
    public UnityEvent<string> OnPlayerSpoke;

    public void Open()
    {
        OnOpened?.Invoke();
    }
}

Inspector'da Door GameObject'inde OnOpened alanı görünür. "+" basıp sahnedeki bir AudioSource'u sürükle, "Play" metodunu seç → kapı açılınca ses çalsın. Kod yazmadan bağlantı.

UnityEvent vs Action karşılaştırması:

  • UnityEvent — Inspector'dan görsel bağlantı, designer dostu, biraz daha yavaş.
  • Action — Sadece kodla bağlanır, hızlı, designer'a görünmez.
  • Hibrit: hem UnityEvent hem Action ekle, ihtiyaca göre.
25.5

📋 ScriptableObject Mimarisi

ScriptableObject (SO), MonoBehaviour'un sahne-bağımsız kuzenidir. Asset dosyası olarak yaşar, sahneye dahil değildir, ama runtime'da çağrılır. Veri için kraldır.

Veri Tabanı Olarak SO

[CreateAssetMenu(menuName = "Game/Item Data")]
public class ItemData : ScriptableObject
{
    public string itemName;
    public Sprite icon;
    public int maxStack;
    public ItemRarity rarity;
    [TextArea] public string description;
    public int goldValue;
}

public enum ItemRarity { Common, Rare, Epic, Legendary }

Project penceresinde sağ tık → Create → Game → Item Data ile asset oluştur. Designer hiç kod görmeden envanter eşyaları tasarlar.

Event Channel Pattern

SO'ları event yayıncısı olarak kullanmak Unity topluluğunun en popüler tasarım kalıplarından biridir. Sahne sınırını aşan haberleşme: 

[CreateAssetMenu(menuName = "Game/Event Channel/Void")]
public class VoidEventChannel : ScriptableObject
{
    public event System.Action OnEventRaised;
    public void Raise() => OnEventRaised?.Invoke();
}

[CreateAssetMenu(menuName = "Game/Event Channel/Int")]
public class IntEventChannel : ScriptableObject
{
    public event System.Action<int> OnEventRaised;
    public void Raise(int value) => OnEventRaised?.Invoke(value);
}

// Kullanım:
public class PlayerHealth : MonoBehaviour
{
    [SerializeField] private IntEventChannel healthChangedChannel;

    public void TakeDamage(int d)
    {
        currentHealth -= d;
        healthChangedChannel.Raise(currentHealth);
    }
}

// HealthBarUI (farklı sahnede bile olabilir):
public class HealthBarUI : MonoBehaviour
{
    [SerializeField] private IntEventChannel healthChangedChannel;

    private void OnEnable() => healthChangedChannel.OnEventRaised += UpdateBar;
    private void OnDisable() => healthChangedChannel.OnEventRaised -= UpdateBar;

    private void UpdateBar(int h) { // fill amount güncelle }
}

Player ve UI birbirini tanımıyor — sadece aynı SO event channel'a referansları var. Sahne yüklendiğinde otomatik bağlanır. Test, refactor, yeni sahneye taşıma çok kolay. Bu mimariyle 2-3 yıllık büyük projelerin kodu temiz kalır.

25.6

⚡ async / await — Modern Asenkron

Coroutine yıllarca Unity'nin standardıydı; ama .NET dünyasının çoğu (web requests, file IO, database) async/await ile çalışır. Unity 2017+'tan beri async desteklenir. 

using System.Threading.Tasks;
using UnityEngine.Networking;

public class LeaderboardClient : MonoBehaviour
{
    private async void Start()
    {
        string json = await FetchLeaderboard();
        Debug.Log(json);
    }

    private async Task<string> FetchLeaderboard()
    {
        using (var req = UnityWebRequest.Get("https://example.com/scores"))
        {
            var op = req.SendWebRequest();
            while (!op.isDone) await Task.Yield();
            return req.downloadHandler.text;
        }
    }
}

Coroutine vs async

🌀 Coroutine

  • Unity native
  • Sadece MonoBehaviour'da
  • StopCoroutine ile durur
  • GameObject deaktif → durur
  • Sahne içi animasyon için ideal
  • Return değeri yok

⚡ async/await

  • .NET standardı
  • Her sınıfta çalışır
  • CancellationToken ile durur
  • GameObject deaktif → DEVAM EDER (!)
  • Web/IO için ideal
  • Task<T> return edilir
⚠️ async'in tehlikesi: GameObject silinince async fonksiyon devam eder. this null olsa bile sonraki satırda hata. Çözüm: CancellationToken kullan, OnDestroy'da iptal et. Veya UniTask (Unity için optimize edilmiş async kütüphanesi) — GameObject yaşam döngüsüne entegre.
25.7

🧩 Generics — Tip Bağımsız Kod

Generic'ler aynı kodu farklı tipler için yazmanı sağlar. List<T>, Dictionary<K,V> bunlardan yararlanır. 

public class ObjectPool<T> where T : MonoBehaviour
{
    private Queue<T> pool = new Queue<T>();
    private T prefab;
    private Transform parent;

    public ObjectPool(T prefab, int initialSize, Transform parent = null)
    {
        this.prefab = prefab;
        this.parent = parent;
        for (int i = 0; i < initialSize; i++) Return(Create());
    }

    private T Create() => GameObject.Instantiate(prefab, parent);

    public T Get()
    {
        T item = pool.Count > 0 ? pool.Dequeue() : Create();
        item.gameObject.SetActive(true);
        return item;
    }

    public void Return(T item)
    {
        item.gameObject.SetActive(false);
        pool.Enqueue(item);
    }
}

// Kullanım:
var bulletPool = new ObjectPool<Bullet>(bulletPrefab, 20);
var enemyPool = new ObjectPool<Enemy>(enemyPrefab, 50);
Bullet b = bulletPool.Get();

Tek bir ObjectPool kodu, hem Bullet hem Enemy hem PowerUp için çalışır. where T : MonoBehaviour constraint'i T'nin MB olduğunu garanti eder, böylece gameObject özelliğini kullanabilirsin.

Unity 2021+ ile UnityEngine.Pool.ObjectPool<T> built-in geliyor — yukarıdakine benzer ama optimize edilmiş.

25.8

🔍 LINQ — Sorgu Dili

LINQ (Language-Integrated Query), koleksiyonlar üzerinde SQL-benzeri sorgular yazmana izin verir. Kod kısalır, niyet okunur. 

using System.Linq;

public class EnemyManager : MonoBehaviour
{
    public List<Enemy> allEnemies;

    // Hayatta olan düşmanlar:
    public List<Enemy> GetAlive() =>
        allEnemies.Where(e => e.health > 0).ToList();

    // En yakın düşman:
    public Enemy GetClosest(Vector3 from) =>
        allEnemies.OrderBy(e => Vector3.Distance(from, e.transform.position))
                  .FirstOrDefault();

    // En zayıf 3 düşman:
    public List<Enemy> GetWeakest(int n) =>
        allEnemies.OrderBy(e => e.health).Take(n).ToList();

    // Toplam XP:
    public int TotalXpReward() =>
        allEnemies.Where(e => e.isDefeated).Sum(e => e.xpReward);

    // Tür bazında grupla:
    public var ByType() =>
        allEnemies.GroupBy(e => e.enemyType);

    // Hepsi öldü mü?
    public bool AllDead() => allEnemies.All(e => e.health <= 0);

    // En az 1 boss var mı?
    public bool AnyBoss() => allEnemies.Any(e => e.isBoss);
}
⚠️ LINQ ve Performans: LINQ her sorguda küçük garbage üretir (enumerator, lambda capture). Update içinde 1000 kere çağırma. Soğuk yol (cold path) ve start-time hesaplama için harika; her frame sıkı döngülerde for kullan.
25.9

🔌 Interfaces — Davranış Sözleşmeleri

Bir interface, sınıfların uygulamak zorunda olduğu metodları listeler. "Sözleşme" gibi düşün. 

public interface IDamageable
{
    void TakeDamage(int amount);
    int CurrentHealth { get; }
}

public interface IInteractable
{
    void Interact(Player player);
}

public class Enemy : MonoBehaviour, IDamageable
{
    public int CurrentHealth { get; private set; } = 100;

    public void TakeDamage(int amount)
    {
        CurrentHealth -= amount;
        if (CurrentHealth <= 0) Destroy(gameObject);
    }
}

public class DestructibleCrate : MonoBehaviour, IDamageable
{
    public int CurrentHealth { get; private set; } = 50;
    public void TakeDamage(int a) { // fıçı kır }
}

// Mermi her şeyle çalışır:
private void OnCollisionEnter(Collision col)
{
    if (col.gameObject.TryGetComponent<IDamageable>(out var dmg))
    {
        dmg.TakeDamage(20);
    }
}

Mermi, Enemy mi DestructibleCrate mi ya da gelecekte yazacağın bir tank mı umurunda değil — IDamageable uyguluyorsa hasar verir. Açık-kapalı prensibi (Open-Closed Principle): kod yeni davranışa açık, eski koda dokunmadan eklenebilir.

25.10

🧪 Extension Methods — Tipleri Genişlet

Mevcut bir tipe yeni metod ekle, kodunda değişiklik yapmadan: 

public static class Vector3Extensions
{
    public static Vector3 WithY(this Vector3 v, float y)
        => new Vector3(v.x, y, v.z);

    public static Vector3 Flatten(this Vector3 v)
        => new Vector3(v.x, 0, v.z);
}

// Kullanım:
transform.position = transform.position.WithY(5);  // y'yi 5 yap, x z aynı
Vector3 flatDir = (target - player).Flatten().normalized;  // y'siz yön

this Vector3 v imzası "bu metodu Vector3 üzerinde çağırılabilir hâle getir" anlamına gelir. Kendi yardımcı sınıfını yazmak yerine kod doğal okunur.

🎓 Bölüm Kontrol Noktası: Coroutine zamana yayılmış fonksiyon, yield return ile durum kontrol. Delegate/event loose coupling. UnityEvent Inspector dostu. ScriptableObject sahne dışı veri + Event Channel pattern. async/await modern asenkron, dikkat: OnDestroy iptal. Generics tip bağımsız (ObjectPool). LINQ sorgular için, hot path'te dikkat. Interface davranış sözleşmesi. Extension method tip genişletme.

🎯 Pratik Projeler

Proje 1: SO Event Channel ile Health Sistemi

Hedef: Player ve HealthBar UI'sı doğrudan referans olmadan birbirini güncellesin.

  1. IntEventChannel : ScriptableObject oluştur (yukarıdaki kodu kopyala).
  2. "PlayerHealthChanged" adında bir asset oluştur.
  3. PlayerHealth script'inde [SerializeField] healthChannel; TakeDamage'de healthChannel.Raise(current).
  4. HealthBarUI script'inde aynı channel'a abone ol, OnEnable/OnDisable'da add/remove.
  5. Her ikisinin de Inspector'da aynı SO asset'i atayın.
  6. Test: TakeDamage çağrıldığında HealthBar otomatik güncelleniyor mu? Birbirinin referansı yok!

Proje 2: ObjectPool ile Mermi Sistemi

Hedef: Düşmanların ateşlediği mermiler Instantiate/Destroy yerine pool'dan gelsin.

  1. Bullet script'i: rigidbody ile ileri hareket, 3 sn sonra Return.
  2. BulletPool : MonoBehaviour, içinde UnityEngine.Pool.ObjectPool<Bullet> veya kendin yazdığın generic pool.
  3. Enemy.Shoot() = pool.Get() → konum ata → mermiyi ateşle.
  4. Bullet 3 sn sonra: pool.Return(this).
  5. Profiler ile GC.Alloc karşılaştır: pool vs Instantiate/Destroy.

Proje 3: IDamageable Interface'i ile Universal Saldırı

Hedef: Kılıç, mermi, patlayıcı — hepsi aynı interface üzerinden hasar versin; düşman, fıçı, ahşap kapı — hepsi aynı interface'e uysun.

  1. IDamageable interface'i: TakeDamage(int), CurrentHealth.
  2. Enemy, DestructibleCrate, WoodDoor — hepsi IDamageable uygulasın.
  3. Bullet.OnCollisionEnter: TryGetComponent<IDamageable> ile hasar.
  4. Sword.OnTriggerEnter: aynı şekilde.
  5. Explosion: Physics.OverlapSphere → her objeyi IDamageable mı kontrol et → varsa hasar.
  6. Yeni bir hasar verici (LightningBolt) eklerken sadece collision kodu yaz; düşman tarafını değiştirme.
