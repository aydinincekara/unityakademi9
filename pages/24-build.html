Bölüm 24

Build & Yayınlama — Oyununu Dünyaya Çıkar

Build Profiles, Player Settings, Windows / macOS / WebGL / Android / iOS hedefleri, IL2CPP vs Mono, Code Stripping, Asset Bundles, mağaza yayını.

⏱ Okuma süresi: ~25 dk 📦 5 platform 🎯 Zorluk: Orta
24.1

📦 Build Nedir?

Unity Editor'da oynadığın oyun, sahnelerin + script'lerin + asset'lerin canlı bir derlemesidir. Editor olmadan bu sahneyi başkasına oynatmak için Unity, hedef platform için tek bir paket oluşturmalı: kod derlenir (.NET → IL2CPP veya Mono), shader'lar derlenir, sahne ve asset'ler serileştirilir, hepsi platform-spesifik bir dosya/klasör'e dönüşür. Bu işleme Build denir.

Platforma göre çıktı farklıdır:

  • Windows → GameName.exe + GameName_Data/ klasörü.
  • macOS → GameName.app bundle (klasör görünümlü tek dosya).
  • Linux → GameName.x86_64 + Data klasörü.
  • WebGL → HTML + JS + WASM dosyaları (tarayıcı için).
  • Android → GameName.apk veya .aab (Google Play için).
  • iOS → Xcode projesi (sonra Xcode ile App Store'a build).

🎮 Gerçek Dünyadan: Among Us Multi-Platform

Among Us, tek bir Unity projesinden Windows, macOS, Android, iOS, Switch, PlayStation, Xbox — neredeyse her platforma build edildi. Unity'nin gücü buradadır: aynı C# kodu, aynı sahne, farklı build hedefleri. Cuphead Switch'e gelirken Unity'nin çoklu platform build pipeline'ı sayesinde aynı yıl içinde 5 platforma çıktı. Genshin Impact PC, mobil ve PlayStation 4/5'i aynı Unity build sisteminden destekliyor.

24.2

📋 Build Profiles Unity 6 YENİ

Unity 6 ile gelen en pratik yeniliklerden biri: Build Profiles. Eskiden Build Settings tek bir aktif platform tutuyordu; her değiştirdiğinde tüm asset'ler yeniden import oluyordu (yarım saat süren bir işlem). Build Profiles ile her platform için ayrı profil kayıtlı tutulur.

  1. File → Build Profiles menüsünü aç.
  2. Sol panelde mevcut platformlar (Windows, Android, WebGL, vs.) görünür.
  3. "+" ile yeni custom profil oluştur: örn. Android — Free Edition, Android — Paid Edition, Steam Beta.
  4. Her profil için sahne listesi, Player Settings, Scripting Define Symbols ayrı yönetilir.
  5. "Switch Profile" tek tıkla aktif platform değişir, asset cache'i tutulduğu için yeniden import gerekmez.
💡 Build Profile Use-Case'leri:
  • Demo vs Tam Sürüm — Demo build sadece 3 sahne içersin.
  • Free vs Premium — Free'de reklam, Premium'da temiz.
  • Test vs Production — Debug log açık vs kapalı.
  • QA vs Yayın — Cheat menüsü dev'de açık, release'de kapalı.
24.3

🚀 İlk Build — Windows Standalone

  1. File → Build Profiles → Windows seç.
  2. Scene List: Build'e dahil edilecek sahneleri sırayla ekle. İlk sahne (index 0) açılışta yüklenir; genelde MainMenu.
  3. Target Platform: Windows. Architecture: x86_64 (Intel/AMD 64-bit).
  4. Compression: Default (uyumlu) veya LZ4HC (daha sıkı).
  5. Development Build: ☑ test sırasında (Profiler, Debug.Log için). Yayın için ☐.
  6. Player Settings'a tıkla, aşağıdaki bölümleri ayarla.
  7. Build'a bas → hedef klasör seç → Unity 30 saniye - 5 dakika içinde paketler.
  8. Çıkış klasöründe GameName.exe'ye çift tıkla; oyun açılır.
24.4

⚙️ Player Settings — Detaylı Yapılandırma

Build Profiles içinde Player Settings butonu Project Settings → Player'a yönlendirir. Burada oyunun kimliği, görünümü, davranışı belirlenir.

Sekmesi: General (Tüm Platformlar)

Player → Genel
Company Name"BenimStudyo"
Product Name"Karanlık Diyarlar"
Version1.0.0
Default Iconicon.png (1024x1024)
Default Cursorcursor.png
  • Company / Product Name → Application.companyName, Application.productName ile çağrılabilir. PlayerPrefs ve save yolları bu isimleri içerir.
  • Version → Steam, App Store gibi mağazalar bu numarayı bekler. Semantic versioning (Major.Minor.Patch) önerilir.
  • Default Icon → 1024x1024 PNG, Unity otomatik küçültür. Platform-spesifik ikon override edilebilir.

Sekmesi: Resolution & Presentation

  • Fullscreen Mode — Fullscreen Window (önerilen, alt+tab dostu), Exclusive Fullscreen (eski), Windowed.
  • Default Screen Width / Height — Pencere başlangıç boyutu.
  • Resizable Window — Kullanıcı pencereyi yeniden boyutlandırabilir mi?
  • Run In Background — Oyun arka planda devam etsin mi (genelde ☑).
  • Display Resolution Dialog — Açılışta ayar diyalogu (eski, modern oyunlarda yok).

Sekmesi: Splash Image

Açılışta görünen logo. Unity ücretsiz lisansta "Made with Unity" logosunu zorunlu tutar; Plus/Pro lisansta kaldırılabilir. Kendi logon eklenebilir (Splash Image bölümünde sırayla).

24.5

⚡ IL2CPP vs Mono — Scripting Backend

Unity, C# kodunu iki şekilde çalıştırabilir: Mono (Just-In-Time derleme) veya IL2CPP (Ahead-Of-Time C++ derleme).

🐢 Mono (Eski)

  • C# → IL → JIT (runtime'da derle)
  • Build hızı: Çok hızlı (30 sn)
  • Runtime performansı: Orta
  • Boyut: Büyük (.NET runtime dahil)
  • iOS desteklemez (Apple yasaklı)
  • Sadece Editor + Standalone

🚀 IL2CPP (Modern)

  • C# → IL → C++ → Native (AOT)
  • Build hızı: Yavaş (5-15 dk)
  • Runtime performansı: Yüksek (%20-40 hızlı)
  • Boyut: Daha küçük
  • Tüm platformlar desteklenir
  • Production için standart

Pratik kural: Geliştirme sırasında Mono (hızlı iterasyon), yayın için IL2CPP (performans + güvenlik). Apple iOS zaten IL2CPP'yi mecbur kılar. WebGL otomatik IL2CPP kullanır.

Code Stripping (Kod Kırpma) ayarı IL2CPP ile çalışır: kullanılmayan .NET kütüphanelerinin dahil edilmemesini sağlar. Low, Medium, High, Minimal seviyeleri. Yüksek seviye küçük boyut ama bazı reflection-kullanan kodlar bozulabilir.

24.6

📱 Android Build

  1. Build Profiles → Android. (İlk seferinde Android modülünü Unity Hub'tan kurman gerekir.)
  2. Player Settings → Other Settings:
    • Package Name: com.benimstudyo.karanlikdiyarlar (Java paket adı tarzı, mağazada benzersiz olmalı).
    • Minimum API Level: Android 7.0 (API 24) — pazarın %95'i.
    • Target API Level: Google Play en güncel istiyor (API 34+).
    • Scripting Backend: IL2CPP.
    • Target Architectures: ARMv7 + ARM64 ☑ (modern Google Play 64-bit zorunlu).
  3. Publishing Settings:
    • Keystore: yeni oluştur (.keystore dosyası), şifre belirle.
    • Key Alias: oyun ismine özel anahtar.
  4. Build Output: APK (test için) veya AAB (Google Play için zorunlu).
  5. Build → .apk dosyası bilgisayar üzerinden telefona aktar veya adb install.
🔑 Keystore'u Asla Kaybetme: Bir oyunu Google Play'de yayınladıktan sonra her güncelleme aynı keystore ile imzalanmalı. Keystore'u kaybetmek = uygulamanı sonsuza kadar güncelleyememek. Bulut yedeği al, şifreyi password manager'a kaydet.
24.7

🍎 iOS Build

iOS'a build için macOS bilgisayar ve Xcode gereklidir. Unity Windows'ta Xcode projesi üretemez (Xcode-only).

  1. Mac üzerinde Unity Hub'tan iOS Build Support modülü.
  2. Build Profiles → iOS. Player Settings → Other Settings:
    • Bundle Identifier: com.benimstudyo.karanlikdiyarlar.
    • Target minimum iOS Version: iOS 13 (modern pazar).
    • Target Device: iPhone + iPad veya sadece iPhone.
  3. Build → Xcode projesi (.xcodeproj) klasörü üretilir.
  4. Mac'te Xcode'da açılır.
  5. Apple Developer hesabı (yıllık $99) ile Provisioning Profile, App Store Connect'te app oluştur.
  6. Xcode'tan "Archive" → "Upload to App Store" → App Store Connect'te TestFlight ile beta test → Submit for Review.
24.8

🌐 WebGL Build

Tarayıcıda doğrudan oynanabilen oyun. itch.io, kongregate, kendi websiten için ideal.

  1. Unity Hub'tan WebGL Build Support kur.
  2. Build Profiles → WebGL.
  3. Player Settings → Publishing Settings:
    • Compression Format: Brotli (en küçük) veya Gzip (yaygın destek).
    • Decompression Fallback: ☑ (eski tarayıcılar için).
  4. Player Settings → Resolution: Default Canvas Width 960, Height 600 (responsive).
  5. Build → index.html + Build/ + TemplateData/ üretilir.
  6. Bunları bir web sunucusuna yükle (itch.io ZIP kabul eder), tarayıcıda aç.
📊 WebGL Sınırları:
  • Threading sınırlı (Web Workers henüz tam destekli değil).
  • Yerel dosya sistemi yok — PlayerPrefs tarayıcı localStorage'a yazar.
  • İlk yükleme yavaş (oyun + asset'ler indirilmeli).
  • Mobil tarayıcılarda performans düşük.
  • İdeal: kısa, hafif oyunlar (Browser jam, prototip, web portfolio).
24.9

📚 Asset Bundles & Addressables

Build her şeyi içine alır → 5 GB exe. Çoğu içerik oyun başlangıcında değil, ilerleyen aşamalarda gerekiyor. Çözüm: bazı asset'leri ayrı paketlere böl, runtime'da indir.

  • Asset Bundles (klasik) — Eski sistem. Belirli asset'leri .bundle dosyasına paketle, runtime'da yükle.
  • Addressables (modern, önerilen) — Asset'lere "address" (string adres) verirsin, runtime'da o adresle yüklersin. Otomatik bundle yönetimi, CDN entegrasyonu.
using UnityEngine;
using UnityEngine.AddressableAssets;
using UnityEngine.ResourceManagement.AsyncOperations;

public class LevelLoader : MonoBehaviour
{
    public AssetReferenceGameObject bossPrefab;  // editor'de sürükle

    private async void SpawnBoss()
    {
        GameObject obj = await bossPrefab.InstantiateAsync().Task;
        obj.transform.position = new Vector3(0, 5, 10);
    }
}

Bu sayede boss prefab'ı binlerce poligonlu olsa bile build'in temel boyutunu büyütmez, sadece boss bölümüne gelindiğinde indirilir.

24.10

🐛 Build Sonrası Debug

Editor'da çalışan oyun, build'de bozulabilir. En yaygın nedenler:

  • Editor-only kod: #if UNITY_EDITOR ile sarılmamış editor referansları runtime'da hata verir.
  • Asset path: Resources.Load path'i case-sensitive, mobilde tam eşleşmeli.
  • Code stripping: Reflection kullanılan tipler kırpılabilir → [Preserve] attribute ile koru veya link.xml ekle.
  • Platform-spesifik bug: Android'de file path yazma izni, iOS'ta thread sınırı, WebGL'de System.IO yetersizlikleri.

Log Görüntüleme

  • Standalone: %appdata%\..\LocalLow\Company\Product\output_log.txt.
  • Android: adb logcat -s Unity ile real-time log.
  • iOS: Xcode → Devices & Simulators → Open Console.
  • WebGL: Tarayıcı Developer Tools → Console.

Build içinde Profiler bağlamak için: Development Build ve Autoconnect Profiler ☑. Unity Editor Profiler penceresi build'e Wifi/USB üzerinden bağlanıp gerçek zamanlı performans verisi gösterir.

24.11

🏪 Mağaza Yayını — Steam, App Store, Google Play

Steam (PC)

  • $100 Steam Direct ücreti (oyun başına, geri ödemeli).
  • Steamworks SDK ile achievements, leaderboards, cloud save entegrasyonu.
  • Build'ini Steamworks'e upload et, Store Page hazırla, Review süreci 1-7 gün.
  • Trailer + screenshot + Türkçe/İngilizce metinler hazırla.

Google Play (Android)

  • $25 tek seferlik developer hesap ücreti.
  • App Bundle (.aab) yükle, Google Play Console'da listing oluştur.
  • Privacy policy URL'i şart (Notion'da basit bir sayfa yeter).
  • İçerik derecelendirmesi (IARC) anketi.
  • İlk sürüm review 1-7 gün.

App Store (iOS)

  • $99/yıl Apple Developer Program.
  • Xcode → Archive → App Store Connect upload.
  • TestFlight ile beta test (max 10,000 tester).
  • Submit for Review — 1-3 gün, daha katı (UI guideline'lar).

itch.io (Bağımsız)

  • Ücretsiz, açılan saatte yayınlanabilir.
  • WebGL, Windows, Mac, Linux ZIP olarak yükle.
  • Game jam, prototip, indie portföy için ideal.
  • "İstediğin kadar öde" seçeneği, demo + tam sürüm ayrı.
24.12

📉 Build Boyutunu Küçültme

  • Texture Compression: PNG → ASTC (Android), BC7 (PC). 4 kat küçülme.
  • Audio Compression: WAV → Vorbis, müzik için Streaming.
  • Code Stripping: Player Settings → Managed Stripping Level = High.
  • Resources/ klasörünü temizle — buradaki her şey build'e girer.
  • Kullanılmayan paketleri Package Manager'dan kaldır.
  • Mesh Compression: Asset Import → Mesh Compression = Medium.
  • Build Report (Window → Analysis → Build Report) ile en büyük asset'leri bul.
  • Editor-only asset'leri StreamingAssets/ dışında tut.
📊 Hedef Boyutlar:
  • Web/Indie oyun: 50-150 MB
  • Mobile casual: 50-200 MB (Google Play 150 MB üstü ek dosya gerektirir)
  • Mobile RPG: 500 MB - 2 GB (Addressables ile başlangıç küçük)
  • PC indie: 500 MB - 4 GB
  • AAA: 30 GB+
📦 Bölüm Kontrol Noktası: Build Profiles ile platform yönetimi. IL2CPP yayın için, Mono geliştirme için. Android keystore'u koru. iOS Xcode mecbur. WebGL kısa oyunlar. Asset Bundles/Addressables runtime asset yönetimi. Mağazalar farklı kurallar; Apple en katı, itch.io en serbest. Boyut için stripping + texture/audio compression + Resources/ temizliği.

🎯 Pratik Projeler

Proje 1: İlk Windows Build

Hedef: Mevcut bir prototipini Windows'a build et, arkadaşına gönder.

  1. File → Build Profiles → Windows seç.
  2. Sahnelerin sıralamasını kontrol et (MainMenu önce, Level01 sonra).
  3. Player Settings → Product Name, Version, Default Icon ayarla.
  4. Build → bir klasör seç → exe + Data klasörü oluşsun.
  5. Klasörü ZIP'le, arkadaşına gönder, exe'yi açtırt.
  6. Bonus: Inno Setup veya Squirrel ile yükleyici (.msi) hazırla.

Proje 2: WebGL Build + itch.io Yayını

Hedef: Aynı projeyi WebGL'e build et, itch.io'ya ücretsiz yükle.

  1. Unity Hub → WebGL Build Support modülü kur.
  2. Build Profiles → WebGL → Player Settings → Resolution: 960x600.
  3. Publishing → Compression Format: Brotli.
  4. Build → bir klasör seç → index.html oluşsun.
  5. Tüm klasörü ZIP'le (index.html en üstte).
  6. itch.io'ya kayıt ol → "Upload new project" → ZIP yükle → "This file will be played in browser" ☑.
  7. Embed boyutu 960x600 ayarla, açıklama + screenshot ekle.
  8. Oyunu paylaşılabilir URL'den arkadaşlara gönder.

Proje 3: Android APK + Telefonda Test

Hedef: Oyunu Android telefonuna kur, dokunmatik kontrolleri test et.

  1. Unity Hub → Android Build Support + JDK + NDK kur.
  2. Telefonda Developer Options → USB Debugging ☑.
  3. Build Profiles → Android → Player Settings → Package Name (com.benim.test123).
  4. Min API Level 24, Target API Level 34, Scripting Backend IL2CPP, ARM64 ☑.
  5. Build And Run (telefon USB'de bağlıyken) → otomatik APK telefona kurulur.
  6. Dokunmatik UI çalışıyor mu? Performans 30 FPS mi 60 FPS mi?
  7. Profiler → USB üzerinden bağlan → CPU/GPU kullanımını izle.
