MonoBehaviour Lifecycle — Script'inin yaşam döngüsü

Unity senin script'ini çalıştırırken belirli sıralı olaylarda özel metotlarını otomatik çağırır: Awake, Start, Update, FixedUpdate, OnDestroy... Hangisi ne zaman çağrılır? Hangisini ne için kullanmalısın? Bu bölüm, Unity programlamasının görünmeyen "saat mekanizmasını" gözler önüne seriyor.

5.1

MonoBehaviour — Unity'nin script temel sınıfı

Bir önceki bölümde her script'inin MonoBehaviour'dan türediğini gördün. Peki MonoBehaviour tam olarak ne yapar?

MonoBehaviour: Unity'nin sağladığı bir temel sınıf. Onu miras alan her script, otomatik olarak Component olur ve Unity'nin lifecycle çağrılarına tabi olur. Yani senin yazdığın metotlar (Start, Update, vb.) Unity tarafından doğru zamanda çağrılır.

MonoBehaviour'dan türemenin getirileri

GameObject'e Component olarak takılabilir.
Inspector'da public/SerializeField alanlar görünür.
Awake, Start, Update gibi lifecycle metotları otomatik çağrılır.
Coroutine başlatabilir (StartCoroutine).
transform, gameObject, GetComponent gibi kısa yollar hazır gelir.

MonoBehaviour'dan türememeyi ne zaman tercih edersin?

Sırf veri tutan, davranışı olmayan class'lar için MonoBehaviour gereksiz yüktür. Örneğin WeaponInfo gibi salt veri taşıyan class'lar düz class olarak yazılır. ScriptableObject ise (Bölüm 25) daha gelişmiş bir veri taşıyıcısıdır.

5.2

Init faz — Awake, OnEnable, Start

Sahne yüklenip Play'e bastığında, Unity her aktif GameObject'in her aktif Component'ini sırayla uyandırır. Bu uyandırma 3 aşamadır:

Awake — En erken çağrı

Awake(), GameObject sahnede aktifleştirildiğinde en erken çağrılır. Component referansları hazırdır ama diğer Component'lerin Start'ı henüz çalışmamış olabilir.

PlayerSetup.cs
private Rigidbody rb;
private Animator anim;

void Awake()
{
    // Kendi Component'lerini bul ve cache'le
    rb = GetComponent<Rigidbody>();
    anim = GetComponent<Animator>();
}
Awake ne için ideal? Kendi Component referanslarını cache'lemek. GetComponent çağrılarını burada yap, sonra değişkende sakla. Update'te GetComponent çağırmak performans katilidir.

OnEnable — Aktifleşince çağrılır

OnEnable(), GameObject veya Component aktif (enabled = true) olduğu her seferinde çağrılır. Sahne ilk açılırken Awake'den hemen sonra; sonradan SetActive(true) yapılınca yine çağrılır.

EnemySpawnEvents.cs
void OnEnable()
{
    GameEvents.OnEnemyKilled += HandleEnemyKilled;
}

void OnDisable()
{
    GameEvents.OnEnemyKilled -= HandleEnemyKilled;
}

OnEnable/OnDisable, event abone olmak için en uygun yer. Aktifken dinler, pasifken çıkar — bellek sızıntısı olmaz.

Start — İlk frame'den hemen önce

Start(), tüm Awake çağrıları bittikten sonra, ilk Update'ten hemen önce bir kez çağrılır. Eğer Component pasif durumdaysa, Start aktifleşene kadar beklemez — sadece script aktifse çağrılır.

PlayerInit.cs
void Start()
{
    // Diğer Component'lere bağımlı başlangıç işlemleri burada
    GameObject manager = GameObject.FindWithTag("GameManager");
    manager.GetComponent<GameManager>().RegisterPlayer(this);
}

Awake vs Start — kritik fark

Awake

Her GameObject yüklendiğinde anında çağrılır. Component pasif olsa bile (script aktif olduğu sürece) çağrılır.

  • Kendi referanslarını al
  • Singleton kurulumu
  • Erken init
Start

Tüm Awake'ler bittikten sonra, ilk frame öncesi çağrılır. Sadece aktif Component'lerde çalışır.

  • Diğer scriptlere ulaş
  • Başlangıç değerleri ata
  • Coroutine başlat
5.3

Update — Her frame, her saniye, her şey

Oyun başladıktan sonra Unity her frame'de ana döngüye girer. Bu döngüde 3 farklı Update vardır — birbirinden farklı amaçlar için.

Update — Genel amaçlı, her frame

En sık kullandığın yer. Her frame'de bir kez çağrılır. Frame hızı sabit değildir (30 FPS'te saniyede 30, 144 FPS'te 144 kez).

PlayerMover.cs
public float hiz = 5f;

void Update()
{
    float yatay = Input.GetAxis("Horizontal");
    float dikey = Input.GetAxis("Vertical");

    Vector3 hareket = new Vector3(yatay, 0, dikey) * hiz * Time.deltaTime;
    transform.Translate(hareket);
}
Time.deltaTime hayati önemde: Frame hızı bilgisayardan bilgisayara değişir. Eğer hareketi sadece hiz ile çarparsan, 60 FPS'te ve 144 FPS'te farklı hızlarda hareket eder. Time.deltaTime ile çarparsan, hareket saniyede X birim olur — frame hızından bağımsız. Bunu unutma, en sık yapılan hatadır.

FixedUpdate — Fizik için, sabit aralıkla

FixedUpdate() sabit zaman aralıklarıyla çağrılır (varsayılan: saniyede 50 kez, 0.02s). Fizik motoru bu döngüde çalışır.

PlayerPhysicsMover.cs
private Rigidbody rb;
public float kuvvet = 10f;

void Awake() { rb = GetComponent<Rigidbody>(); }

void FixedUpdate()
{
    float yatay = Input.GetAxis("Horizontal");
    rb.AddForce(new Vector3(yatay, 0, 0) * kuvvet);
}
Pratik kural: Rigidbody hareketi varsa → FixedUpdate. Transform'u doğrudan değiştiriyorsan veya Input okuyorsan → Update. Fizik sabit FPS'te çalışmak ister; aksi takdirde sıçramalı görüntü olur.

LateUpdate — Update'lerden sonra

LateUpdate(), tüm Update'ler bittikten sonra her frame'de bir kez çağrılır. En klasik kullanım: kamera takibi.

CameraFollow.cs
public Transform hedef;
public Vector3 offset = new Vector3(0, 5, -10);

void LateUpdate()
{
    // Oyuncu Update'te hareket etti → şimdi kamerayı güncelle
    transform.position = hedef.position + offset;
}

Eğer kamerayı Update'te güncellersen, oyuncu o frame'de hareket etmeden önce kamera güncellenmiş olur — bir frame gerilik hissedersin. LateUpdate'te güncellersen kamera oyuncuyla aynı frame'de senkronize hareket eder.

3 Update'in karşılaştırması

Update

Her frame, FPS'e göre değişken sıklıkta. Input, animasyon, genel logic.

FixedUpdate

Sabit aralık (varsayılan 0.02s = 50/s). Fizik ve Rigidbody.

LateUpdate

Update'lerden sonra her frame. Kamera, takip, geçit logic.

Yarı yol — Şimdi ne biliyorsun?

Awake → OnEnable → Start sırasıyla her script uyanır. Sonra her frame'de FixedUpdate (fizik) → Update (logic) → LateUpdate (kamera) çağrılır. Time.deltaTime ile FPS'ten bağımsız hareket sağlanır. Sıra şimdi: yok olma fazı ve özel callback'ler.

5.4

Bitiş fazı — OnDisable, OnDestroy

Bir GameObject veya Component devre dışı bırakıldığında ya da yok edildiğinde Unity yine senin script'ine haber verir.

OnDisable — Pasifleşirken

GameObject veya Component enabled = false yapıldığında veya SetActive(false) çağrıldığında çağrılır. Sahne kapanırken de çağrılır.

OnDestroy — Yok edilirken

Component veya GameObject Destroy ile yok edildiğinde, sahne değişirken veya oyun kapanırken çağrılır.

CleanupExample.cs
void OnEnable()
{
    GameEvents.OnLevelComplete += HandleLevelComplete;
}

void OnDisable()
{
    // Event'ten çık — bellek sızıntısı önler
    GameEvents.OnLevelComplete -= HandleLevelComplete;
}

void OnDestroy()
{
    // Dosya kapat, kaynak temizle
    PlayerPrefs.Save();
}
OnDestroy'da dikkatli ol: Oyun kapanırken Unity bazı sistemleri zaten kapatmış olabilir. FindObjectOfType gibi pahalı işlemleri burada yapma; null kontrolü ekle.
5.5

Tam lifecycle akışı — Bir frame'de neler oluyor?

Tüm parçaları bir arada görelim. Aşağıdaki sıralama Unity'nin resmi Script Lifecycle dokümanından özetlendi:

1
Editor'de Play'e basıldı — Tüm aktif GameObject'lerin script'leri yüklenir.
2
Awake() — Her aktif Component'te bir kez. Sıra: Execution Order ayarına göre.
3
OnEnable() — Aktif Component'lerde çağrılır.
4
Start() — Tüm Awake'ler bittikten sonra, ilk frame öncesi. Sadece aktif olanlarda.
5
=== Ana döngü başlar === Her frame için aşağıdakiler tekrar edilir:
6
FixedUpdate() — Sabit aralıkta. Bir frame'de hiç çağrılmayabilir veya birkaç kez çağrılabilir.
7
OnTrigger / OnCollision — Bu frame'de fizik çarpışmaları varsa.
8
Input olayları — Klavye/fare girişleri işlenir.
9
Update() — Tüm aktif scriptlerde, sıralı. En sık iş bu metotta yapılır.
10
Coroutine ilerleme — yield koşullarına bakılır.
11
LateUpdate() — Tüm Update'lerden sonra. Kamera vb. son güncellemeler.
12
Rendering — Sahne ekrana çizilir.
13
OnGUI() — Eski IMGUI çizimleri (modern projelerde kullanılmaz).
14
=== Bir sonraki frame === Adım 6'dan tekrarlanır.
15
Stop/Çıkış — OnDisable() → OnDestroy() sırasıyla çağrılır.
Şunu hep aklında tut: Awake/Start birer kerelik; Update/FixedUpdate/LateUpdate her frame'de döner; OnDisable/OnDestroy son anlarda çağrılır. İhtiyaca göre doğru metoda yaz.
5.6

Coroutine — "Bekle ve devam et" desenler için

Bazen "şu an bunu yap, 2 saniye bekle, sonra şunu yap" gibi zamanlı akışlar gerekir. Update'in içinde sayaç tutmak yerine Unity sana Coroutine sunar.

Basit bir Coroutine

DelayedAction.cs
using System.Collections;
using UnityEngine;

public class DelayedAction : MonoBehaviour
{
    void Start()
    {
        StartCoroutine(UcSaniyeSonra());
    }

    IEnumerator UcSaniyeSonra()
    {
        Debug.Log("Beklemeye başladım");
        yield return new WaitForSeconds(3f);
        Debug.Log("3 saniye geçti!");
    }
}

Coroutine'lar IEnumerator dönen metotlardır. İçinde yield return ile beklenebilir. Update'i bloklamadan zamanlı işler yapmak için idealdir.

Çeşitli yield türleri

yield return new WaitForSeconds(2f); → 2 saniye bekle.
yield return new WaitForSecondsRealtime(2f); → Time.timeScale'den etkilenmez.
yield return null; → Bir frame bekle.
yield return new WaitForFixedUpdate(); → Sonraki FixedUpdate'i bekle.
yield return new WaitUntil(() => can > 0); → Koşul gerçekleşene kadar bekle.
Daha derin Coroutine kullanımı Bölüm 25 (C# İleri)'de — async/await ile farkı, performans, iptal etme.
5.7

Script Execution Order — Sıralamayı sen belirle

Birden çok script aynı sahnede çalışıyorsa, Awake/Start/Update sırası varsayılan olarak belirsizdir. Eğer "GameManager mutlaka PlayerController'dan önce uyanmalı" diyorsan:

1
Edit > Project Settings > Script Execution Order'ı aç.
2
+ butonuyla script'ini ekle. Sıralı liste oluşur.
3
Daha düşük sayı = daha erken çalışır. -100, -50, 0 (Default), 100, 200...
4
Apply, sonra kapat. Sıralama artık deterministik.
Tavsiye: Execution Order'a fazla bel bağlama. Eğer sıralama gerekiyorsa, Awake'te tüm kurulumu bitir; Start'ta diğerleriyle ilet. Singleton pattern de iyi bir çözüm: GameManager.Instance her zaman erişilebilir, kim önce uyandı önemli olmaz.

Lifecycle gerçek oyunlarda nasıl önemli?

Among Us
Lobi - oyun geçişi

Sahne yüklendiğinde her oyuncunun script'i Awake'te kendini hazırlar, Start'ta network'e bağlanır. Sıralama hatalı olursa "ghost player" hatası çıkar.

Cuphead
Boss fazları

Her faz geçişi OnEnable/OnDisable ile yapılır. Yeni faz GameObject'i aktif olur, eski faz event'lerden çıkar. Bellek sızıntısı olmaz.

Pokémon Go
Konum güncellemeleri

GPS verisi Coroutine içinde periyodik alınır. Update'i tıkamadan, her 2 saniyede bir konum sorgulanır.

Senin oyunun için
Her tür mekanik

Karakter hareketi → Update. Fizik tepkimesi → FixedUpdate. Kamera takibi → LateUpdate. Yetenek cooldown → Coroutine. Her birinin yeri belli.

Pratik: 3 mini proje

P1

Lifecycle gözlem scripti

Hedef: Tüm lifecycle metotlarını yazıp her birinde Debug.Log ile sırayı gör.

1
LifecycleObserver.cs oluştur. Awake, OnEnable, Start, Update, FixedUpdate, LateUpdate, OnDisable, OnDestroy metotlarını ekle.
2
Her birinde Debug.Log("Awake çağrıldı") gibi mesaj yazdır.
3
Empty GameObject'e ekle, Play'e bas, 2 saniye sonra Stop'la. Console'da sırayı incele.
4
Update ve FixedUpdate'in farklı sıklıklarda çağrıldığını gözlemle.
Tamamlama Kontrolü
  • Awake → OnEnable → Start sırası net
  • Update sayıca çok daha fazla
  • OnDisable → OnDestroy Stop'la birlikte çağrılıyor
Lifecycle'ı gözlemleyerek pekiştir.
P2

Kamera takibi (LateUpdate)

Hedef: Bir küp WASD ile hareket etsin, kamera LateUpdate'te küpün arkasından takip etsin.

1
Küp + zemin sahne hazırla. Küp'e PlayerMover.cs ekle, Update'te WASD ile hareket.
2
Main Camera'ya CameraFollow.cs ekle. public Transform hedef; ve public Vector3 offset = new Vector3(0, 5, -10);
3
LateUpdate içinde: transform.position = hedef.position + offset;
4
Inspector'dan hedef alanına küpü sürükle.
5
Test: kamera takip etmeli ve titreme olmamalı.
Tamamlama Kontrolü
  • Küp WASD ile hareket ediyor
  • Kamera arkasından takip ediyor
  • Titreme/gecikme yok (LateUpdate sayesinde)
LateUpdate'in neden önemli olduğunu yaşa.
P3

Coroutine ile geri sayım

Hedef: 5 saniyelik geri sayım, her saniyede Console'a sayı yaz, bitince "Başla!" yaz.

1
Countdown.cs. using System.Collections; ekle.
2
Start'ta StartCoroutine(GeriSay());
3
IEnumerator GeriSay() içinde for döngüsü 5'ten 1'e: her iterasyonda Debug.Log(i); ve yield return new WaitForSeconds(1f);
4
Döngü sonrası Debug.Log("Başla!");
Tamamlama Kontrolü
  • 5, 4, 3, 2, 1 her saniyede tek tek görünüyor
  • Sonra "Başla!" yazıyor
  • Update'i bloklamıyor
Coroutine'in temel kullanımını kavra.
