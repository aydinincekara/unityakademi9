Save Sistemi & Menü Tasarımı

PlayerPrefs, JSON, ScriptableObject save, binary save, encryption — ilerleyişi koruma. Ana menü, duraklama menüsü, ayarlar — UI Toolkit ve UGUI ile.

⏱ Okuma süresi: ~28 dk 💾 4 save yöntemi 🎯 Zorluk: Orta
23.1

💾 Neden Save Sistemine İhtiyaç Var?

Oyun çalışırken tüm değişkenler RAM'de yaşar. Player'ın canı, envanteri, hangi bölümde olduğu, ses ayarları — hepsi bilgisayar kapanınca silinir. Save sistemi bu RAM verisini diske kalıcı bir dosyaya yazma, ardından sonraki oturumda geri okuma sanatıdır.

Unity'de save için 4 yaygın yöntem vardır, her birinin yeri farklıdır:

  • PlayerPrefs — Ayarlar, en yüksek skor, basit anahtar-değer çiftleri.
  • JSON — Karakter durumu, envanter, oyun ilerleyişi — insanın okuyabileceği metin.
  • Binary — Büyük save verileri, hız önemli, manipülasyon zorlaşsın.
  • ScriptableObject — Editor zamanı veri, runtime yapılandırma (cihaz kaydetme dışı).

🎮 Gerçek Dünyadan: Hollow Knight'ta Save

Hollow Knight'ın save dosyası Unity'nin JsonUtility'sinden geçmiş bir yapıdır. Oyuncunun açtığı her geçit, yendiği boss, bulduğu eşya, bench koordinatı — hepsi tek bir JSON. Oyun, başlangıçta bu JSON'u okur, kaldığı yerden devam ettirir. Yedek alanlar dahi JSON'u düzenleyerek kendi kendine başarımlar açabilir. Bu, JSON tabanlı save'in hem gücü hem zayıflığıdır.

23.2

🔑 PlayerPrefs — Basit ve Hızlı

PlayerPrefs, Unity'nin sunduğu anahtar-değer (key-value) deposu'dur. Platform bağımsız olarak işletim sisteminin uygun yerine yazar:

  • Windows → Registry (HKCU\Software\<Company>\<Product>).
  • macOS → ~/Library/Preferences/<bundle_id>.plist.
  • Linux → ~/.config/unity3d/<Company>/<Product>.
  • iOS/Android → uygulamanın preferences alanı.

3 Veri Tipi Destekler

// Yazma
PlayerPrefs.SetInt("HighScore", 1500);
PlayerPrefs.SetFloat("MusicVolume", 0.7f);
PlayerPrefs.SetString("PlayerName", "Ayşe");
PlayerPrefs.Save();  // Diske flush et (yoksa quit'te otomatik)

// Okuma — varsayılan değer ile
int high = PlayerPrefs.GetInt("HighScore", 0);
float vol = PlayerPrefs.GetFloat("MusicVolume", 1f);
string name = PlayerPrefs.GetString("PlayerName", "Player");

// Anahtar var mı?
if (PlayerPrefs.HasKey("HighScore")) { // var }

// Sil
PlayerPrefs.DeleteKey("HighScore");
PlayerPrefs.DeleteAll();  // hepsini sil (dikkat!)
⚠️ PlayerPrefs'in Sınırları: Karmaşık veri için tasarlanmadı. 30+ anahtar yazıyorsan veya bir Dictionary kaydetmek istiyorsan, JSON'a geç. Ayrıca PlayerPrefs verisi kolay manipüle edilebilir — kullanıcı registry'ye girip skoru değiştirir. Ciddi veri için kullanma.
23.3

📄 JSON ile Save — Yapısal Veri

Karakterin tüm durumunu kaydetmek için: önce verileri tutacak bir serializable class tanımla, sonra Unity'nin JsonUtility'siyle string'e çevir, son olarak dosyaya yaz.

Adım 1: Save Veri Modeli

[System.Serializable]
public class PlayerSaveData
{
    public string playerName;
    public int level;
    public float currentHealth;
    public Vector3 position;
    public List<string> inventory;
    public List<int> unlockedAchievements;
    public string sceneName;
    public string saveTime;
}

Önemli: [Serializable] attribute zorunludur. Field'lar public olmalı veya [SerializeField] ile işaretli private. Dictionary direkt desteklenmez — List<Pair> veya iki paralel List kullan.

Adım 2: SaveSystem Class

using System.IO;
using UnityEngine;

public static class SaveSystem
{
    private static string path =>
        Path.Combine(Application.persistentDataPath, "save.json");

    public static void Save(PlayerSaveData data)
    {
        string json = JsonUtility.ToJson(data, prettyPrint: true);
        File.WriteAllText(path, json);
        Debug.Log($"Save yazıldı: {path}");
    }

    public static PlayerSaveData Load()
    {
        if (!File.Exists(path))
        {
            Debug.LogWarning("Save dosyası yok");
            return null;
        }
        string json = File.ReadAllText(path);
        return JsonUtility.FromJson<PlayerSaveData>(json);
    }

    public static bool HasSave() => File.Exists(path);
    public static void Delete() { if (File.Exists(path)) File.Delete(path); }
}

Application.persistentDataPath her platformda uygulamanın kalıcı veri klasörünü verir. Windows'ta %appdata%\..\LocalLow\<Company>\<Product>, Android'de app-specific folder. Save dosyaları için bu yeri kullan, asla Application.dataPath kullanma (oraya yazma izni yoktur).

Adım 3: Kullanım

public class GameManager : MonoBehaviour
{
    public Transform player;

    public void SaveGame()
    {
        PlayerSaveData data = new PlayerSaveData
        {
            playerName = "Ayşe",
            level = 5,
            currentHealth = 85f,
            position = player.position,
            inventory = new List<string> { "sword", "shield" },
            sceneName = SceneManager.GetActiveScene().name,
            saveTime = System.DateTime.Now.ToString("yyyy-MM-dd HH:mm")
        };
        SaveSystem.Save(data);
    }

    public void LoadGame()
    {
        PlayerSaveData data = SaveSystem.Load();
        if (data == null) return;
        player.position = data.position;
        // vd. değerleri sahneye uygula
    }
}
23.4

📦 Binary Save — Hız ve Boyut

JSON insan-okur ama büyür ve yavaşlar. Büyük save'ler için (örn. açık dünya oyunu, 50 MB save) binary serialization 2-5x daha küçük dosya ve 3-10x daha hızlı yükleme verir. 

using System.IO;
using System.Runtime.Serialization.Formatters.Binary;

public static class BinarySaveSystem
{
    private static string path =>
        Path.Combine(Application.persistentDataPath, "save.dat");

    public static void Save(PlayerSaveData data)
    {
        using (var stream = File.Create(path))
        {
            BinaryFormatter bf = new BinaryFormatter();
            bf.Serialize(stream, data);
        }
    }

    public static PlayerSaveData Load()
    {
        if (!File.Exists(path)) return null;
        using (var stream = File.OpenRead(path))
        {
            BinaryFormatter bf = new BinaryFormatter();
            return (PlayerSaveData)bf.Deserialize(stream);
        }
    }
}
📌 .NET'in BinaryFormatter Uyarısı: Microsoft, BinaryFormatter'ı .NET 5+'ta güvenlik nedeniyle obsolete ilan etti. Production'da Protobuf-net, MessagePack, ya da Unity'nin yeni Json.NET (Newtonsoft) tabanlı paketini tercih et. Burada öğrenmek için basit kalsın diye BinaryFormatter gösteriyoruz.
23.5

🔐 Save'i Şifrele — Hile Önleme

Save dosyası açık JSON ise oyuncu Notepad'le açıp "health": 100'ü "health": 99999 yapar. Kabaca koruma için AES şifreleme ekleyebilirsin: 

using System.IO;
using System.Security.Cryptography;
using System.Text;

public static class EncryptedSave
{
    private static readonly byte[] key = Encoding.UTF8.GetBytes("32-CHAR-LONG-SECRET-KEY-12345678");
    private static readonly byte[] iv = Encoding.UTF8.GetBytes("16-CHAR-LONG-IV!");

    public static string Encrypt(string plain)
    {
        using (var aes = Aes.Create())
        {
            aes.Key = key; aes.IV = iv;
            var encryptor = aes.CreateEncryptor();
            byte[] inputBytes = Encoding.UTF8.GetBytes(plain);
            byte[] cipher = encryptor.TransformFinalBlock(inputBytes, 0, inputBytes.Length);
            return System.Convert.ToBase64String(cipher);
        }
    }

    public static string Decrypt(string encoded)
    {
        using (var aes = Aes.Create())
        {
            aes.Key = key; aes.IV = iv;
            var decryptor = aes.CreateDecryptor();
            byte[] cipher = System.Convert.FromBase64String(encoded);
            byte[] plain = decryptor.TransformFinalBlock(cipher, 0, cipher.Length);
            return Encoding.UTF8.GetString(plain);
        }
    }
}

// Kullanım:
string json = JsonUtility.ToJson(data);
string encrypted = EncryptedSave.Encrypt(json);
File.WriteAllText(path, encrypted);
⚠️ Mutlak Güvenlik Yok: Anahtarı oyunun kodunda taşıyorsan dedektif bir oyuncu çözebilir. AAA studio'lar sunucu tabanlı save (cloud save + server validation) kullanır. Single-player için AES yeterli engeldir — büyük çoğunluk için.
23.6

📋 ScriptableObject + Save Kombinasyonu

Modern Unity tasarımında save mimarisi sıklıkla ScriptableObject ile birleştirilir. Mantık:

  1. Veri konteyneri olarak PlayerData : ScriptableObject yarat.
  2. Runtime'da değişen değerler bu SO'da tutulur (level, health, inventory).
  3. Save zamanı SO'yu JSON'a serialize et.
  4. Load'da JSON'u okuyup SO'ya geri yaz.
  5. Tüm sistemler aynı SO'dan okuyup yazar — merkezi durum.
[CreateAssetMenu(menuName = "Game/Player Data")]
public class PlayerData : ScriptableObject
{
    public string playerName;
    public int level;
    public float currentHealth;
    public List<string> inventory;

    public void Save()
    {
        string json = JsonUtility.ToJson(this);
        File.WriteAllText(Path.Combine(
            Application.persistentDataPath, "player.json"), json);
    }

    public void Load()
    {
        string p = Path.Combine(Application.persistentDataPath, "player.json");
        if (!File.Exists(p)) return;
        JsonUtility.FromJsonOverwrite(File.ReadAllText(p), this);
    }
}

FromJsonOverwrite mevcut SO'nun field'larını JSON'dan üzerine yazar — yeni instance oluşturmaz. SO referansları korunur.

23.7

🎬 Ana Menü Tasarımı

Bir oyunun yüzü Ana Menü'dür. Tasarım sırası:

  1. Yeni sahne: MainMenu. Build Settings'e ekle, ilk sahne yap.
  2. Canvas ekle (Render Mode: Screen Space - Overlay).
  3. Background için RawImage (oyun temasının arka planı) veya tam ekran Image.
  4. VerticalLayoutGroup içinde butonlar: "Devam Et", "Yeni Oyun", "Ayarlar", "Çıkış".
  5. Her butona OnClick event'i bağla.
using UnityEngine;
using UnityEngine.SceneManagement;
using UnityEngine.UI;

public class MainMenuController : MonoBehaviour
{
    [SerializeField] private Button continueButton;
    [SerializeField] private GameObject settingsPanel;

    private void Start()
    {
        // Save yoksa Devam Et butonu kapalı
        continueButton.interactable = SaveSystem.HasSave();
    }

    public void OnContinueClicked()
    {
        PlayerSaveData data = SaveSystem.Load();
        SceneManager.LoadScene(data.sceneName);
    }

    public void OnNewGameClicked()
    {
        SaveSystem.Delete();
        SceneManager.LoadScene("Level01");
    }

    public void OnSettingsClicked() => settingsPanel.SetActive(true);
    public void OnQuitClicked() => Application.Quit();
}
23.8

⏸ Duraklama Menüsü (Pause Menu)

Oyun içinde Escape tuşuna basılınca oyun duruyor, menü beliriyor, geri dönüş veya ana menüye dönüş seçeneği veriyor. Klasik kalıp: 

using UnityEngine;
using UnityEngine.SceneManagement;

public class PauseMenu : MonoBehaviour
{
    [SerializeField] private GameObject menuPanel;
    public static bool IsPaused { get; private set; }

    private void Update()
    {
        if (Input.GetKeyDown(KeyCode.Escape))
        {
            if (IsPaused) Resume();
            else Pause();
        }
    }

    public void Pause()
    {
        menuPanel.SetActive(true);
        Time.timeScale = 0f;  // fizik ve animasyon dursun
        IsPaused = true;
        AudioListener.pause = true;
    }

    public void Resume()
    {
        menuPanel.SetActive(false);
        Time.timeScale = 1f;
        IsPaused = false;
        AudioListener.pause = false;
    }

    public void SaveAndQuit()
    {
        // game manager'a save'i çağırt
        FindFirstObjectByType<GameManager>().SaveGame();
        Time.timeScale = 1f;  // menü sahnesinde normal olsun
        SceneManager.LoadScene("MainMenu");
    }
}
⚠️ Time.timeScale Tuzakları: Pause sırasında Time.timeScale = 0, Update zaman delta'sı sıfır olur ama Update kendisi çalışmaya devam eder. UI ve input hâlâ çalışır. Ancak Animator "Animate Physics" değil "Normal" ise duracaktır. WaitForSeconds coroutine'leri donar. WaitForSecondsRealtime kullanarak menü içi animasyon yapabilirsin.
23.9

⚙️ Ayarlar Menüsü — Slider, Dropdown, Toggle

Ses, çözünürlük, kalite, kontrol seçimleri. Tüm değerleri PlayerPrefs'e yazıp Awake'te yükle. 

using UnityEngine;
using UnityEngine.UI;
using UnityEngine.Audio;
using TMPro;

public class SettingsMenu : MonoBehaviour
{
    [SerializeField] private AudioMixer mixer;
    [SerializeField] private Slider musicSlider, sfxSlider;
    [SerializeField] private TMP_Dropdown qualityDropdown, resolutionDropdown;
    [SerializeField] private Toggle fullscreenToggle;

    private Resolution[] resolutions;

    private void Start()
    {
        // Çözünürlük seçenekleri
        resolutions = Screen.resolutions;
        resolutionDropdown.ClearOptions();
        var opts = new List<string>();
        int currentIdx = 0;
        for (int i = 0; i < resolutions.Length; i++)
        {
            opts.Add($"{resolutions[i].width}x{resolutions[i].height}");
            if (resolutions[i].width == Screen.currentResolution.width)
                currentIdx = i;
        }
        resolutionDropdown.AddOptions(opts);
        resolutionDropdown.value = currentIdx;

        // Önceki ayarları yükle
        musicSlider.value = PlayerPrefs.GetFloat("MusicVol", 0.7f);
        sfxSlider.value = PlayerPrefs.GetFloat("SfxVol", 1f);
        qualityDropdown.value = QualitySettings.GetQualityLevel();
        fullscreenToggle.isOn = Screen.fullScreen;
    }

    public void SetMusicVolume(float v)
    {
        mixer.SetFloat("MusicVol", Mathf.Log10(Mathf.Max(v, 0.0001f)) * 20);
        PlayerPrefs.SetFloat("MusicVol", v);
    }

    public void SetQuality(int idx) => QualitySettings.SetQualityLevel(idx);
    public void SetFullscreen(bool on) => Screen.fullScreen = on;

    public void SetResolution(int idx)
    {
        Resolution r = resolutions[idx];
        Screen.SetResolution(r.width, r.height, Screen.fullScreen);
    }
}

Ses ayarı için logaritmik: AudioMixer'daki ses dB cinsindendir, slider 0-1 lineer. Mathf.Log10(v) * 20 dönüşümü insan kulağının logaritmik algısına uyar.

23.10

📚 Birden Fazla Save Slotu

AAA oyunlar genellikle 3-6 save slotu sunar. Mantık aynı, sadece her slot için ayrı dosya: 

public static class SlotSaveSystem
{
    private static string PathFor(int slot) =>
        Path.Combine(Application.persistentDataPath, $"slot_{slot}.json");

    public static void Save(int slot, PlayerSaveData data)
    {
        File.WriteAllText(PathFor(slot), JsonUtility.ToJson(data));
    }

    public static PlayerSaveData Load(int slot)
    {
        if (!File.Exists(PathFor(slot))) return null;
        return JsonUtility.FromJson<PlayerSaveData>(
            File.ReadAllText(PathFor(slot)));
    }

    public static List<PlayerSaveData> LoadAll(int slots = 3)
    {
        var list = new List<PlayerSaveData>();
        for (int i = 0; i < slots; i++) list.Add(Load(i));
        return list;
    }
}

UI tarafında her slot için bir buton; üzerinde sahne adı, level, oynama süresi, save tarihi gösterilir. Boş slot için "+" işareti.

23.11

⏱ Otomatik Save (Autosave)

Modern oyunlar 30-60 saniyede bir veya önemli olaylardan sonra otomatik kayıt yapar. Coroutine ile: 

private Coroutine autosaveRoutine;

private void Start()
{
    autosaveRoutine = StartCoroutine(AutosaveLoop());
}

private IEnumerator AutosaveLoop()
{
    while (true)
    {
        yield return new WaitForSeconds(60f);
        SaveGame();
        UIManager.ShowToast("Oyun otomatik kaydedildi");
    }
}

// Boss bölgesi giriş, item alma, level atlama gibi olaylarda da:
public void OnImportantEvent()
{
    SaveGame();
}
23.12

☁️ Cloud Save — Unity Gaming Services

Mobile/multiplayer oyunlar için Unity Cloud Save servisi kullanılır. Unity Dashboard'tan projeyi UGS'ye bağla, paketi yükle: 

using Unity.Services.CloudSave;
using Unity.Services.Core;
using Unity.Services.Authentication;

private async void Start()
{
    await UnityServices.InitializeAsync();
    await AuthenticationService.Instance.SignInAnonymouslyAsync();
}

public async void CloudSave(PlayerSaveData data)
{
    var dict = new Dictionary<string, object>
    {
        { "level", data.level },
        { "health", data.currentHealth },
        { "inventory", JsonUtility.ToJson(data.inventory) }
    };
    await CloudSaveService.Instance.Data.Player.SaveAsync(dict);
}

Cloud save oyuncunun farklı cihazlardan girince aynı progresi görmesini sağlar. Free tier küçük oyunlar için yeterli, paid plan büyük veriler için.

Time.timeScale = 0 ile pause, AudioListener.pause ile ses durdurma. AudioMixer log dönüşümü. Birden fazla slot, autosave, cloud save — ihtiyaca göre kademeli ekleme.

🎯 Pratik Projeler

Proje 1: PlayerPrefs ile Yüksek Skor

Hedef: Basit bir top toplama oyunu yap, oturumlar arasında en yüksek skoru hatırlatsın.

  1. Sahnede topları topla, score değişkenini arttır.
  2. Oyun bittiğinde mevcut score > PlayerPrefs.GetInt("HighScore") ise yeni high score kaydet.
  3. Ana menüde HighScore'u TextMeshPro ile göster.
  4. "Sıfırla" butonu: PlayerPrefs.DeleteKey("HighScore").

Proje 2: JSON ile Karakter Save/Load

Hedef: Karakterin konumu, canı ve envanteri JSON dosyasına yazılsın, oyun açıldığında geri yüklensin.

  1. PlayerSaveData class'ını oluştur (Serializable, public field'lar).
  2. SaveSystem static class'ını yaz (Save, Load metodları).
  3. GameManager içinde Esc menüsünde Save butonu ekle.
  4. Oyun başlangıcında SaveSystem.HasSave() ise oyuncuya "Devam Et?" sor.
  5. Save dosyasının yolunu Debug.Log ile yazdır, dosyayı Notepad'de aç, JSON içeriğini incele.

Proje 3: 3 Slot'lu Ana Menü

Hedef: Ana menüde 3 save slotu görünsün; her slotta varsa "Devam Et", yoksa "Yeni Oyun" yazsın.

  1. SlotSaveSystem class'ını oluştur (slot indexli dosyalar).
  2. Ana menü sahnesinde 3 buton + her birinde TextMeshPro ile slot bilgisi.
  3. MainMenuController.Start()'ta SlotSaveSystem.LoadAll(3) çağır, butonların yazılarını güncelle.
  4. Boş slot: "Yeni Oyun" → Level01'i yükle, yeni save oluştur.
  5. Dolu slot: "Lv 5 — 12:34" → o sahneyi yükle.
  6. Bonus: Her slotun yanına "Sil" küçük butonu ekle.
