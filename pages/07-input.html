Input Sistemi — Oyuncunun Sesini Duymak

Oyun, oyuncuya tepki verdiği an doğar. Klavye basışı, fare hareketi, dokunmatik dokunuşu, gamepad sallaması — tüm bunlar sayısal veriye çevrilip script'inize akar. Bu bölümde Unity'nin eski Input Manager'ını (hâlâ en yaygın yöntem) sıfırdan öğreneceğiz; yeni Input System'ı ise Bölüm 22'de derinlemesine işleyeceğiz.

⏱ 30 dakika ⌨ Klavye, fare, gamepad 📱 Touch input
7.1

İki Input Sistemi — Hangisini Seçmeliyim?

Unity 2026 itibarıyla Input için iki ayrı sistem sunuyor. Editor menülerinde Project Settings altında ikisini de görebilirsin: eski Input Manager ve yeni Input System Package. Yeni gelen bir geliştiriciye en kafa karıştıran sorulardan biri "Hangisini öğreneyim?".

Eski Input Manager (Built-in)
  • Yıllardır var, projelerin %90'ında hâlâ kullanılıyor
  • Tek satırlık API: Input.GetKey(KeyCode.W)
  • Kurulum gerektirmez — yeni projede hazır gelir
  • Gamepad, touch, sensör desteği sınırlı
  • Tutorial'ların büyük çoğunluğu bunu anlatır
  • Hızlı prototip için ideal
Yeni Input System (Package)
  • Aksiyon-tabanlı (Action-based) modern mimari
  • Birden çok cihaz desteği — gamepad, VR, mobil aynı kodla
  • Rebinding (oyuncu tuş atayabilir) hazır gelir
  • Package Manager'dan kurulur (com.unity.inputsystem)
  • Öğrenme eğrisi daha dik — daha çok kurulum
  • Unity 6 önerisi yeni projelerde
💡 Bu bölümde neden eskisini öğreniyoruz?

İki nedenle: birincisi, Input Manager kavramsal olarak daha basit ve seni "input"un ne demek olduğuna alıştırır; ikincisi, internetteki Unity tutorial'larının ezici çoğunluğu Input Manager'a dayanıyor — eski projelere katılırken, tutorial izlerken okuyabilmen gerekiyor. Yeni sistemi Bölüm 22'de detaylı işleyeceğiz; o zaman iki dünyayı kıyaslayabilecek seviyede olacaksın.

7.2

Klavye — GetKey Üçlemesi

Bir tuşun durumunu sorgulamak için üç farklı metot var. İsimleri benzer, davranışları çok farklı; üçünün arasındaki farkı oturtmak Input öğrenmenin yarısıdır.

📄 KeyboardBasics.cs 
using UnityEngine;

public class KeyboardBasics : MonoBehaviour
{
    void Update()
    {
        // 1) GetKey — her karede true döner (tuş basılı kaldığı sürece)
        if (Input.GetKey(KeyCode.W))
        {
            Debug.Log("W basılı tutuluyor");
        }

        // 2) GetKeyDown — yalnızca basıldığı karede true
        if (Input.GetKeyDown(KeyCode.Space))
        {
            Debug.Log("Zıpla!");   // Tek seferlik aksiyon
        }

        // 3) GetKeyUp — yalnızca bırakıldığı karede true
        if (Input.GetKeyUp(KeyCode.Space))
        {
            Debug.Log("Boşluk bırakıldı");
        }
    }
}
GetKey
"Şu an basılı mı?" — sürekli hareket, çekim modu
GetKeyDown
"Şimdi mi basıldı?" — zıplama, ateşleme, menü
GetKeyUp
"Şimdi mi bırakıldı?" — yay çekmeyi durdur, üfle

Pratik kural: Sürekli yapılan işlerde (yürüme, kamera döndürme, gaz vermek) GetKey kullan. Bir kez yapılan tetik aksiyonlarda (zıpla, ateş et, menü aç) GetKeyDown kullan. Bırakıldığında tetiklenen geri-besleme (yayı bırak, ateşi kes) için GetKeyUp kullan.

⚠ GetKeyDown'ı FixedUpdate'e koyma

GetKeyDown ve GetKeyUp kareye bağlı olaylardır. Eğer bunları FixedUpdate içine koyarsan basışı kaçırma riski vardır — çünkü FixedUpdate kareden bağımsız çalışır ve bazen aynı kare içinde hiç çağrılmaz, bazen iki kez çağrılır. Tetik aksiyonlar (zıplama, ateşleme) her zaman Update'te yakalanmalı; ardından bir bool jumpRequested flag'i set edilip FixedUpdate'te tüketilmeli.

7.3

KeyCode vs String — İki Yazım Stili

Input.GetKey iki parametre türünü kabul eder: KeyCode enum'u veya string. İkisinin de doğru, ama tercihleri farklıdır.

📄 KeyCodeVsString.cs 
using UnityEngine;

public class KeyCodeVsString : MonoBehaviour
{
    void Update()
    {
        // KeyCode enum — derleme zamanı kontrol, IntelliSense desteği
        if (Input.GetKeyDown(KeyCode.Escape)) { }
        if (Input.GetKeyDown(KeyCode.LeftShift)) { }
        if (Input.GetKeyDown(KeyCode.Alpha1)) { }   // 1 tuşu (alfanumerik)
        if (Input.GetKeyDown(KeyCode.Keypad1)) { }   // numpad 1

        // String — yazım hatası fark edilmez (uyarı vermez)
        if (Input.GetKeyDown("escape")) { }
        if (Input.GetKeyDown("left shift")) { }
    }
}

KeyCode'u tercih et — IDE seni otomatik tamamlar, yazım hatası yapamazsın, refactor güvenlidir. string versiyonu sadece dinamik tuş ataması yapılan (örneğin kullanıcı menüsünden tuş seçen) senaryolarda işe yarar.

7.4

Axis Input — Analog Hareketin Sırrı

Input.GetAxis, klavyenin "açık/kapalı" doğasını analog yumuşaklığa çevirir. Horizontal ve Vertical adında iki varsayılan eksen vardır; geri döndürdükleri değer -1f ile +1f arasında değişir.

📄 AxisExamples.cs 
using UnityEngine;

public class AxisExamples : MonoBehaviour
{
    public float hiz = 5f;

    void Update()
    {
        // Yatay: A/Sol Ok = -1, D/Sağ Ok = +1, basılı değilse 0
        float h = Input.GetAxis("Horizontal");

        // Dikey: W/Yukarı Ok = +1, S/Aşağı Ok = -1
        float v = Input.GetAxis("Vertical");

        Vector3 hareket = new Vector3(h, 0f, v) * hiz * Time.deltaTime;
        transform.Translate(hareket);
    }
}

GetAxis vs GetAxisRaw

Unity ekstra bir varyant daha sunar: Input.GetAxisRaw. Aralarındaki fark yumuşatma (smoothing):

Input.GetAxis
  • Değer 0'dan 1'e yumuşak geçer (interpolated)
  • Klavye basışı bile analog hissedilir
  • Araba, uçak gibi sürüklenmeli kontrol için iyi
  • Karakter platform oyunları için kötü — kayma hissi verir
Input.GetAxisRaw
  • Sadece -1, 0, +1 döner — keskin
  • Tuşa bastığın an direkt değer değişir
  • Platform oyunu, FPS, retro 2D için ideal
  • Gamepad ile kullandığında joystick'in gerçek değerini verir
💡 Yumuşatmayı kendin yap

Çoğu profesyonel proje GetAxisRaw ile ham değeri alır ve yumuşatmayı kendisi yapar — Mathf.Lerp ile veya bir SmoothDamp çağrısıyla. Bu sayede her durumda kontrolün hızına ve yumuşatma karakterine sen karar verirsin.

7.5

Fare Input — Konum, Tıklama, Tekerlek

📄 MouseInput.cs 
using UnityEngine;

public class MouseInput : MonoBehaviour
{
    void Update()
    {
        // Tıklama — index 0=sol, 1=sağ, 2=orta
        if (Input.GetMouseButtonDown(0)) Debug.Log("Sol tık");
        if (Input.GetMouseButtonDown(1)) Debug.Log("Sağ tık");
        if (Input.GetMouseButton(0))     Debug.Log("Sol basılı");

        // Fare konumu — ekran piksel koordinatları (sol-alt 0,0)
        Vector3 mp = Input.mousePosition;
        Debug.Log($"Fare: {mp.x}, {mp.y}");

        // Fare delta — son kareden bu kareye ne kadar hareket etti
        float mx = Input.GetAxis("Mouse X");
        float my = Input.GetAxis("Mouse Y");

        // Tekerlek — yukarı +, aşağı -
        float tekerlek = Input.GetAxis("Mouse ScrollWheel");
        if (tekerlek != 0f) Debug.Log($"Tekerlek: {tekerlek}");
    }
}

FPS Kamera için Klasik Kalıp

using UnityEngine;

public class FpsCamera : MonoBehaviour
{
    public float hassasiyet = 2f;
    private float dikeyAci = 0f;

    void Start()
    {
        Cursor.lockState = CursorLockMode.Locked;   // İmleci gizle ve kilitle
    }

    void Update()
    {
        float mx = Input.GetAxis("Mouse X") * hassasiyet;
        float my = Input.GetAxis("Mouse Y") * hassasiyet;

        // Karakter Y ekseninde döner (sağa-sola)
        transform.Rotate(0f, mx, 0f);

        // Kamera X ekseninde döner (yukarı-aşağı), kısıtlı
        dikeyAci -= my;
        dikeyAci = Mathf.Clamp(dikeyAci, -80f, 80f);
        Camera.main.transform.localEulerAngles = new Vector3(dikeyAci, 0f, 0f);
    }
}
⚠ Cursor.lockState ve Editor

CursorLockMode.Locked'ı Editor'de denerken kilitten kurtulamayabilirsin. Editor'de Escape tuşu otomatik olarak kilit modunu kırar — bunu kendin override etmen gerekir. Build aldığında bu davranış kaybolur ve oyunda Escape tuşuyla menüye dönmeyi sen yönetirsin.

⚡ Mini-Checkpoint — Input Refleksi

Devam etmeden önce şu beş ayrımı zihninde oturt:

  1. GetKey sürekli, GetKeyDown ve GetKeyUp tek seferlik.
  2. GetAxis yumuşatılmış (-1..+1), GetAxisRaw ham (-1, 0, +1).
  3. Tetik aksiyonlar Update'te yakalanır, FixedUpdate'te değil.
  4. Fare delta için Input.GetAxis("Mouse X/Y"), ekran konumu için Input.mousePosition.
  5. KeyCode enum'u string'e tercih edilir — IDE desteği ve tip güvenliği.
7.6

Input Manager — Project Settings

Edit › Project Settings › Input Manager menüsünden Unity'nin varsayılan eksenlerini görebilir ve özelleştirebilirsin. Burada her axis bir kural setidir: hangi tuşlara pozitif, hangilerine negatif gibi.

Gravity: tuş bırakıldığında değerin sıfıra dönme hızı. Sensitivity: tuşa basıldığında değerin hedefe gitme hızı. İkisi de saniyedeki birim cinsindendir. Yüksek değerler "keskin" hissini verir; Gravity ve Sensitivity ikisini birden 1000 yaparsan GetAxisRaw gibi davranır (yumuşatma yok).

Özel Axis Tanımlamak

Varsayılan Horizontal ve Vertical'a ek olarak kendi axis'lerini tanımlayabilirsin. Örneğin Crouch adında bir axis, LeftCtrl ile +1 veren bir tetik:

  1. Edit › Project Settings › Input Manager açılır.
  2. Axes listesi genişletilir, Size bir artırılır.
  3. Listenin sonunda görünen yeni axis seçilir, Name'i Crouch yapılır.
  4. Positive Button alanına left ctrl yazılır.
  5. Script içinde Input.GetAxis("Crouch") > 0.5f ile sorgulanır.
💡 Neden axis yerine doğrudan tuş okumayalım?

İyi soru — küçük projelerde fark etmez. Ama gamepad desteği eklemek istediğinde, Linux'ta tuş farklılıklarını karşılamak istediğinde veya oyuncuya tuş atama menüsü vermek istediğinde axis soyutlaması işe yarar. Eski Input Manager'da gamepad butonları joystick button 0, joystick button 1 gibi numaralarla gelir — axis tanımı bu karmaşayı senin için saklar.

7.7

Touch Input — Mobil İçin Temeller

Mobil platformlarda klavye yerine parmak var. Unity'de touch input'u okumak için Input.touches dizisini kullanırız.

📄 TouchBasics.cs 
using UnityEngine;

public class TouchBasics : MonoBehaviour
{
    void Update()
    {
        if (Input.touchCount > 0)
        {
            Touch t = Input.GetTouch(0);   // İlk parmak

            switch (t.phase)
            {
                case TouchPhase.Began:
                    Debug.Log($"Parmak değdi: {t.position}");
                    break;

                case TouchPhase.Moved:
                    Debug.Log($"Hareket: {t.deltaPosition}");
                    break;

                case TouchPhase.Stationary:
                    Debug.Log("Parmak duruyor");
                    break;

                case TouchPhase.Ended:
                    Debug.Log("Parmak kalktı");
                    break;
            }
        }
    }
}

Touch'un dört fazı: Began (parmağın değdiği an), Moved (kayıyor), Stationary (basılı duruyor), Ended (kalktı). Canceled de var — sistem touch'u iptal ettiğinde (örn. telefon görüşmesi gelince) tetiklenir.

Çoklu Parmak — Pinch Zoom

using UnityEngine;

public class PinchZoom : MonoBehaviour
{
    void Update()
    {
        if (Input.touchCount == 2)
        {
            Touch t0 = Input.GetTouch(0);
            Touch t1 = Input.GetTouch(1);

            // Önceki kareden bu kareye iki parmak arasındaki mesafe farkı
            Vector2 oncekiT0 = t0.position - t0.deltaPosition;
            Vector2 oncekiT1 = t1.position - t1.deltaPosition;

            float onceMesafe = Vector2.Distance(oncekiT0, oncekiT1);
            float simdiMesafe = Vector2.Distance(t0.position, t1.position);

            float fark = onceMesafe - simdiMesafe;
            Camera.main.fieldOfView += fark * 0.05f;
        }
    }
}
💡 Editor'de touch'u simüle etmek

Editor'de telefon olmadığı için touch testlerini direkt yapamazsın. İki yöntem var: Unity Remote uygulamasını telefona kur ve USB ile bağla — telefondaki touch direkt Editor'e iletilir. Veya Project Settings › Input Manager › Active Input Handling'i geçici olarak fare ile simüle edecek şekilde ayarla. Yeni Input System bu simülasyonu çok daha temiz çözer (Bölüm 22).

7.8

Gamepad Tuşları — Joystick Button İsimleri

Eski Input Manager gamepad'i klavyenin uzantısı gibi görür: butonlara JoystickButton0, JoystickButton1... olarak numaralarla erişirsin; tetikler ve sol/sağ joystick'ler axis olarak gelir. 

using UnityEngine;

public class GamepadOld : MonoBehaviour
{
    void Update()
    {
        // Sol joystick — Horizontal/Vertical zaten klavye + gamepad birleşik gelir
        float sx = Input.GetAxis("Horizontal");
        float sy = Input.GetAxis("Vertical");

        // Butonlar — A=0, B=1, X=2, Y=3 (Xbox tarzı)
        if (Input.GetKeyDown(KeyCode.JoystickButton0))
            Debug.Log("A tuşu");

        if (Input.GetKeyDown(KeyCode.JoystickButton1))
            Debug.Log("B tuşu");
    }
}
⚠ Gamepad ve Eski Input'un sınırları

Eski Input Manager Xbox, PlayStation ve generic gamepad'leri farklı düzenlerle algılar — buton numaraları cihazdan cihaza değişebilir. PS5 DualSense'in haptic feedback'ine erişemezsin; VR kontrolcüleri okuyamazsın. Modern, çok-platformlu input için yeni Input System tek doğru yoldur — Bölüm 22'de bunu çözeceğiz.

🎮 Gerçek hayattan örnek — İlk top-down hareket sistemin

Klasik bir top-down (yukarıdan bakış) oyunda karakter WASD ile hareket eder, fare imlecinin olduğu yere döner, sol tıkla ateş eder. Eski Input Manager'ın yapabildiklerinin tam ortasında yer alan bir kalıp: 

using UnityEngine;

public class TopDownController : MonoBehaviour
{
    public float hiz = 6f;
    public GameObject mermiPrefab;

    void Update()
    {
        // 1) WASD — düz hareket (raw, keskin hissedilsin)
        float h = Input.GetAxisRaw("Horizontal");
        float v = Input.GetAxisRaw("Vertical");
        Vector3 yon = new Vector3(h, 0f, v).normalized;
        transform.position += yon * hiz * Time.deltaTime;

        // 2) Fare imlecine doğru dön
        Ray ray = Camera.main.ScreenPointToRay(Input.mousePosition);
        if (Physics.Raycast(ray, out RaycastHit hit))
        {
            Vector3 bakHedef = hit.point;
            bakHedef.y = transform.position.y;       // Yana eğilmesin
            transform.LookAt(bakHedef);
        }

        // 3) Sol tık — ateş et
        if (Input.GetMouseButtonDown(0))
        {
            Instantiate(mermiPrefab, transform.position, transform.rotation);
        }
    }
}

Bu 20 satırlık script üç farklı input kanalını birleştirir: klavye axisleri, fare konumu (ray ile dünyaya projekte edilmiş), fare butonu. Eski Input Manager'ın gücünü gösteren bir örnek — kısa, okunaklı, hızlı yazılır.

🛠

Pratik — Üç Mini Proje

🎯 Proje 1 — Klavye ile Renk Değiştir

Sahnede bir Cube oluştur. Bir script bağla. R, G, B tuşlarına bastığında küpün rengi sırasıyla kırmızı, yeşil, mavi olsun.

Kontrol: GetComponent<Renderer>().material.color = Color.red; satırı işine yarayacak. GetKeyDown kullan — sürekli tetiklenmesin.
🎯 Proje 2 — Mouse ile Top Fırlat

Bir top prefab'ı oluştur (Sphere + Rigidbody). Boş bir GameObject olarak Fırlatıcı yarat ve ona script bağla. Sol tıklayınca fareye doğru top fırlatsın (Rigidbody.AddForce ile).

Kontrol: Top fareye doğru gitmeli. İpucu: Camera.main.ScreenPointToRay(Input.mousePosition) ile dünyada bir yön elde edebilirsin.
🎯 Proje 3 — Sprint Tuşuyla Koşma

Karakterin Vertical axis'iyle ileri-geri hareket etsin. LeftShift basılıyken hızı 2 katına çıksın (sprint). Bıraktığında normal hıza dönsün.

Kontrol: float aktifHiz = Input.GetKey(KeyCode.LeftShift) ? hiz * 2f : hiz; gibi tek satırla halledebilirsin.
