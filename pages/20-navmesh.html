NavMesh & AI Navigation — Akıllı Karakter Hareketi

NavMesh Bake, NavMeshAgent, Obstacle, Off-Mesh Link, NavMesh Areas, Unity 6 AI Navigation paketi ile runtime baking — düşman/NPC hareketi.

⏱ Okuma süresi: ~26 dk 🤖 4 AI davranış örneği 🎯 Zorluk: Orta-İleri
20.1

🗺 NavMesh Nedir?

Bir düşmanın oyuncuyu takip etmesi sandığından çok daha zordur. transform.position = Vector3.MoveTowards(transform.position, player.position, speed * Time.deltaTime); yazdığında düşman duvarlara çarpar, masaların üstüne çıkar, merdivenleri görmezden gelir. Çünkü o "yürüyemez" — sadece koordinata hareket eder.

NavMesh (Navigation Mesh), sahnenin yürünebilir yüzeylerinden türetilmiş soyut bir poligon ağı'dır. Mavi şeffaf bir örtü gibi zeminlerin ve merdivenlerin üzerine yayılır. Karakterin AI'ı bu örtüye bakar ve "A noktasından B noktasına en kısa yol bu poligonları geçer" hesabını yapar. Duvar, masa, falez NavMesh'in dışındadır — AI orayı görmez bile.

NavMesh'i bir kez bake edersin (pişirirsin) ve sonra NavMeshAgent komponenti olan herhangi bir karakter, agent.SetDestination(target); dediğinde tek satırla en kısa yol'u (A* algoritması ile) bulur, fiziksel olarak takip eder.

🎮 Gerçek Dünyadan: The Elder Scrolls V: Skyrim

Tamriel'in ovalarındaki kurtlar, mağaralardaki goblinler, şehirlerdeki vatandaşlar — hepsi NavMesh üzerinde hareket eder. Bethesda'nın Creation Engine'ı bu işi kendi NavMesh sistemi ile yapar, ancak prensip aynıdır: tasarımcı dünyayı kurar, motor bir kez NavMesh bake eder, NPC'ler binlerce satır path-finding kodu olmadan zekice gezinir. Unity'de Cuphead, Hollow Knight'ın 3D haritalı bölümleri, Risk of Rain 2'nin tüm düşman AI'sı NavMeshAgent üzerine kuruludur.

20.2

🆚 İki Sistem: Klasik vs AI Navigation Paketi

Unity 6'da NavMesh için iki sistem kullanılabilir:

🧱 Klasik NavMesh (Built-in)

  • Window → AI → Navigation menüsü
  • Mesh'leri Navigation Static işaretle
  • Bake butonu sahne tabanlı bir NavMesh üretir
  • Tüm sahne için tek bir NavMesh
  • Runtime'da değiştirmek zor
  • Eski projeler için hâlâ destekleniyor

🚀 AI Navigation Paketi YENİ

  • Package Manager'dan com.unity.ai.navigation
  • NavMeshSurface componenti — istediğin GameObject'e ekle
  • Birden fazla yüzey, runtime bake, farklı agent tipleri
  • NavMeshLink (prefab arası bağlantı)
  • NavMeshModifier ile alan tipleri
  • Modüler ve prefab dostu — yeni projeler için önerilir
📦 Hangisini Kullanmalı? Yeni bir proje açıyorsan, ilk iş Package Manager'dan AI Navigation paketini yüklemek. Eski Window → AI → Navigation menüsü hâlâ açılır ama yeni sistem her açıdan üstün: prefab'a NavMeshSurface koyup runtime bake edersin, procedural sahneler bile NavMesh sahibi olur.
20.3

🍳 NavMesh Bake — Sahneyi Pişirmek

AI Navigation paketi yüklü olduğunu varsayarak (Package Manager → AI Navigation → Install) adım adım bake edelim:

  1. Sahnede yürünebilir yüzeyleri (zemin, merdiven, platform) toplayıcı bir empty GameObject oluştur: NavMeshSource.
  2. Yürünebilir tüm zeminleri bu objenin child'ı yap (Hierarchy'de drag-drop).
  3. NavMeshSource'a NavMeshSurface bileşeni ekle (Add Component → Navigation → NavMeshSurface).
  4. Inspector'da:
    • Agent Type: Humanoid (varsayılan).
    • Collect Objects: Children — sadece bu objenin altındakileri tara.
    • Use Geometry: Render Meshes (mesh'in görünür yüzünü kullan) veya Physics Colliders (collider'ı kullan).
  5. Bake butonuna bas. Sahnede mavi şeffaf NavMesh örtüsü belirir.

Eğer bir mesh NavMesh'e dahil edilmemeli (örn. duvarın içindeki dekor) ya da farklı şekilde değerlendirilmeli (örn. su = pahalı maliyet) ise üzerine NavMeshModifier bileşeni ekle ve Ignore From Build ya da Override Area seçeneklerini ayarla.

Max Slope45°

Agent Radius ve Height, NavMesh'in nasıl şekilleneceğini belirler. Geniş bir agent için duvar diplerinde NavMesh çekilir — şişman düşman dar geçide girmesin diye. Max Slope 45°'den dik yüzeyler yürünmez sayılır, NavMesh oraya gitmez. Step Height 40 cm'lik basamağı agent atlayabilir kabul eder.

20.4

🚶 NavMeshAgent — Hareketli Karakterin Beyni

Bir düşman karakter prefab'ına (Capsule mesh + Mesh Renderer + Animator) NavMeshAgent bileşeni eklediğin an, sahnede NavMesh varsa otomatik onun üzerine yapışır.

NavMeshAgent
Agent TypeHumanoid
Base Offset0
Speed3.5 m/s
Angular Speed120°/sn
Acceleration8
Stopping Distance0.5
Auto Braking☑ True
Radius0.5
Height2.0
Obstacle AvoidanceHigh Quality
Priority50
  • Speed — Maksimum hareket hızı (m/s).
  • Angular Speed — Dönme hızı. Düşük olursa karakter köşelerde geniş dönüş çizer, gerçekçi olur.
  • Acceleration — Sıfırdan hıza ulaşma. Düşük = ağır hisseden tank, yüksek = çevik.
  • Stopping Distance — Hedeften ne kadar uzakta dursun. 0.5 = yarım metre. Saldırı menziline gelince durması için 2-3 m yapılır.
  • Auto Braking — Hedefe yaklaşırken yavaşlasın mı? Patrol için kapat (sürekli sabit hız), saldırı için aç.
  • Obstacle Avoidance: None (kapalı) → Low → Medium → Good → High → High Quality. Birbirine yakın çok sayıda agent varsa daha pahalı ama daha akıcı.
  • Priority — İki agent çakışırsa hangisi yol verir? Düşük = daha öncelikli.
20.5

🐺 İlk AI: Oyuncuyu Takip Eden Düşman

En basit AI: oyuncuyu sürekli takip eden bir karakter.

using UnityEngine;
using UnityEngine.AI;

[RequireComponent(typeof(NavMeshAgent))]
public class EnemyChaser : MonoBehaviour
{
    [SerializeField] private Transform player;
    [SerializeField] private float updateInterval = 0.2f;

    private NavMeshAgent agent;
    private float nextUpdate;

    private void Awake()
    {
        agent = GetComponent<NavMeshAgent>();
    }

    private void Update()
    {
        if (player == null) return;

        // Her frame SetDestination çağırmak pahalıdır
        // 0.2 saniyede bir güncelle yeterli
        if (Time.time >= nextUpdate)
        {
            agent.SetDestination(player.position);
            nextUpdate = Time.time + updateInterval;
        }
    }
}
⚠️ Performans Notu: SetDestination() A* path query'sini tetikler — pahalıdır. Her frame değil, 0.1-0.3 saniyede bir çağırmak yeterlidir. Yüzlerce düşmanı olan haritalarda bu fark = 60 FPS vs 25 FPS.
20.6

🚶‍♂️ Patrol AI — Waypoint Sistemi

Devriye dolaşan bir nöbetçi karakter:

using UnityEngine;
using UnityEngine.AI;

[RequireComponent(typeof(NavMeshAgent))]
public class PatrolAI : MonoBehaviour
{
    [SerializeField] private Transform[] waypoints;
    [SerializeField] private float waitTime = 2f;

    private NavMeshAgent agent;
    private int currentIndex = 0;
    private float waitTimer;
    private bool waiting = false;

    private void Awake()
    {
        agent = GetComponent<NavMeshAgent>();
        agent.autoBraking = false;  // patrol için sabit hız
        GoToNextWaypoint();
    }

    private void Update()
    {
        if (waiting)
        {
            waitTimer -= Time.deltaTime;
            if (waitTimer <= 0)
            {
                waiting = false;
                GoToNextWaypoint();
            }
            return;
        }

        // Hedefe ulaştı mı?
        if (!agent.pathPending &&
            agent.remainingDistance < 0.5f)
        {
            waiting = true;
            waitTimer = waitTime;
        }
    }

    private void GoToNextWaypoint()
    {
        if (waypoints.Length == 0) return;
        agent.SetDestination(waypoints[currentIndex].position);
        currentIndex = (currentIndex + 1) % waypoints.Length;
    }
}

Bu nöbetçi: ilk waypoint'e gider → 2 saniye bekler → bir sonraki waypoint'e gider → tekrar. Sahnede empty GameObject'lerle waypoint'leri yerleştir ve Inspector'da waypoints array'ine sırayla sürükle.

20.7

🧠 Davranış State Machine — Patrol + Chase + Attack

Gerçek bir düşman birden fazla davranışa sahip olmalı: devriye geziniyor, oyuncuyu fark edince kovalıyor, menzile gelince saldırıyor. State machine pattern bunun standart yoludur. 

using UnityEngine;
using UnityEngine.AI;

public enum EnemyState { Patrol, Chase, Attack }

[RequireComponent(typeof(NavMeshAgent))]
public class EnemyAI : MonoBehaviour
{
    [SerializeField] private Transform[] waypoints;
    [SerializeField] private Transform player;
    [SerializeField] private float sightRange = 10f;
    [SerializeField] private float attackRange = 2f;
    [SerializeField] private float attackCooldown = 1.5f;

    private NavMeshAgent agent;
    private EnemyState state = EnemyState.Patrol;
    private int wpIndex = 0;
    private float lastAttack;

    private void Awake() => agent = GetComponent<NavMeshAgent>();

    private void Update()
    {
        float dist = Vector3.Distance(transform.position, player.position);

        switch (state)
        {
            case EnemyState.Patrol:
                PatrolBehaviour();
                if (dist < sightRange) state = EnemyState.Chase;
                break;

            case EnemyState.Chase:
                agent.SetDestination(player.position);
                if (dist < attackRange) state = EnemyState.Attack;
                else if (dist > sightRange * 1.5f) state = EnemyState.Patrol;
                break;

            case EnemyState.Attack:
                agent.SetDestination(transform.position);  // dur
                FacePlayer();
                if (Time.time > lastAttack + attackCooldown)
                {
                    Attack();
                    lastAttack = Time.time;
                }
                if (dist > attackRange) state = EnemyState.Chase;
                break;
        }
    }

    private void PatrolBehaviour()
    {
        if (!agent.pathPending && agent.remainingDistance < 0.5f)
        {
            agent.SetDestination(waypoints[wpIndex].position);
            wpIndex = (wpIndex + 1) % waypoints.Length;
        }
    }

    private void FacePlayer()
    {
        Vector3 dir = (player.position - transform.position).normalized;
        dir.y = 0;
        transform.rotation = Quaternion.Slerp(transform.rotation,
            Quaternion.LookRotation(dir), 8f * Time.deltaTime);
    }

    private void Attack()
    {
        Debug.Log("Saldırı!");
        // Hasar ver, animasyon tetikle, vs.
    }
}

Üç state, üç davranış. Chase'den Patrol'e geri dönüş için sightRange * 1.5 kullanılır — bu hysteresis'tir, agent tam sınırda gidip gelmesin diye.

20.8

🚧 NavMesh Obstacle — Dinamik Engeller

Bake edilmiş NavMesh statiktir — runtime'da bir varil sahneye düşerse NavMesh onu fark etmez, agent içinden geçer. Çözüm: o varile NavMeshObstacle bileşeni ekle.

NavMeshObstacle (Cylinder şekli)
ShapeBox / Capsule
Center(0, 0, 0)
Size(1, 2, 1)
Carve☑ True
Move Threshold0.1
Time To Stationary0.5
Carve Only Stationary☑ True
  • Carve OFF — Agent'lar obstacle'ı bir "kütle" olarak görür ve etrafından dolaşmaya çalışır (lokal avoidance). Hafif maliyetli ama %100 garanti değil.
  • Carve ON — Obstacle, NavMesh'ten o anda "delik açar" (oyma). Agent'lar onu duvar gibi sayar. Pahalı ama kesin sonuç.
  • Carve Only Stationary — Obstacle hareket ederken oyma yapma, dururken yap. Hareketli engellerde performans için zorunlu.

İdeal kullanım: Düşmanlar kalıcı kapılar (kilitliyken kapı=obstacle, açılınca devre dışı), trafik konileri, devrilebilir bariyerler için.

20.9

🌉 Off-Mesh Link / NavMeshLink — Atlama & Köprü

NavMesh sürekli bir yüzeydir; ama gerçek dünyada agent'lar uçurumdan atlamalı, halatla iniş yapmalı, kapıdan geçmeli. NavMeshLink bu kopuk noktaları birbirine bağlar.

  1. Empty GameObject oluştur, üzerine NavMeshLink bileşeni ekle.
  2. Inspector'da Start Transform ve End Transform alanlarına iki nokta ata (örn. uçurumun iki kenarındaki marker'lar).
  3. Bidirectional ☑ (her iki yönde kullanılabilir).
  4. Cost Modifier = 5 (bu yolu kullanmak normal yürümekten 5 kat pahalı — agent yalnızca alternatif yoksa kullanır).
  5. Sahneyi Bake et — agent artık iki nokta arasını "link" üzerinden geçebilir.

Agent link'i geçerken hareketi otomatik düz interpolation'dur — eğer "zıplama" animasyonu istiyorsan agent.isOnOffMeshLink'i kontrol edip Animator parametresi tetiklersin: 

private void Update()
{
    if (agent.isOnOffMeshLink)
    {
        animator.SetBool("Jumping", true);
        // Manual link traversal
        OffMeshLinkData data = agent.currentOffMeshLinkData;
        Vector3 target = data.endPos + Vector3.up * agent.baseOffset;
        agent.transform.position = Vector3.MoveTowards(
            agent.transform.position, target, agent.speed * Time.deltaTime);
        if (agent.transform.position == target)
        {
            agent.CompleteOffMeshLink();
            animator.SetBool("Jumping", false);
        }
    }
}
20.10

🏞 NavMesh Areas — Farklı Yüzey Tipleri

Sahnede her yüzey aynı maliyetli değildir. Agent çamurdan değil de yoldan gitmeyi tercih etmeli; ya da sadece zırhlı düşmanlar suya girebilmeli. Bu, Area Type'lar ile yönetilir.

  1. Window → AI → Navigation (Areas sekmesi) — varsayılan üç alan var: Walkable (cost 1), Not Walkable, Jump.
  2. Yeni alan ekle: Water (cost 5), Road (cost 0.5), Mud (cost 3).
  3. İlgili mesh'lere NavMeshModifier bileşeni ekle, Override Area ☑ ve alan tipini seç (örn. Water).
  4. Tekrar bake et — NavMesh artık farklı renkte alanları gösterir.
  5. NavMeshAgent'ın Area Mask'inden hangi alanlara girebileceğini ayarla (örn. zırhsız düşman Water'ı uncheck eder, suya giremez).

Path-finding artık cost'a bakar: düz yolda yürümek 1 birimse, çamurda 3 birim — agent bir yol seçerken çamuru dolanmayı tercih eder eğer dolambaç < 3x kısa yol olursa.

20.11

⚡ Runtime Bake — Procedural Haritalar

AI Navigation paketinin en güçlü özelliği: oyun çalışırken NavMesh bake etmek. Procedural dungeon, oyuncunun yarattığı yapılar, runtime'da yüklenen seviyeler için kritik. 

using UnityEngine;
using Unity.AI.Navigation;

public class RuntimeNavMeshBaker : MonoBehaviour
{
    [SerializeField] private NavMeshSurface surface;

    public void RebakeAfterDungeonGen()
    {
        surface.BuildNavMesh();
        Debug.Log("NavMesh runtime'da bake edildi");
    }
}

Tek satır: surface.BuildNavMesh(). Procedural dungeon generator odaları kurar, bittikten sonra bu metod çağrılır, NavMesh oluşur ve düşman AI'sı çalışmaya başlar.

💡 Performans: Runtime bake bir frame'de bitmez (büyük haritada 1-3 sn alabilir). Yükleniyor ekranı arkasına alın ya da UpdateNavMeshAsync() ile async çağırın.
20.12

👥 Birden Fazla Agent Tipi

Bir oyunda insan-boy düşman ile dev-boy boss aynı NavMesh'i kullanamaz — dev geçemediği bir koridoru insan rahat geçebilir. Çözüm: her boy için ayrı agent type.

  1. Window → AI → Navigation → Agents sekmesi.
  2. "+" ile yeni agent type ekle: Giant (Radius 1.5, Height 4, Step Height 0.6, Max Slope 30).
  3. Sahnede iki ayrı NavMeshSurface oluştur, biri Humanoid, diğeri Giant Agent Type ile.
  4. İki surface'ı da bake et — iki ayrı NavMesh katmanı oluşur.
  5. İnsan düşmanlar Humanoid Agent Type'lı NavMeshAgent kullanır, dev boss Giant'lı.

Aynı sahneye birden fazla NavMesh "katmanı" eklenebilir. Bu sistem Hollow Knight'in arı düşmanları (uçan = farklı agent) ya da Demon's Souls'un Tower Knight (dev boss) gibi çeşitliliği destekler.

🎯 Pratik Projeler

Proje 1: Devriye + Kovalama AI'sı

Hedef: 4 waypoint arasında dolaşan bir nöbetçi yap; oyuncu görüş menziline girince koşarak takip etsin, oyuncu kaçınca devriyeye dönsün.

  1. Düz bir zemin, NavMeshSurface ile bake.
  2. 4 empty GameObject ile waypoint (W1, W2, W3, W4).
  3. Düşman: Capsule + NavMeshAgent (speed 3, angular 200, acceleration 8).
  4. Yukarıdaki EnemyAI scriptini ekle, waypoints'leri array'e bağla, player referansı ver.
  5. sightRange=10, attackRange=2 ayarla.
  6. Test: Düşman waypoint'leri dolaşıyor mu? Yaklaşınca koşuyor mu? Uzaklaşınca devriyeye dönüyor mu?

Proje 2: Köprüden Atlayan Düşman (NavMeshLink)

Hedef: İki ayrı platform arasında uçurum olsun; agent normalde uzun yoldan dolaşsın, ama NavMeshLink eklersen kısa yoldan zıplasın.

  1. İki ayrı platform yarat, aralarına uçurum koy, ama yan tarafta dolambaçlı bir yol yap.
  2. Tüm sahneyi NavMeshSurface ile bake et.
  3. Test: Agent uzun yoldan dolaşıyor mu?
  4. Şimdi iki platformun karşılıklı kenarlarına empty GameObject'ler koy, aralarına NavMeshLink ekle (Start ve End transform'larını ata).
  5. Tekrar Bake et, NavMeshLink artık aktif.
  6. Agent'a SetDestination ile karşı platformu ver — link'i kullanarak kestirme yapacak.
  7. Bonus: Cost Modifier'ı 5 yap, agent yine uzun yoldan gitmeyi tercih ediyor mu test et.

Proje 3: Su = Yasak Bölge (NavMesh Areas)

Hedef: Bir gölün ortasına ada koy; insan düşman göle giremesin (etrafından dolaşsın), ama özel "balık" düşman göl içinde rahatça yüzsün.

  1. Sahnede zemin + ada (üst plane) + ortada gölü temsil eden plane oluştur.
  2. Window → AI → Navigation → Areas'ta yeni alan: "Water" (cost 5).
  3. Göl plane'ine NavMeshModifier ekle, Override Area = Water.
  4. NavMeshSurface'ı bake et — göl bölgesi farklı renkle (varsayılan mavi) görünür.
  5. İnsan düşman: NavMeshAgent → Area Mask'ten "Water" kategorisini kaldır. Göle yaklaştığında etrafından dolaşır.
  6. Balık düşman: Area Mask'te tüm alanlar açık. Göl içine girer.
  7. Test: SetDestination ile her iki düşmana göl ortasındaki adayı ver. Davranışı gözle.
