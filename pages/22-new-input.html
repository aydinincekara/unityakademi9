Bölüm 22

New Input System — Modern Giriş Yönetimi

Action-based input, InputAction asset, Bindings, Control Schemes, PlayerInput component, Touch, Rebinding — klavye / mouse / gamepad / dokunmatik aynı kodla.

⏱ Okuma süresi: ~27 dk 🎮 Multi-platform input 🎯 Zorluk: Orta
22.1

🎮 Neden Yeni Bir Sistem?

Bölüm 7'de eski Input Manager'ı (Edit → Project Settings → Input) gördün. Input.GetAxis("Horizontal"), Input.GetKeyDown(KeyCode.Space) gibi çağrılar. Çalışıyor, evet — ama 2025 yılında ciddi sorunları var:

  • Cihaz farkındalığı yok: GetKey sadece klavye, gamepad için ayrı Input.GetJoystickNames(), mobilde Input.touches[i]. Her cihaz için ayrı kod.
  • Soft kalmış: 2010'da yazıldı, yeni gamepad'ler (PS5 DualSense, Switch Pro, Xbox Series), yeni input metodları (VR controller, dokunmatik) yarım destekli.
  • Yeniden bağlama (rebinding) zor: Kullanıcının "Jump'ı Space yerine W'ye al" demesi için kendi sistemini yazmak zorundasın.
  • Multiplayer zor: İki oyuncu yan yana, biri klavye biri gamepad — eski sistemde acı verici.
  • Polling tabanlı: Update'te sürekli "tuşa basıldı mı?" diye sormak gerekir; event tabanlı değil.

Unity, 2019'da New Input System paketini tanıttı. Action tabanlı, event-driven, cihaz-bağımsız, multiplayer-friendly. Unity 6'da artık varsayılan.

🎮 Gerçek Dünyadan: Cross-Platform Oyun Geliştirme

Hades, Hollow Knight, Stardew Valley, Among Us — PC + Switch + iOS + Android. Her platformda farklı input cihazı, ama aynı oyun mantığı. New Input System ile "Jump action tetiklendi" kodu tek seferlik yazılır; binding tarafında her cihaz için Space/A button/Touch tap eşleştirilir. Yeni bir cihaz desteklenince sadece binding eklenir, kod değişmez.

22.2

📦 Kurulum & Active Input Handling

  1. Window → Package Manager → Unity Registry → "Input System" → Install.
  2. Kurulduğunda Unity bir popup gösterir: "Active Input Handling değiştirilecek, restart gerekecek" → Apply.
  3. Restart sonrası kontrol: Edit → Project Settings → Player → Other Settings → Active Input Handling:
    • Input Manager (Old) — Sadece eski sistem.
    • Input System Package (New) — Sadece yeni sistem.
    • Both — İkisi birden (yeni başlayanlar için önerilen, eski Asset Store paketleriyle uyum).
📌 "Both" Seçeneği: Eski Standard Assets veya tutorial'larda Input.GetKey kullanılıyorsa Both seçili olmalı. Performansa minimum etkisi vardır. Yeni projeye sıfırdan başlıyorsan Input System Package daha temizdir.
22.3

📜 Input Action Asset — Eylemler Sözleşmesi

Yeni sistemin merkezi Input Action Asset'tir (.inputactions uzantılı). Bu asset, projedeki tüm giriş eylemlerini ve cihaz bağlantılarını içerir.

  1. Project window: sağ tık → Create → Input Actions. Adı "PlayerControls".
  2. Çift tık ile editör açılır. Üç sütun: Action Maps, Actions, Properties.

Action Maps — Bağlam Grupları

Bir Action Map, belirli bir bağlam'daki tüm input action'larıdır. Tipik oyunda:

  • Gameplay — Move, Jump, Fire, Look, Interact
  • UI — Navigate, Submit, Cancel, Point, Click
  • Menu — OpenInventory, Pause, Confirm

Sadece bir Action Map'i o anda aktif tutarsın. Pause açıldığında Gameplay deaktive, Menu aktive olur — bu sayede oyuncu menüdeyken yanlışlıkla Fire'a basamaz.

22.4

🎯 Action Tipleri

Her action'ın bir Action Type'ı vardır:

  • Button — Anlık tetik: Jump, Fire, Interact. Performed event ile tek seferlik.
  • Value — Sürekli değer: Move (Vector2), Throttle (float), Look (Vector2). Her frame okuyabilirsin.
  • Pass Through — Tüm değişiklikleri filtreleme yapmadan iletir. Birden çok input device aynı action'ı paylaşıyorsa kullanışlı.

Control Type — Beklenen Veri Tipi

  • Button — boolean (basıldı / basılmadı)
  • Axis — float, -1 ile 1 arası (gas pedalı, throttle)
  • Vector2 — yatay+dikey (Move, Look, Stick)
  • Vector3 — 3D yön (Accelerometer)
  • Stick — özelleştirilmiş Vector2 (dead zone destekli)
22.5

🔌 Bindings — Cihaz Eşleştirme

Bir action tanımlandıktan sonra ona Binding'ler eklersin: hangi gerçek tuş/buton/eksen bu action'ı tetikler?

Jump Action — 3 Cihaz Binding'i

  1. Gameplay map'ine sağ tık → Add Action → "Jump". Action Type: Button.
  2. Jump üzerine sağ tık → Add Binding. Path: Keyboard / Space.
  3. Tekrar Add Binding. Path: Gamepad / Button South (Xbox: A, PS: X, Switch: B).
  4. Tekrar Add Binding. Path: Touchscreen / Touch 0 / Tap.

Artık kod tarafında sadece "Jump performed" event'ini dinlersin; oyuncunun hangi cihazı kullandığı tamamen şeffaf.

2D Vector Composite — Move

Move action'ı için (WASD) 2D Vector Composite kullanılır:

  1. Move (Value, Vector2) action'ına sağ tık → Add Up/Down/Left/Right Composite → "2D Vector".
  2. Up: W, Down: S, Left: A, Right: D otomatik atanır (değiştirebilirsin).
  3. Aynı action'a alternatif binding: ok tuşları, gamepad sol stick, vs. eklenebilir.
  4. Mode: Digital (0/1), Digital Normalized (diagonal'da hız sabit), Analog (sürekli).
22.6

🎛 Control Schemes — Cihaz Grupları

Control Scheme, bir cihaz kombinasyonunu temsil eder: "Keyboard&Mouse" (klavye + mouse beraber), "Gamepad" (sadece gamepad), "Touch" (sadece dokunmatik).

  1. Action editöründe sol üst dropdown'da All Control Schemes → Add Control Scheme.
  2. Adı: "Keyboard&Mouse". Requirements: Keyboard, Mouse.
  3. Bir Control Scheme daha: "Gamepad". Requirements: Gamepad.
  4. Her binding üzerine tıkla → sağda Use in control scheme seç (sadece K&M veya sadece Gamepad).

Bu, multiplayer'da kritik: PlayerInput component oyuncunun cihazını algılar ve sadece o scheme'in binding'lerini kullanır. İki oyuncu, biri klavye diğeri gamepad, aynı sahnede.

22.7

💻 Kullanım — 3 Yöntem

Action'ları C# tarafında kullanmanın üç yolu var:

Yöntem 1: PlayerInput Component (En Kolay)

  1. Player GameObject'ine Add Component → PlayerInput.
  2. Actions: PlayerControls asset'ini sürükle.
  3. Default Map: Gameplay.
  4. Behavior:
    • Send Messages — OnJump(), OnMove() metodları otomatik çağrılır (eski SendMessage tarzı, refactor zor).
    • Broadcast Messages — Children dahil.
    • Invoke Unity Events — Inspector'da her action için event slot'u; UnityEvent ekleyip metod ata.
    • Invoke C# Events — Kodla playerInput.actions["Jump"].performed += OnJump;.
using UnityEngine;
using UnityEngine.InputSystem;

public class PlayerController : MonoBehaviour
{
    [SerializeField] private float moveSpeed = 5f;
    [SerializeField] private float jumpForce = 5f;
    private Rigidbody rb;
    private Vector2 moveInput;

    private void Awake() => rb = GetComponent<Rigidbody>();

    // Send Messages mode'da otomatik çağrılır
    public void OnMove(InputValue value)
    {
        moveInput = value.Get<Vector2>();
    }

    public void OnJump(InputValue value)
    {
        if (value.isPressed)
        {
            rb.AddForce(Vector3.up * jumpForce, ForceMode.Impulse);
        }
    }

    private void FixedUpdate()
    {
        Vector3 move = new(moveInput.x, 0, moveInput.y);
        rb.MovePosition(rb.position + move * moveSpeed * Time.fixedDeltaTime);
    }
}

Yöntem 2: Generated C# Class (En Profesyonel)

Action asset'in inspector'ında Generate C# Class ☑ → Apply. Unity otomatik bir C# sınıf oluşturur (örn. PlayerControls.cs). Bu sınıf strongly-typed action'lar sunar — IntelliSense ile auto-complete. 

using UnityEngine;
using UnityEngine.InputSystem;

public class PlayerController : MonoBehaviour
{
    private PlayerControls controls;
    private Vector2 moveInput;

    private void Awake()
    {
        controls = new PlayerControls();
        controls.Gameplay.Move.performed += ctx => moveInput = ctx.ReadValue<Vector2>();
        controls.Gameplay.Move.canceled += ctx => moveInput = Vector2.zero;
        controls.Gameplay.Jump.performed += OnJump;
    }

    private void OnEnable() => controls.Gameplay.Enable();
    private void OnDisable() => controls.Gameplay.Disable();

    private void OnJump(InputAction.CallbackContext ctx)
    {
        Debug.Log("Zıpla!");
    }
}

Yöntem 3: Doğrudan Action Reference

Action'a serialize field olarak referans ver, polling ile oku:

[SerializeField] private InputActionReference moveAction;

private void OnEnable() => moveAction.action.Enable();
private void OnDisable() => moveAction.action.Disable();

private void Update()
{
    Vector2 input = moveAction.action.ReadValue<Vector2>();
}

Bu yöntem PlayerInput'siz tek bir component'in input alması gerektiğinde temizdir. Önce action asset'i içindeki action'a "Reference" oluşturmalısın (asset inspector'ında, Generate ☐, Reference ☑).

22.8

🔔 Action Event'leri — started / performed / canceled

Her action 3 event yayınlar:

  • started — Action ilk tetiklenmeye başladığı an (örn. tuşa basıldı, ama henüz hold/tap belli değil).
  • performed — Action başarıyla "performed" edildi. Button için tıklama anı, Value için her değer değişimi.
  • canceled — Action sonlandı (tuş bırakıldı, stick merkeze döndü). Move için canceled'ta moveInput = Vector2.zero yapmak hayati.
controls.Gameplay.Fire.started   += ctx => Debug.Log("Tetik basıldı");
controls.Gameplay.Fire.performed += ctx => Debug.Log("Ateş!");
controls.Gameplay.Fire.canceled  += ctx => Debug.Log("Tetik bırakıldı");
22.9

⚙ Interactions & Processors — Davranış Ayarı

Interactions

Bir binding'in nasıl tetikleneceğini belirler. Action'a değil binding'e eklenir:

  • Tap — Hızlı basıp bırakma (varsayılan 0.5 sn altında).
  • Hold — Belirli süre basılı tutma (örn. "Hold E to Pick Up" 0.5 sn).
  • Multi Tap — Çift tıklama (Tap Count = 2). RPG'de "Sprint" için W+W gibi.
  • Slow Tap — Yavaş basma.
  • Press — Basıldığı an (default).

Processors

Binding'den gelen ham değeri işler:

  • Invert — Eksen tersine çevrilir (Y-axis flip seçenekleri için).
  • Scale — Değer çarpılır (sensitivity, mouse look için).
  • Normalize — Vector büyüklüğü 1'e sabitlenir.
  • Stick Deadzone — Stick'in merkezindeki ölü bölge (gamepad drift'ini engeller).
  • Clamp — Değer belli aralığa kısıtlanır.
22.10

🖱 UI Input Module

Yeni Input System UI ile birlikte çalışmak için EventSystem'i güncellemen gerek:

  1. Sahnedeki EventSystem GameObject'ini seç (Canvas oluşturduğunda otomatik gelir).
  2. Inspector'da Standalone Input Module komponenti varsa "Replace with InputSystemUIInputModule" tıkla — sistem otomatik dönüştürür.
  3. Yeni komponent Actions Asset alanına PlayerControls.asset'ini bağla.
  4. UI Action Map'inden Navigate / Submit / Cancel / Point / Click / ScrollWheel action'larını uygun action'lara bağla.

Artık gamepad ile menülerde dolaşma, UI Button'lara A ile tıklama otomatik çalışır.

22.11

🔄 Rebinding — Kullanıcı Kontrolleri Değiştirsin

Modern oyunlarda Settings menüsünde "Jump tuşunu değiştir" özelliği standart. New Input System bunu native destekler: 

using UnityEngine;
using UnityEngine.InputSystem;
using TMPro;

public class RebindButton : MonoBehaviour
{
    [SerializeField] private InputActionReference action;
    [SerializeField] private TMP_Text displayText;
    [SerializeField] private int bindingIndex = 0;

    private InputActionRebindingExtensions.RebindingOperation rebinding;

    public void StartRebind()
    {
        action.action.Disable();
        displayText.text = "Yeni tuşa bas...";

        rebinding = action.action.PerformInteractiveRebinding(bindingIndex)
            .WithControlsExcluding("Mouse")
            .OnMatchWaitForAnother(0.1f)
            .OnComplete(op => {
                op.Dispose();
                action.action.Enable();
                UpdateDisplay();
            })
            .Start();
    }

    private void UpdateDisplay()
    {
        displayText.text = action.action.GetBindingDisplayString(bindingIndex);
    }

    private void Start() => UpdateDisplay();
}

Bu script bir UI Button'a bağlı; tıklandığında bir sonraki tuşu yakalayıp action'a binding olarak atar. GetBindingDisplayString() binding'in kullanıcı-okuyabilir adını verir ("Space", "Gamepad ButtonSouth (X)", vs.).

Kaydetme — PlayerPrefs

Rebinding'lerin kalıcı olması için: 

// Kaydet
string rebinds = controls.asset.SaveBindingOverridesAsJson();
PlayerPrefs.SetString("rebinds", rebinds);

// Yükle (Awake'te)
if (PlayerPrefs.HasKey("rebinds"))
{
    controls.asset.LoadBindingOverridesFromJson(PlayerPrefs.GetString("rebinds"));
}
22.12

📱 Dokunmatik (Touch) Input

Mobil için Enhanced Touch API var. Çoklu dokunma, basınç, swipe gesture'larını destekler.

  1. Sahneye TouchSimulator ekle (mobil cihaz olmadan editor'de mouse'u touch gibi simüle eder): EnhancedTouchSupport.Enable(); Awake'te.
  2. Action map'inde Touch Control Scheme oluştur, requirement: Touchscreen.
  3. Move için iki touch binding: Touch 0 / Position (sol joystick), Touch 1 / Position (sağ joystick) — virtual joystick için.
using UnityEngine;
using UnityEngine.InputSystem;
using UnityEngine.InputSystem.EnhancedTouch;

public class TouchTracker : MonoBehaviour
{
    private void OnEnable() => EnhancedTouchSupport.Enable();
    private void OnDisable() => EnhancedTouchSupport.Disable();

    private void Update()
    {
        foreach (var touch in Touch.activeTouches)
        {
            if (touch.phase == TouchPhase.Began)
                Debug.Log($"Yeni touch: {touch.screenPosition}");
        }
    }
}
22.13

👥 Local Multiplayer — Birden Fazla Oyuncu

Tek ekran iki oyuncu? PlayerInputManager bileşeniyle çok kolay:

  1. Empty GameObject "InputManager" + Add Component → PlayerInputManager.
  2. Player Prefab alanına PlayerInput component'li player prefab'ı sürükle.
  3. Joining: Join Players When Button Is Pressed (oyuncu butona basınca katılır).
  4. Max Player Count: 2 (veya 4).

Oyun başladığında: oyuncu 1 klavyede WASD'a basar → otomatik player prefab spawn'lanır, klavye/mouse ona atanır. Oyuncu 2 gamepad'da A'ya basar → ikinci prefab spawn'lanır, gamepad ona atanır. Her birinin kendi PlayerInput'u olduğundan input'lar karışmaz.

22.14

🎮 Gamepad Rumble & DualSense Özellikleri U6

New Input System gamepad titreşimini (rumble) destekler. DualSense (PS5) için adaptive trigger ve haptic feedback de kısıtlı destekle gelir. 

using UnityEngine.InputSystem;

// Rumble başlat (low frequency, high frequency)
Gamepad.current?.SetMotorSpeeds(0.5f, 0.8f);

// 0.3 saniye sonra durdur
Invoke(nameof(StopRumble), 0.3f);

private void StopRumble()
{
    Gamepad.current?.SetMotorSpeeds(0f, 0f);
}
🎬 Bölüm Kontrol Noktası: Input Action Asset = projedeki tüm input sözleşmesi. Action Maps bağlam grupları (Gameplay, UI, Menu). Action Type: Button / Value / Pass Through. Binding cihaz eşleştirir. Control Scheme cihaz kombinasyonu (K&M, Gamepad). 3 kullanım: PlayerInput (kolay), Generated C# (profesyonel), InputActionReference (özel). Rebinding native destekli, PlayerPrefs ile kalıcı.

🎯 Pratik Projeler

Proje 1: Cross-Platform Karakter Kontrolü

Hedef: Aynı kodla klavye (WASD + Space) ve gamepad (sol stick + A) ile çalışan 3D karakter hareketi.

  1. Input System paketini kur.
  2. PlayerControls.inputactions oluştur. Gameplay map'inde:
    • Move (Value, Vector2) — 2D Vector Composite (WASD) + Gamepad Left Stick.
    • Jump (Button) — Space + Gamepad Button South.
    • Look (Value, Vector2) — Mouse Delta + Gamepad Right Stick.
  3. Player GameObject (Capsule + Rigidbody + PlayerInput).
  4. PlayerInput'ta Actions: PlayerControls, Behavior: Invoke C# Events.
  5. PlayerController script yaz (yukarıdaki Yöntem 2 örneği).
  6. Klavye ve gamepad ile test et — aynı karakter, iki cihaz.

Proje 2: Rebinding Menüsü

Hedef: Settings panelinde 3 action için (Move Up, Jump, Fire) "Change Key" butonları, kullanıcı tıklayıp yeni tuşa basınca binding güncellensin.

  1. Canvas → Panel → Settings layout. Her action için Label + Button + Display Text.
  2. Yukarıdaki RebindButton script'ini her butona ekle, action ve binding index ata.
  3. "Save" butonu: SaveBindingOverridesAsJson + PlayerPrefs.
  4. "Reset to Defaults" butonu: action.action.RemoveAllBindingOverrides();
  5. Awake'te yüklenen PlayerPrefs'i geri uygula.

Proje 3: 2 Player Couch Co-op

Hedef: Tek ekranda iki oyuncu, biri klavye biri gamepad, ayrı karakterleri kontrol etsin.

  1. Action asset'inde Control Scheme'leri tanımla: Keyboard&Mouse, Gamepad.
  2. Player Prefab: PlayerInput (Actions: PlayerControls, Default Scheme: yok), PlayerController script.
  3. Sahnede InputManager GameObject + PlayerInputManager. Player Prefab'ı ata. Joining: Join Players When Button Is Pressed.
  4. Karakter spawn pozisyonlarını PlayerInputManager.OnPlayerJoined event'inden ayarla.
  5. Oyuncu 1 klavye Space'e bas → katılır. Oyuncu 2 gamepad A'ya bas → katılır.
  6. Split-screen kamera bonus: her PlayerInput'un kendi Camera'sı, viewport ayarları (0, 0, 0.5, 1) ve (0.5, 0, 0.5, 1).
