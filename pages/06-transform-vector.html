Transform & Vector3 — Uzayda Nesnelerin Dili

Her GameObject'in vazgeçilmez Component'i: Transform. Pozisyon, döndürme, ölçek — üç vektör ve bir hiyerarşi. Bu bölümde Unity'nin uzay anlayışını, Vector3'ün matematiğini, World space ile Local space arasındaki ince farkı, ve script'ten Transform'a dokunmanın tüm yollarını öğreneceksin.

⏱ 35 dakika 🎯 Pozisyon/Rotasyon/Ölçek 📐 Vector3 matematiği
6.1

Transform — Her GameObject'in Anayasası

Unity'de bir GameObject sahneye düştüğü an üç bilgiyi taşır: nerede, hangi açıda, ne kadar büyük. Bu üç bilgiyi tutan tek Component Transform'dur. Inspector'ı açtığında en üstte gördüğün, silinemeyen, kapatılamayan o ilk satır.

Bir GameObject'ten her Component'i tek tek söküp atabilirsin — Rigidbody'yi, Collider'ı, MeshRenderer'ı, hatta script'lerini. Ama Transform sökülmez. Çünkü Transform yoksa "nesne nerede?" sorusunun cevabı da yoktur. Transform, GameObject'in varoluş koşuludur.

↔
Transform
Position X 2.5Y 0Z -1.3
Rotation X 0Y 45Z 0
Scale X 1Y 1Z 1

Yukarıdaki Inspector görüntüsünde Transform'un üç satırını görüyorsun. Her satır bir Vector3 — üç sayıdan oluşan bir veri yapısı. Position'da bu üç sayı metre cinsinden konumu, Rotation'da derece cinsinden açıyı, Scale'de oransal büyüklüğü temsil eder. Üçü de aynı yapıya yaslandığı için Unity'de uzay matematiğinin dili tek bir tip üzerinden konuşulur: Vector3.

💡 Component zorunluluğu

Unity Editor'ünde bir GameObject oluşturduğunda — örneğin sağ tık → Create Empty — sahnedeki yeni nesneye otomatik olarak yalnızca Transform eklenir. Cube, Sphere gibi 3D primitive'ler için ek olarak MeshFilter, MeshRenderer, Collider gelir; ama Transform her zaman varsayılan ve tektir. Kod tarafında erişimi de aynı kolaylıkta: her MonoBehaviour'da transform özelliği hazır gelir (büyük harf yok; küçük "t" ile).

6.2

Vector3 — Üç Sayının Hikayesi

Vector3, Unity'nin uzay matematiğinde temel taşıdır. Üç ondalık sayıdan (float) oluşur: X, Y ve Z. Bu üç sayı bağlama göre farklı anlam taşır — pozisyon için "noktanın koordinatları", yön için "ok ucunun göstereceği taraf", hız için "saniyedeki hareket miktarı". Tip aynı kalır, anlam bağlamla değişir.

📄 Vector3Examples.cs 
using UnityEngine;

public class Vector3Examples : MonoBehaviour
{
    void Start()
    {
        // Yeni bir Vector3 oluşturmanın yolları
        Vector3 nokta = new Vector3(2f, 0f, -1f);
        Vector3 sifir = Vector3.zero;        // (0, 0, 0)
        Vector3 bir   = Vector3.one;         // (1, 1, 1)
        Vector3 ileri = Vector3.forward;     // (0, 0, 1)
        Vector3 yukari = Vector3.up;         // (0, 1, 0)
        Vector3 sag   = Vector3.right;       // (1, 0, 0)

        // Tek bileşene erişim ve değiştirme
        Debug.Log(nokta.x);   // 2
        Debug.Log(nokta.y);   // 0
        Debug.Log(nokta.z);   // -1

        // Vector3'ler arasında toplama, çıkarma, ölçeklendirme
        Vector3 toplam = nokta + ileri;     // (2, 0, 0)
        Vector3 iki_kat = nokta * 2f;       // (4, 0, -2)
        Vector3 fark = nokta - sag;          // (1, 0, -1)
    }
}

Yukarıdaki kod parçasında Vector3'ün dilbilgisini görüyorsun: oluşturma, sabitler, bileşen erişimi, aritmetik. Vector3.zero, Vector3.one, Vector3.forward gibi statik sabitler her gün kullanacağın kısayollar — kendin new Vector3(0, 0, 1) yazmak yerine Vector3.forward demek hem okunaklıdır hem de tutarlıdır.

Vector3 Sabitleri — Yön Tabelası

Vector3.right

+X ekseni — sağ
Vector3.up
(0, 1, 0)
+Y ekseni — yukarı
Vector3.forward
(0, 0, 1)
+Z ekseni — ileri
Vector3.left
(-1, 0, 0)
−X ekseni — sol
Vector3.down
(0, -1, 0)

Vector3.back
(0, 0, -1)

⚠ Unity'nin koordinat sistemi sol-eldir (left-handed)

Unity, OpenGL/Blender'dan farklı olarak left-handed bir koordinat sistemi kullanır. Pratik etkisi: +Z senin için "ileri", +Y "yukarı", +X "sağ"dır. Eğer Blender'dan model alıyorsan Z ile Y eksenleri yer değiştirebilir — bu çoğu zaman import sırasında otomatik çözülür ama bilmek hata ayıklamayı kolaylaştırır.

6.3

Position — Nesneyi Hareket Ettir

Bir nesnenin sahnedeki konumunu değiştirmek istediğinde iki temel yol var: doğrudan atama ve Translate ile artırma. İkisi de doğru sonucu üretir ama semantikleri farklıdır. Doğrudan atama "git şu noktaya" demektir; Translate "şu kadar yer değiştir" demektir.

📄 MoveExamples.cs 
using UnityEngine;

public class MoveExamples : MonoBehaviour
{
    void Update()
    {
        // 1) Doğrudan atama — "bu konuma git"
        transform.position = new Vector3(0f, 1f, 5f);

        // 2) Mevcut konuma ekleyerek hareket — "ileri kay"
        transform.position += Vector3.forward * Time.deltaTime;

        // 3) Translate — yukarıdakinin yerel uzaydaki kısayolu
        transform.Translate(Vector3.forward * Time.deltaTime);

        // 4) Sadece bir bileşeni değiştir
        Vector3 p = transform.position;
        p.y = 3f;
        transform.position = p;
    }
}

Dördüncü örnekteki kalıbı dikkatle incele: transform.position.y = 3f; yazmak derleme hatası verir. Çünkü transform.position bir struct döndürür ve struct'lar değer-tipi olduğu için doğrudan üzerinde değişiklik yapamazsın. Önce bir değişkene kopyalar, değiştirir, sonra geri atarsın. Bu Unity'de en sık karşılaşılan "neden çalışmıyor?" tuzaklarından biridir.

💡 Time.deltaTime — kare-bağımsız hareket

Time.deltaTime, son kareyle bu kare arasında geçen süreyi saniye cinsinden verir. 60 FPS oynayan birinde değer ~0.0166, 30 FPS oynayan birinde ~0.0333 olur. Update içinde hareketi her zaman Time.deltaTime ile çarp; aksi halde nesnenin hızı oynayanın bilgisayarına göre değişir — 120Hz monitördeki oyuncu, 60Hz monitördeki oyuncudan iki kat hızlı koşar.

6.4

Rotation — Euler ve Quaternion

Döndürme, Unity'de iki farklı temsille hayat bulur. Inspector'da gördüğün üç sayı — X, Y, Z derece cinsinden — Euler açılarıdır. İnsan için sezgisel ama matematiksel olarak sorunlu. Motor içeride Quaternion kullanır; dört sayılı, daha karmaşık ama "gimbal lock" denen döndürme tıkanmasından muaf.

Inspector (Euler) — İnsan Dili
  • X, Y, Z her biri derece cinsinden (0–360)
  • Sezgisel: "90° döndür" denilince zihinde canlandırması kolay
  • Inspector ekranında doğrudan yazılıp değiştirilebilir
  • İki Euler açısının toplamı her zaman beklenen sonucu vermez (gimbal lock)
Motor İçi (Quaternion) — Matematik
  • x, y, z, w dört bileşen
  • Karmaşık görünür ama interpolasyon (Slerp) için ideal
  • Gimbal lock üretmez — her döndürme tek ve geçerlidir
  • Quaternion.Euler(0,90,0) ile Euler'den dönüştürülür
📄 RotationExamples.cs 
using UnityEngine;

public class RotationExamples : MonoBehaviour
{
    void Update()
    {
        // Euler ile — "Y ekseninde 90 derece dönmüş ol"
        transform.eulerAngles = new Vector3(0f, 90f, 0f);

        // Quaternion ile — aynı sonuç, motor diliyle
        transform.rotation = Quaternion.Euler(0f, 90f, 0f);

        // Rotate — saniyede 90 derece (Y ekseni etrafında)
        transform.Rotate(0f, 90f * Time.deltaTime, 0f);

        // LookAt — başka bir nesneye bakacak şekilde dön
        Transform hedef = GameObject.Find("Player").transform;
        transform.LookAt(hedef);

        // Slerp — yumuşak (interpolated) dönüş
        Quaternion hedefAci = Quaternion.Euler(0f, 180f, 0f);
        transform.rotation = Quaternion.Slerp(
            transform.rotation,
            hedefAci,
            Time.deltaTime * 2f
        );
    }
}

Slerp (Spherical Linear Interpolation) Unity'de yumuşak dönüşler için yıldız fonksiyondur. Üçüncü parametre 0 ile 1 arasında bir oran beklemez — her kare çağrıldığında "şu kadar yaklaş" der. Time.deltaTime * 2f verirsen kare-bağımsız bir dönüş hızı elde edersin: ne kadar yüksek FPS olursa olsun hedefe ulaşma süresi aynı kalır.

6.5

Scale — Boyutlandırma ve Mirror Etkisi

Ölçek, nesnenin orijinal boyutuna göre çarpan değer kullanır. Vector3.one yani (1,1,1) "olduğu gibi"; (2,2,2) "iki katı"; (0.5f, 1f, 1f) "sadece X'te yarısı". Üç eksende farklı değerler verirsen non-uniform scaling yaparsın — fiziği ve normal map'leri bozabilir; eğer model orijinal olarak orantılıyken sahnede ezilmesini istemiyorsan üç ekseni de aynı tutmak en güvenlisidir.

📄 ScaleExamples.cs 
using UnityEngine;

public class ScaleExamples : MonoBehaviour
{
    void Start()
    {
        // Iki katı büyüt
        transform.localScale = Vector3.one * 2f;

        // Sadece yüksekliği değiştir — yere yapışık kal
        transform.localScale = new Vector3(1f, 0.5f, 1f);

        // Negatif scale ile ayna efekti — modelin tersini bas
        transform.localScale = new Vector3(-1f, 1f, 1f);
    }
}
⚠ Negatif scale yan etkileri

Bir eksende negatif ölçek (örn. X = -1) modeli aynalar ama aynı zamanda yüzey normal'lerini ters çevirir; ışığın gördüğü yüzler farklı yorumlanır. Bazı shader'lar (özellikle cull face ayarlı olanlar) modeli içten görmeye başlar. Karakter "yön değiştirme" için negatif scale sık kullanılır ama 3D'de daima animasyonla veya 180° dönüşle yapmak daha güvenlidir; negatif scale 2D sprite'larda sorunsuzdur.

⚡ Mini-Checkpoint — Transform Refleksi

Bir sonraki bölüme geçmeden önce şu beş kavramı zihninde yerine oturt:

  1. transform her MonoBehaviour'da hazır gelir — yazılışı küçük "t" ile.
  2. Vector3 üç float'lı bir struct — pozisyon, yön, hız, ölçek hepsinin dili.
  3. transform.position.y = 3f yazmak hata verir — önce kopyala, değiştir, geri ata.
  4. Inspector'daki Rotation Euler; motor içinde Quaternion tutulur.
  5. Update içinde her hareketi Time.deltaTime ile çarp — kare-bağımsız hareket için.
6.6

Vector3 İşlemleri — Hesabın Mutfağı

Vector3 sadece veri taşımaz, üzerinde matematik de yapar. İki nokta arasındaki mesafe, iki yön arasındaki açı, bir hedefe doğru yumuşak hareket — hepsi Vector3 sınıfının statik metotlarıyla çözülür.

📄 VectorMath.cs 
using UnityEngine;

public class VectorMath : MonoBehaviour
{
    public Transform hedef;
    public float hiz = 3f;

    void Update()
    {
        // İki nokta arası mesafe (metre)
        float mesafe = Vector3.Distance(transform.position, hedef.position);

        // Lerp — A'dan B'ye yumuşak geçiş (her karede biraz yaklaş)
        transform.position = Vector3.Lerp(
            transform.position,
            hedef.position,
            Time.deltaTime * 2f
        );

        // MoveTowards — sabit hızda yaklaş, hedefte dur
        transform.position = Vector3.MoveTowards(
            transform.position,
            hedef.position,
            hiz * Time.deltaTime
        );

        // Yön vektörü — A'dan B'ye doğru bir Vector3 üret
        Vector3 yon = (hedef.position - transform.position).normalized;

        // Magnitude — bir vektörün uzunluğu
        float uzunluk = (hedef.position - transform.position).magnitude;
    }
}

Lerp ve MoveTowards Farkı

Vector3.Lerp ile Vector3.MoveTowards sık karıştırılır. İkisinin de işi "A'dan B'ye yumuşak geçiş" gibi görünür ama davranışları farklıdır:

Vector3.Lerp
  • Her karede mesafenin oranını alır — başta hızlı, sonda yavaş
  • Hedefe matematiksel olarak hiç ulaşmaz; sonsuza dek yaklaşır
  • "Yumuşak takip" veya "ease-out" görünümü için ideal
  • Üçüncü parametre 0–1 arası bir oran beklenir; Time.deltaTime * X ile kullanılır
Vector3.MoveTowards
  • Sabit hızda mesafe kadar yer değiştirir — başta da sonda da aynı hız
  • Hedefe ulaşınca durur; "tam değer" garantili
  • Düşman AI, sabit yürüyen NPC için ideal
  • Üçüncü parametre metre/saniye türünden hızdır
💡 Normalize ne demek?

Bir vektörü normalize etmek, yönünü koruyarak uzunluğunu 1 yapmak demektir. (3, 0, 4) uzunluğu 5 olan bir vektördür; normalize edilince (0.6, 0, 0.8) olur — aynı yönü gösterir ama "birim vektör"dür. Yön hesaplarında neredeyse her zaman normalize edilmiş vektörlerle çalışırsın çünkü yönü uzunlukla karıştırmamak istersin.

6.7

World Space vs Local Space — İki Uzay

Unity her Transform'u iki uzayda birden tutar: World Space (sahne kökeni referans) ve Local Space (parent referans). Bir GameObject'in başka bir GameObject'in çocuğu olduğu durumda bu iki uzay birbirinden ayrılır ve hangisinde çalıştığını bilmek hata ayıklamanın yarısıdır.

position
World Space — sahnenin (0,0,0)'ına göre konum
localPosition
Local Space — parent'a göre konum
rotation
World Space — mutlak Quaternion açı
localRotation
Local Space — parent'a göre Quaternion
lossyScale
World Space — hiyerarşi çarpılmış ölçek (read-only)
localScale
Local Space — parent'a göre ölçek

Sahnede tek başına duran bir nesnede world ve local aynıdır. Ama bir araba modelini yaparken "araba" parent, "tekerlek" child olduğunda iş değişir: tekerleğin localPosition'u arabanın merkezine göre (1.2, 0, 0.8) olabilir; world position ise arabanın nerede durduğuna bağlıdır. Araba 10 metre ileri kaydığında tekerleğin localPosition'u değişmez ama world position'u 10 metre artar.

📄 SpaceExamples.cs 
using UnityEngine;

public class SpaceExamples : MonoBehaviour
{
    void Start()
    {
        // Parent'a göre yarım metre yukarı
        transform.localPosition = new Vector3(0f, 0.5f, 0f);

        // Sahnede mutlak konum (parent'ı yok say)
        transform.position = new Vector3(5f, 0f, 0f);

        // Translate — varsayılan olarak Local Space'te çalışır
        transform.Translate(Vector3.forward * 2f);
        // "Nesnenin kendi ileri yönünde 2 metre" anlamına gelir

        // Translate — World Space modunu açıkça belirt
        transform.Translate(Vector3.forward * 2f, Space.World);
        // "Sahne ekseninde Z+ yönünde 2 metre"
    }
}
⚠ Yaygın hata — Translate ile yön karışıklığı

transform.Translate(Vector3.forward * 2f) genelde "ileri 2 metre git" olarak okunur ama gerçekte "kendi ileri yönünde 2 metre" demektir. Nesne 90° dönmüşse, "ileri"si artık world Z değil, kendi local Z'sidir. Eğer her zaman sahnede +Z yönünde hareket istiyorsan Space.World parametresini eklemelisin.

6.8

Pratik Kalıp — Transform Aramaları

Sahnedeki başka bir nesneyi script'inden bulup Transform'una erişmek için birkaç yol var. Her birinin maliyeti ve doğru kullanım yeri farklıdır.

📄 TransformFind.cs 
using UnityEngine;

public class TransformFind : MonoBehaviour
{
    public Transform oyuncuRef;   // Inspector'dan sürükle-bırak — EN İYİSİ

    void Start()
    {
        // 1) Inspector referansı — sıfır maliyet, en güvenli yol
        Vector3 p1 = oyuncuRef.position;

        // 2) Find — isimle ara, YAVAŞ, sahnenin tamamını tarar
        Transform bulunan = GameObject.Find("Player").transform;

        // 3) FindWithTag — Tag'le ara, Find'den hızlı ama yine de yavaş
        Transform tagli = GameObject.FindWithTag("Player").transform;

        // 4) Child arama — parent'in çocuğunu isimle veya indeksle
        Transform cocuk = transform.Find("Tekerlek_Sol");
        Transform ilkCocuk = transform.GetChild(0);

        // 5) Parent'a erişim
        Transform ust = transform.parent;
    }
}

Genel kural: Inspector referansı her şeyden iyidir. public Transform x; yazıp Inspector'dan sürükleyip bıraktığında Unity sahneyi taramaz, bağlantı doğrudandır. Find ve FindWithTag kolay ama her çağrıda sahne hiyerarşisinde gezinir; Update içinde çağırılırsa performans yer. Bu metotları sadece Awake/Start'ta bir kere çağırıp sonucu bir değişkende saklamak iyi bir alışkanlıktır.

🎮 Gerçek hayattan örnek — Bir karakter kontrolcüsünün ilk satırları

Üçüncü-şahıs bir oyun yaptığını düşün. Karakterin yapacağı ilk üç şey: ileri/geri hareket, sola/sağa dönme, kameranın baktığı yöne göre yön ayarlama. Üçü de Transform üzerinden yapılır: 

using UnityEngine;

public class SimpleCharacter : MonoBehaviour
{
    public float hiz = 5f;
    public float donusHizi = 120f;

    void Update()
    {
        float v = Input.GetAxis("Vertical");     // W/S, ileri-geri
        float h = Input.GetAxis("Horizontal");   // A/D, sol-sağ

        // Kendi ileri yönünde hareket — Local Space
        transform.Translate(Vector3.forward * v * hiz * Time.deltaTime);

        // Y ekseni etrafında dön — A/D ile
        transform.Rotate(0f, h * donusHizi * Time.deltaTime, 0f);
    }
}

Neden çalışır? Karakter kendi ileri yönünde (local Z+) hareket eder; A/D ile Y etrafında döndüğünde "ileri"si değişir; bir sonraki karede yeni ileri yönünde hareket eder. Translate'in varsayılan Local Space davranışı bu basitliği mümkün kılar — eğer Space.World deseydin karakter dönse bile hep aynı yönde giderdi.

Pratik — Üç Mini Proje

🎯 Proje 1 — Sallanan Lamba

Sahneye bir Cube (Lamba) ekle. Cube'a bir script bağla. Script her karede Lamba'yı Mathf.Sin(Time.time) kullanarak Z ekseni etrafında ±15 derece arasında salla.

Kontrol noktası: Play'e bastığında Cube saat sarkacı gibi gidip gelmeli. İpucu: transform.eulerAngles = new Vector3(0, 0, Mathf.Sin(Time.time) * 15f);
🎯 Proje 2 — Hedefe Yaklaşan Top

Sahnede iki Sphere oluştur: Top ve Hedef. Top'a bağladığın script'te public Transform hedef; alanını Inspector'dan ata. Top, Update içinde Vector3.MoveTowards ile saniyede 2 metre hızla Hedef'e yaklaşsın.

Kontrol noktası: Top, Hedef'e ulaşınca durmalı. Mesafe sıfıra inince Debug.Log("Vardım!") yazsın.
🎯 Proje 3 — Kendi Etrafında Dönen Gezegen

Boş bir GameObject oluştur, adını Gunes koy. Onun çocuğu olarak bir Sphere ekle ve localPosition'unu (3, 0, 0) yap (Gunes'in 3 metre sağına yerleşsin). Gunes'e bir script bağla; Update içinde transform.Rotate(0, 30 * Time.deltaTime, 0) çağır. Sphere otomatik olarak Gunes'in etrafında dönecek.

Kontrol noktası: Sphere, Gunes'in etrafında saniyede 30° hızla yörünge çizmeli. Bonus: ikinci bir Sphere ekleyip onu 5 metre uzağa koy, hızını değiştir — basit bir güneş sistemi çıksın.
