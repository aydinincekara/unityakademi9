Bölüm 21

URP, HDRP & Unity 6 — Modern Render Boru Hatları

Built-in, URP, HDRP karşılaştırması, Render Graph, GPU Resident Drawer, Volume sistemi, Render Pipeline Asset, Forward+ vs Deferred — Unity 6'nın grafik çekirdeği.

⏱ Okuma süresi: ~30 dk 🎨 Pipeline karşılaştırma 🎯 Zorluk: İleri
21.1

🎨 Render Pipeline Nedir?

Bir kare resimde gördüğün her piksel, GPU'da çalışan onlarca adımdan geçer: kameranın frustum'unda hangi mesh'ler var, hangi ışıklar onları aydınlatıyor, gölgeleri kim yapıyor, post-processing efektleri (bloom, motion blur, color grading) hangi sırada uygulanıyor... Bu adımların sıralanışı ve organizasyonuna Render Pipeline denir.

Unity uzun yıllar Built-in Render Pipeline (BRP) ile geldi — kodu motorun içine gömülmüş, esnekliği sınırlı bir sistem. 2018'de Unity, kullanıcının kendi render adımlarını C# ile yazabilmesi için Scriptable Render Pipeline (SRP) API'sini açtı. Bu API üzerine iki resmi pipeline kuruldu: Universal Render Pipeline (URP) ve High Definition Render Pipeline (HDRP).

Unity 6 ile birlikte URP ve HDRP Render Graph mimarisine taşındı — performans ve esneklik açısından devasa bir sıçrama. URP artık 2024'ten sonraki yeni projeler için Unity'nin önerdiği varsayılandır. Built-in artık "legacy" (eski) olarak işaretlenmiştir.

🎮 Gerçek Dünyadan: Genshin Impact vs Call of Duty: Mobile

Genshin Impact, Unity ile URP üzerine kuruludur — anime tarzı kalem-mürekkep shading, mobil + PC + PS uyumluluğu için URP'nin esnek ama performanslı yapısı seçildi. Call of Duty: Mobile ise yine Unity'de ama daha gerçekçi rendering için yer yer HDRP özelliklerini özelleştirilmiş bir URP kurgusuyla kullanır. Subnautica: Below Zero URP ile, Unity Demo "Enemies" HDRP ile yapıldı. Pipeline seçimi oyunun hedefini belirler.

21.2

🆚 Üç Pipeline — Hangisini Seçmeli?

🟫 Built-in (Legacy)

  • Eski Unity projeleri için
  • Tüm platformlarda çalışır
  • SRP'den önce yazılmış asset'lerle uyumlu
  • Shader: Surface Shader / Vertex-Fragment
  • Esneklik düşük, değişiklik için C++ gerekir
  • Yeni projelerde önerilmez

🟢 URP — Universal

  • Tüm platformlar — mobil, konsol, PC, VR, WebGL
  • Forward+ Rendering (Unity 6'da)
  • Shader Graph desteği
  • Volume Profile post-process
  • Render Graph mimarisi (U6)
  • Yeni projeler için önerilen

🔴 HDRP — High Definition

  • Sadece yüksek-end PC + konsol
  • Fiziksel doğru ışıklandırma
  • Ray tracing desteği
  • Volumetric fog, sub-surface scattering, screen-space refraction
  • Decal Layers, Light Layers
  • Cinematic ve AAA projeleri için
🤔 Karar Kılavuzu:
  • Mobil oyun mu? → URP
  • Stilize/cel-shaded oyun? → URP
  • Cross-platform indie? → URP
  • VR oyun? → URP (HDRP VR'da performans yetersiz)
  • AAA gerçekçi PC/konsol? → HDRP
  • Cinematic/sinematik? → HDRP
  • Eski projeyi güncellemek? → BRP'de kal, geçişi planla
21.3

📦 URP Kurulumu — Sıfırdan

Unity 6 ile 3D (URP) şablonu seçtiğinde URP zaten kuruludur. Eğer Built-in bir projeyi URP'ye taşıyorsan:

  1. Window → Package Manager → Unity Registry → Universal RP → Install.
  2. Assets → Create → Rendering → URP Asset (with Universal Renderer).
    • İki dosya oluşur: NewUniversalRenderPipelineAsset.asset (pipeline config) ve NewUniversalRenderPipelineAsset_Renderer.asset (renderer config).
  3. Edit → Project Settings → Graphics → "Scriptable Render Pipeline Settings" alanına az önce oluşturduğun pipeline asset'i sürükle.
  4. Edit → Project Settings → Quality → her quality level (Low, Medium, High) için de aynı pipeline asset'i ata.
  5. Sahnedeki Built-in shader'lı materyaller pembe olur (URP bunları okuyamıyor). Edit → Rendering → Materials → Convert All Built-in Materials to URP ile otomatik dönüştür.
⚠️ Geri Dönüşü Yok: Bir projeyi URP'ye taşıdıktan sonra geri Built-in'e dönmek hem materyalleri hem pipeline-spesifik shader'ları geri çevirmeyi gerektirir — yedeksiz girme. Git/Plastic SCM ile snapshot al, sonra geçiş yap.

⚙️ URP Pipeline Asset — Ayar Merkezi

URP Asset'i seçtiğinde Inspector tüm pipeline davranışını gösterir. Önemli alanlar:

URP Pipeline Asset
Render Scale1.0 (0.5-2.0)
Anti Aliasing (MSAA)2x / 4x / 8x
HDR☑ True
Main LightPer Pixel
Cast Shadows☑ True
Main Shadow Resolution2048
Additional LightsPer Pixel
Per Object Limit8
Cast Shadows (Additional)☑ True
Shadow Distance50
Cascade Count4
Post Processing☑ Enabled
  • Render Scale — 0.7 yaparsan render çözünürlüğü %70 olur (sonra ekrana up-scaled). FPS arttırmak için kritik mobil ayarı.
  • MSAA — Kenar yumuşatma. 4x mobil için iyi denge, 8x kalite ama %30 daha pahalı.
  • HDR — High Dynamic Range. Bloom efekti için zorunlu.
  • Additional Lights / Per Object Limit — Bir mesh'i kaç ışık aydınlatabilir? Mobil'de 4, PC'de 8. Daha yüksek = daha pahalı.
  • Shadow Distance — Kameradan kaç metreye kadar gölge hesaplansın. 50 m makul orta seviye.
  • Cascade Count — Directional light gölgesi için detay katmanları (1, 2, 4). Mobilde 2, PC'de 4.
21.5

⚡ Forward+ Rendering Unity 6 YENİ

Klasik Forward Rendering: Her mesh çizildiğinde sahnedeki tüm ışıklar tek tek hesaplanır. Çok ışıklı sahnelerde mesh sayısı × ışık sayısı kadar shader çalıştırması → kuadratik patlama.

Deferred Rendering: Önce sahnenin tüm yüzey bilgileri (G-buffer) toplanır, sonra ışıklar tek seferde uygulanır. Çok ışıkta verimli ama bellek pahalı, MSAA ile çalışmıyor, transparent objeler için ek geçiş gerekir.

Forward+ (Unity 6): Ekran ızgaralara bölünür (örn. 16×16 piksel hücre); her hücre için "burayı etkileyen ışıklar" listesi GPU'da hazırlanır. Her mesh çizilirken sadece kendi hücresindeki ışıklara bakar. Sonuç:

  • Çok sayıda ışıkta yüksek performans (Forward'tan 5-10x hızlı)
  • Forward gibi MSAA destekler
  • Deferred'ın bellek yükü yok
  • Transparent objeler için ekstra geçiş yok
  • Per-object 8 ışık limiti kalktı — sahnede yüzlerce ışık olabilir

URP Renderer asset'inde Rendering Path'i Forward+ seç. Çok ışıklı oyunlar (sci-fi koridor, fener taşıyan karakterler, parıltılı RPG) için kritik bir iyileştirmedir.

21.6

🚀 GPU Resident Drawer YENİ

Unity 6'nın en heyecan verici render iyileştirmesi. Klasik draw call mekanizmasında, her frame Unity sahneyi CPU'da tarar, hangi mesh'in nerede olduğunu hesaplar, GPU'ya tek tek gönderir. Bu CPU tarafı bir boğaz'dır.

GPU Resident Drawer, mesh ve transform verilerini GPU belleğinde sürekli tutar. Değişen şeyleri (taşınan objeleri) sadece delta olarak yollar. Static objeleri her frame yeniden göndermez. Üstüne GPU Occlusion Culling (GPU'da çalışan görünmezlik elemesi) ekler — kameranın görmediği objeler hiç çizilmez.

  1. URP Asset → Advanced bölümünde SRP Batcher ☑ açık olmalı.
  2. URP Renderer Asset → GPU Resident Drawer dropdown → Instanced Drawing.
  3. GPU Occlusion Culling ☑ (RT destekli GPU'da daha verimli).
  4. Sahnede mesh sayısı 1000+ olduğunda kazanım belirgin: 30 FPS → 60 FPS gibi.
📊 Sayısal Örnek: Unity'nin demo "Lost Crypt" sahnesi (3000+ static mesh) — eski sistemle 35 FPS, GPU Resident Drawer ile 110 FPS. CPU tarafı %70 daha boş. Bu sayede AI, fizik, gameplay için daha çok bütçe kalır.
21.7

📐 Render Graph — Yeni Mimari YENİ

Unity 6'da hem URP hem HDRP'nin altyapısı Render Graph mimarisine taşındı. Eski sistemde render adımları (depth pre-pass, opaque, transparent, post-process) tek bir uzun lineer fonksiyondu — bir şey değiştirmek istiyorsan tüm zinciri anlamalıydın.

Render Graph mimarisinde her adım bir node'dur ve kaynaklarını (texture, buffer) deklare eder. Unity, frame öncesi bu graph'ı analiz eder:

  • Kullanılmayan node'ları otomatik cull eder (örn. post-process açık değilse o adımları atlar).
  • Render Target'ları tekrar kullanır (memory aliasing).
  • GPU üzerinde paralel iş bölümü.
  • Render Graph Viewer (Window → Analysis → Render Graph Viewer) ile bir frame'in tüm grafiğini görselleştir.

Geliştirici olarak Custom Renderer Feature yazarken yeni API'ler kullanırsın: 

using UnityEngine.Rendering;
using UnityEngine.Rendering.Universal;
using UnityEngine.Rendering.RenderGraphModule;

public class MyCustomPass : ScriptableRenderPass
{
    public override void RecordRenderGraph(RenderGraph renderGraph,
        ContextContainer frameData)
    {
        UniversalResourceData resourceData = frameData.Get<UniversalResourceData>();
        TextureHandle source = resourceData.activeColorTexture;

        using (var builder = renderGraph.AddRasterRenderPass<PassData>(
            "My Custom Effect", out var data))
        {
            builder.SetRenderAttachment(source, 0);
            builder.SetRenderFunc((PassData d, RasterGraphContext ctx) =>
            {
                // GPU komutları burada
            });
        }
    }

    class PassData { }
}
21.8

🌫 Volume Sistemi — Post-Processing & Atmosfer

Eski Post-Processing Stack v2 yerine URP ve HDRP'de Volume Framework kullanılır. Mantığı: sahneye Volume bileşeni olan trigger collider'lar yerleştirirsin, kamera o trigger'a girdiğinde o Volume'ün Profile'ında tanımlı efektler aktif olur.

Global Volume Kurulumu

  1. GameObject → Volume → Global Volume ekle.
  2. Inspector'da Profile alanının yanındaki New'a bas — bir Volume Profile asset oluşur.
  3. Add Override butonuyla efekt ekle: Post-processing → Bloom.
  4. Bloom panelinde Threshold = 0.9, Intensity = 1, Scatter = 0.7 dene.
  5. Sahnedeki parlak nesneler artık glow yapar.

Yaygın Override'lar

  • Bloom — Parlak yüzeylerden taşan ışık halesi.
  • Color Adjustments — Exposure, Contrast, Saturation. Sahnenin genel tonunu kontrol et.
  • White Balance — Sıcak/soğuk renk dengesi.
  • Color Grading (Lift Gamma Gain) — Sinematik renk derecelendirme.
  • Vignette — Köşeleri karart, odak ortada.
  • Film Grain — Parazit, sinema dokusu.
  • Motion Blur — Hızlı hareket bulanıklığı.
  • Depth of Field — Yakın net, uzak bulanık (foto efekti).
  • Tonemapping — HDR → LDR dönüşümü (ACES, Neutral seçenekleri).
  • Chromatic Aberration — Renk kanallarının kayması (cyberpunk/glitch hissi).

Local Volume — Bölgesel Etki

Sahnenin sadece belirli bir bölgesinde farklı atmosfer istiyorsan: Volume bileşenini olan bir GameObject'in üzerine BoxCollider ekle, Is Trigger ☑, Volume'ün Mode'unu Local yap. Kamera o kutuya girince efektler aktif olur. Blend Distance ile yumuşak geçiş ayarlanır (kamera girip çıkarken efektler fade in/out).

Örnek: Bir mağaraya girdiğinde yeşil-mavi color grading aktifleşsin, ana dünyada normal renkler kalsın → mağaraya BoxCollider trigger + Local Volume.

21.9

🎛 Shader Graph — Görsel Shader Yazımı

Shader yazmak için artık HLSL kodu bilmek zorunda değilsin. Shader Graph sürükle-bırak node tabanlı bir editör verir. Hem URP hem HDRP destekler.

  1. Assets → Create → Shader Graph → URP → Lit Shader Graph.
  2. Çift tıklayınca node editörü açılır.
  3. Sol kenarda Blackboard, Property ekle (örn. Color tipinde "BaseColor").
  4. Sağ panelde Master Stack: Vertex, Fragment.
  5. Fragment'in "Base Color" girişine BaseColor property'sini sürükle.
  6. Save Asset → bir Material oluştur, shader'ını bu graph yap.

Yaygın Node'lar

  • Sample Texture 2D — Texture örnekle, UV ile birleşir.
  • Multiply / Add / Lerp — Renk/sayı kombinasyonları.
  • Time — Animasyonlu shader'lar için (Time.x = saniye).
  • Noise — Perlin/Simplex/Voronoi noise.
  • UV → Tiling and Offset — Texture tekrarı/kaydırması.
  • Fresnel — Yüzeye dik bakışlarda farklı görünüş (kalkan, su, hologram).
  • Position / Normal — Vertex pozisyon ve normal.
  • Custom Function — Kendi HLSL kodunu node yap.
💡 Kontrol Noktası: Bir su shader'ı yapmak için: Time × Tiling → Voronoi Noise → Normal Map oluştur → BaseColor mavi+beyaz Lerp → Smoothness 0.9 → Saydam yüzey. Shader Graph ile bu yarım saatlik iş, HLSL'de yarım günlük iş.
21.10

🌅 HDRP'ye Hızlı Bakış

HDRP, URP'nin tüm özelliklerinin yanı sıra cinematic kalite hedefler. Önemli ek özellikler:

  • Physically Based Sky — Atmosfer fiziksel modellemesi, gerçekçi gün döngüsü.
  • Volumetric Fog & Clouds — Hacimsel sis ve bulutlar.
  • Subsurface Scattering — Cilt, mum, mermer gibi yarı-saydam materyaller.
  • Screen Space Reflections / Refractions — Su, cam, ayna.
  • Ray Tracing — DXR/RTX destekli yansıma, gölge, GI.
  • Light Layers — Bazı ışıklar sadece belirli mesh katmanlarını aydınlatsın.
  • Decal Layers — Decaller (kan izleri, yol çizgileri) seçici uygulansın.
  • Custom Pass — İleri seviye render geçişleri.

HDRP'nin sistem gereksinimleri yüksektir: minimum DX11 GPU, önerilen RTX serisi. Mobil ve WebGL desteklenmez. AAA cinematic deneyimi yoksa URP daha mantıklı bir seçimdir.

21.11

🧩 Renderer Feature — Pipeline'ı Uzatmak

URP Renderer Asset'inin altında Add Renderer Feature butonu vardır. Ekleyebileceğin hazır feature'lar:

  • Render Objects — Belirli bir layer'daki objeleri ayrı bir pass'te çiz (outline, custom shader uygulamak için).
  • Screen Space Ambient Occlusion (SSAO) — Köşelerde gölge derinliği.
  • Decal — Sahne yüzeylerine "yapıştırılan" texture'lar.
  • Screen Space Shadows — Daha kaliteli güneş gölgesi.
  • Full Screen Pass — Tüm ekrana shader uygula (custom post-process).

Outline Efekti Örneği

  1. Renderer Asset → Add Renderer Feature → Render Objects.
  2. Name: "Outline". Event: AfterRenderingOpaques.
  3. Layer Mask: "Outline" (önceden bir layer oluştur).
  4. Material Override: Bir outline shader'ı (Cull = Front, vertex'leri normal yönünde genişlet).
  5. Karakterin bir kopyasını "Outline" layer'ına koy ve biraz büyük render et.
21.12

📊 Pipeline Performans İpuçları

  • SRP Batcher ☑ açık tut — aynı shader'a sahip objeleri tek draw call'da çizer.
  • GPU Instancing ☑ materyallerde aç — aynı mesh'in 100 örneğini tek draw call.
  • Mobil'de HDR off, MSAA 2x, Render Scale 0.8.
  • Shadow Distance'ı abartma — 30-50 m mobil için yeterli.
  • Per Object Light Limit 4'te tut (mobil için).
  • Light Probes + Baked Lighting ile runtime ışık yükünü azalt.
  • Post-process zincirinde Bloom + Color Adjustments + Tonemapping yeterli; her efekt %1-3 GPU yer.
  • Volume Profile'ları tek bir asset'e topla, sahne başına kopyalama.
  • Frame Debugger (Window → Analysis → Frame Debugger) ile draw call sıralamasını incele.
🎨 Bölüm Kontrol Noktası: URP = çapraz platform varsayılan, HDRP = AAA cinematic, Built-in = legacy. Unity 6 Forward+ ile çok ışıkta performans, GPU Resident Drawer ile binlerce mesh ucuz. Volume sistemi post-process'i ve atmosferi yönetir. Shader Graph node tabanlı shader. Renderer Feature pipeline'a custom geçiş ekler.

🎯 Pratik Projeler

Proje 1: URP Sahnesi + Bloom + Color Grading

Hedef: Yeni 3D URP projesi oluştur, sahneye parlak emissive küpler koy, post-process ile sinematik bir görünüm kazandır.

  1. Unity Hub → New Project → 3D (URP) şablonu.
  2. Sahneye 5-6 küp ekle, materyallerinin Emission'ını farklı renklerle (kırmızı, mavi, yeşil, sarı) aç. Intensity 3.
  3. GameObject → Volume → Global Volume.
  4. Volume Profile'a Bloom (Threshold 1, Intensity 1.2, Scatter 0.7), Color Adjustments (Saturation +20, Contrast +10), Tonemapping (ACES) ekle.
  5. Camera'da Post Processing ☑ olduğunu kontrol et.
  6. Karşılaştır: Volume yokken vs varken sahne. Belirgin bir fark gör.

Proje 2: Local Volume ile Mağara Atmosferi

Hedef: Bir koridorun ortasına gelindiğinde renkler soğusun, vignette artsın; dışarı çıkınca normale dönsün.

  1. Koridor yap (ProBuilder ile veya küp duvarlarla).
  2. Empty GameObject → Volume → Box Collider (Is Trigger ☑).
  3. Volume Mode = Local.
  4. Blend Distance = 2 (kamera kutuya yaklaştıkça yumuşak geçiş).
  5. Yeni Volume Profile oluştur, içine White Balance (Temperature -50), Color Adjustments (Saturation -30), Vignette (Intensity 0.5) ekle.
  6. Test: Karakteri koridorun başından sonuna yürüt, atmosferin yumuşakça değiştiğini gör.

Proje 3: Forward+ ile 50 Işıklı Sahne Testi

Hedef: Sahnedeki ışık sayısını arttırarak Forward vs Forward+ performans farkını öğren.

  1. Sahneye 50 adet Point Light yerleştir (script ile rastgele konumlandır).
  2. URP Renderer Asset → Rendering Path = Forward. Bir sahne için FPS not et (Stats penceresinden).
  3. Sahne kayar mı? Per Object Light Limit'i aşar mı uyarı?
  4. Aynı renderer'da Rendering Path = Forward+ yap.
  5. Tekrar oyna, FPS karşılaştır.
  6. Bonus: Frame Debugger ile draw call sayılarını karşılaştır.
